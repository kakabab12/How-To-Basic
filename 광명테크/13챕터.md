`src/pipeline/realtime_loop.py` — 여기서 1~8층을 전부 연결한다.

```python
class GesturePipeline:
    def __init__(self, config):
        self.preprocessor    = Preprocessor(config)
        self.detector        = create_gesture_detector(config)
        self.pose_estimator  = PoseEstimator(config) if config["person_lock"]["enabled"] else None
        self.gesture_filter  = GestureFilter(config)
        self.class_map       = config["model"]["class_map"]
        self.infer_fps_meter = FpsMeter()
        self._person_lock    = None

    def process(self, frame):
        frame = self.preprocessor.preprocess_frame(frame)          # ② 거울
        person_lock = self._ensure_person_lock(frame.shape[1])

        detections = self.detector.infer(frame)                    # ③ 손
        if self.pose_estimator is not None:
            persons = self.pose_estimator.infer(frame)             # ④ 포즈
            person_lock.update(frame, persons)                     # ⑤ 잠금
        observations = person_lock.attach_detections(detections, self.class_map)  # ⑥ 귀속

        gesture_event = self.gesture_filter.filter_observations(observations)     # ⑦ 판정
        self.infer_fps_meter.update()

        annotated = draw_bbox(frame.copy(), detections)
        annotated = draw_person_lock(annotated, person_lock)
        annotated = draw_status(annotated, self.infer_fps_meter.avg_fps, gesture_event)
        return annotated, gesture_event
```

`process()` 한 함수가 **2단계의 흐름도 그대로**다. 여기가 전체 그림을 가장 잘 보여주는 자리다.

**`PoseEstimator(config) if enabled else None`** — 이 한 줄이 8·10단계에서 말한
"끄면 rtmlib을 임포트조차 안 한다"의 실행 지점이다.

#### 왜 `step()`을 밖에서 부르게 했나

```python
class PipelineRunner:
    def step(self):
        frame = self.camera.capture_frame()
        annotated, event = self._pipeline.process(frame)
        if event is not None:
            self._event_sender.send(event)
            if self._on_event is not None:
                self._on_event(event)
        return annotated, event
```

파일 첫 주석이 이유를 설명한다:

> cv2.imshow는 항상 메인 스레드에서 호출해야 하므로, 검출→잠금→판정→시각화는
> PipelineRunner.step()에 담아 두고 scripts/run_demo.py가 자신의 메인 루프에서 매 프레임
> 직접 호출한다 — 카메라 캡처만 CameraStream이 자체 스레드로 돈다.

**스레드는 딱 하나(카메라)만 쓴다.** OpenCV의 GUI 제약(메인 스레드 전용) 때문이고,
7단계의 MediaPipe VIDEO 모드 타임스탬프 요구(단일 스레드 전제)와도 맞아떨어진다.

#### 최종 실행 지점

`scripts/run_demo.py` — **직접 실행하는 유일한 파일**이다.

```python
ROOT_DIR = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))
sys.path.insert(0, ROOT_DIR)          # scripts/에서 실행해도 src/를 찾게 한다

def on_gesture_detected(label: str, confidence: float):
    print(f">>> GESTURE: {label} ({confidence:.2f})")     # ★ 키오스크 연동 지점

def main():
    config = load_config(DEFAULT_CONFIG_PATH)
    init_logging(config)
    runner = run_pipeline(config, on_event=lambda e: on_gesture_detected(e.class_name, e.conf))
    try:
        while True:
            annotated, _event = runner.step()
            cv2.imshow(WINDOW_NAME, annotated)
            if cv2.waitKey(1) & 0xFF == ord("q"):
                break
    finally:
        runner.stop()                  # finally — Ctrl+C에도 카메라를 반드시 놓는다
        cv2.destroyAllWindows()
```

**팀원이 고칠 곳은 `on_gesture_detected()` 하나뿐이다.** 내용을 실제 키오스크 동작으로
바꿔 끼우면 된다. `label`은 항상 `next_item`이다.

> **⚠ 낡은 docstring**: 이 파일 맨 위에는 아직
> `label은 move_left/move_right/select/go_home 등`이라고 적혀 있다. 단순화 전 문구다.
> 이 파일은 낡은 저장소 1 버전과 **바이트 단위로 동일**해서, 단순화 때 손이 안 닿았다.
