# 4챕터 — 설정이 단 하나의 진실 공급원

> **광명테크 제스처 인식** · 4/15챕터 · [📚 목차](README.md)

---

`configs/config.yaml` 첫 줄이 규칙을 선언한다:

> 모든 튜닝값의 단일 출처. 수정은 이 파일에서만 한다.

코드에 매직 넘버를 박지 않았다는 뜻이다. `PersonLock.__init__`을 보면 철저히 지켜진다:

```python
lock_cfg = config["person_lock"]
self._kpt_conf = lock_cfg["kpt_conf_threshold"]
self._lock_frame_count = lock_cfg["lock_frame_count"]
self._follow_radius_px = lock_cfg["follow_radius_ratio"] * frame_width_px
```

**`_ratio`로 끝나는 값은 전부 "프레임 폭 대비 비율"**이고, 생성자에서 픽셀로 환산된다.
해상도를 720p에서 1080p로 바꿔도 설정을 안 고쳐도 되게 만든 장치다.
그래서 `PersonLock`은 **프레임 폭을 알아야만 생성**할 수 있고, 첫 프레임이 와야 폭을
알 수 있으므로 지연 생성한다:

```python
self._person_lock = None  # 첫 프레임의 폭을 알아야 만들 수 있다

def _ensure_person_lock(self, frame_width_px):
    if self._person_lock is None:
        self._person_lock = PersonLock(self._config, frame_width_px)
    return self._person_lock
```

`config_loader.py`는 **상대 경로를 절대 경로로 바꾸는 일**을 한다. 어느 폴더에서 실행하든
모델을 찾게 하려는 것이다:

```python
root_dir = os.path.dirname(os.path.dirname(os.path.abspath(config_path)))
config["model"]["mediapipe"]["hand_landmarker_path"] = os.path.join(
    root_dir, config["model"]["mediapipe"]["hand_landmarker_path"]
)
```

**설정 전체 목록:**

| 키 | 값 | 의미 |
|---|---|---|
| `camera.device_id` | `0` | 웹캠 번호 |
| `camera.mirror` | `true` | 거울 모드 (6·9단계에서 중요) |
| `camera.windows_backend` | `auto` | `dshow`/`msmf` 선택 (5단계) |
| `camera.fourcc` | `mjpg` | USB 대역폭 때문 (5단계) |
| `model.mediapipe.flip_orientation` | **`true`** | 손등 판정 부호 반전 (7단계) |
| `model.mediapipe.flip_handedness` | `true` | 좌/우 라벨 반전 (7단계) |
| `model.mediapipe.num_hands` | `2` | 동시 추적 손 개수 |
| `model.mediapipe.bbox_pad_ratio` | `0.15` | 랜드마크→박스 패딩 |
| `model.device` | `cpu` | 전용 GPU 미확인 |
| `model.pose_mode` | `lightweight` | RTMPose 크기 (8단계) |
| `model.max_infer_fps` | `30` | **⚠ 코드 어디서도 안 쓴다** — 죽은 설정 |
| `person_lock.enabled` | `true` | **끄면 이벤트가 안 뜬다** (10단계) |
| `person_lock.kpt_conf_threshold` | `0.3` | 키포인트 최소 신뢰도 |
| `person_lock.lock_frame_count` | `3` | 잠금 확정 프레임 |
| `person_lock.follow_radius_ratio` | `0.25` | 추적 반경 |
| `person_lock.release_sec` | `2.0` | 잠금 해제 시간 |
| `person_lock.wrist_match_ratio` | `0.14` | 손목 귀속 거리 |
| `person_lock.sharpness_weight` | `0.5` | 선명도 vs 크기 비중 |
| `detect.conf_threshold` | `0.5` | 검출 신뢰도 |
| `detect.cooldown_sec` | `1.0` | 연타 방지 |
| `gestures.next_item.stable_frame_count` | `5` | 확정 프레임 수 |
| `gestures.next_item.max_static_move_ratio` | `0.08` | 정적 인정 이동량 |

> **`max_infer_fps: 30`은 죽은 설정이다.** "추론 루프 FPS 상한 — 초과분은 대기"라고
> 주석이 붙어 있지만 `grep`해 보면 `config.yaml` 외에는 **어디서도 읽지 않는다.**
> 이 값을 바꿔도 아무 일도 일어나지 않는다.
