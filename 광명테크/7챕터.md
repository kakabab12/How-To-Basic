> **광명테크 제스처 인식** · 7/15챕터 · [📚 목차](README.md)

---

`src/inference/detector_mediapipe.py` — **이 프로젝트의 심장**이다.

#### 아이디어

손가락을 폈는지 굽혔는지 재지 않는다. **손이 뒤집혔는지(회전 방향)만** 본다.

랜드마크는 단 3개만 쓴다:

```python
LM_WRIST = 0        # 손목
LM_INDEX_MCP = 5    # 검지 뿌리
LM_PINKY_MCP = 17   # 새끼 뿌리
```

**왜 이 3개인가**: 손목과 손가락 뿌리(MCP)는 **손바닥 뼈**라서 주먹을 쥐어도 거의 안 움직인다.
손가락 끝(TIP)은 접히면 랜드마크가 요동친다. 그래서 안 쓴다.

#### 원리

손목에서 검지 뿌리로 가는 벡터 `v1`, 손목에서 새끼 뿌리로 가는 벡터 `v2`를 만든다.
**두 벡터의 외적(cross product) 부호**가 손바닥/손등에 따라 뒤집힌다.

손을 180° 뒤집으면 검지와 새끼의 **좌우 배치가 바뀌기** 때문이다. 오른손과 왼손은 서로
거울상이므로 판정 기준도 반대다.

```python
def is_back_of_hand(landmarks, handedness_label, flip_orientation=False):
    wrist     = landmarks[LM_WRIST]
    index_mcp = landmarks[LM_INDEX_MCP]
    pinky_mcp = landmarks[LM_PINKY_MCP]

    v1x, v1y = index_mcp[0] - wrist[0], index_mcp[1] - wrist[1]
    v2x, v2y = pinky_mcp[0] - wrist[0], pinky_mcp[1] - wrist[1]
    cross = v1x * v2y - v1y * v2x        # 2D 외적 = z 성분만

    if handedness_label == "right":
        is_palm_facing = cross > 0
    elif handedness_label == "left":
        is_palm_facing = cross < 0       # 왼손은 거울상이라 부호 반대
    else:
        return False

    if flip_orientation:
        is_palm_facing = not is_palm_facing
    return not is_palm_facing
```

**이 함수의 장점 3가지:**

| 장점 | 설명 |
|---|---|
| **학습 불필요** | 곱셈 4번, 뺄셈 3번. 데이터 수집도 학습도 없다 |
| **순수 함수** | `mediapipe` 없이 좌표 배열만 넣으면 테스트된다 → `test_mediapipe_classify.py` 11건 |
| **견고함** | 손바닥 뼈 3점만 쓰므로 주먹을 쥐어도 안 흔들린다 |

#### `flip_orientation` — 이론이 틀렸던 기록

부호는 **이론적으로 유도**했는데, 실측해 보니 반대였다. `config.yaml` 주석이 그 순간을
기록하고 있다:

```yaml
flip_orientation: true    # 손등 판정 부호 반전 — 실측 결과 이론상 유도했던 부호가
                          #   반대였음(손바닥을 손등으로 오판) 2026-07-15 확인,
                          #   그래서 기본값을 true로 바꿈
```

> **⚠ 함정**: `is_back_of_hand`의 docstring에는 아직 `실측 전 이론적으로 유도한 부호라
> 실제 카메라로 검증 필요`라고 적혀 있고, README 표에도 기본값이 `false`로 적혀 있다.
> **둘 다 낡았다.** `config.yaml`이 맞다. (15단계 참고)

#### handedness — MediaPipe의 알려진 버그를 우회한다

```python
def user_side_from_label(label, is_mirror, flip_handedness):
    """공식 문서는 "거울(셀피) 입력 가정" 라벨이라고 하지만, Tasks API에서 라벨이
    반대로 나오는 사례가 보고되어 있다(google-ai-edge/mediapipe#4724)."""
    if label not in OPPOSITE_SIDE:
        return None
    side = label if is_mirror else OPPOSITE_SIDE[label]   # 문서 기준 매핑
    return OPPOSITE_SIDE[side] if flip_handedness else side
```

**보정이 2단이다**: ① 거울 모드 보정(문서 기준) → ② `flip_handedness`로 다시 뒤집기
(문서가 틀린 경우 대비). 구글 이슈 번호까지 주석에 박아 뒀다.

#### 추론 루프

```python
def infer(self, frame):
    h_px, w_px = frame.shape[:2]
    rgb = frame[:, :, ::-1]        # BGR → RGB (OpenCV vs MediaPipe 규약 차이)
    mp_image = self._mp.Image(image_format=self._mp.ImageFormat.SRGB, data=rgb.copy(order="C"))

    # VIDEO 모드는 단조 증가 타임스탬프(ms)를 요구한다 — 추론 루프 단일 스레드 전제
    timestamp_ms = max(int(self._clock() * 1000.0), self._last_timestamp_ms + 1)
    self._last_timestamp_ms = timestamp_ms
    result = self._landmarker.detect_for_video(mp_image, timestamp_ms)

    detections = []
    for hand_landmarks, handedness in zip(result.hand_landmarks, result.handedness):
        conf = float(handedness[0].score) if handedness else 1.0
        if conf < self._conf_threshold:
            continue
        raw_label = handedness[0].category_name.lower() if handedness else None
        points_px = [(lm.x * w_px, lm.y * h_px) for lm in hand_landmarks]   # 정규→픽셀
        if not is_back_of_hand(points_px, raw_label, self._flip_orientation):
            continue                                   # ← 손바닥이면 여기서 버린다
        hand_side = user_side_from_label(raw_label, self._is_mirror, self._flip_handedness)
        detections.append(Detection(class_id=0, class_name="손등팔등", conf=conf,
                                    bbox=..., hand_side=hand_side))
    return detections
```

세부 포인트:

- **`RunningMode.VIDEO`** — 이미지 모드보다 빠르다(추적을 이용). 대신 **단조 증가하는
  타임스탬프**를 요구하고, 그래서 **추론 루프가 단일 스레드여야 한다**는 전제가 생긴다.
  `max(..., last+1)`은 같은 밀리초에 두 프레임이 와도 깨지지 않게 하는 방어다.
- **랜드마크는 0~1 정규 좌표**로 나오므로 `lm.x * w_px`로 픽셀로 바꾼다.
- **손바닥이면 `continue`** — 즉 이 검출기는 **손등일 때만 Detection을 만든다.**
  `class_name`은 항상 `"손등팔등"` 하나뿐이다.

> **⚠ 낡은 docstring**: 클래스 설명에 `infer(frame) -> list[Detection] (fist만)`이라고
> 적혀 있다. `fist`는 단순화 전 이름이다. 지금은 `손등팔등`이다.
