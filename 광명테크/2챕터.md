# 2챕터 — 프레임 한 장이 이벤트가 되기까지

> **광명테크 제스처 인식** · 2/15챕터 · [📚 목차](README.md)

---

먼저 전체 흐름을 외우고 들어가면 나머지가 전부 이 그림의 세부 사항으로 읽힌다.

```
   웹캠
    │  ① CameraStream (전용 스레드, 항상 최신 프레임만)
    ▼
  프레임 (BGR)
    │  ② Preprocessor — 거울 반전
    ▼
  ┌──────────────┬──────────────────┐
  │              │                  │
  ▼ ③           ▼ ④                │  두 모델이 같은 프레임을
MediaPipe     RTMPose               │  각자 본다
Hand Landmarker (rtmlib)            │
  │ 손 21점      │ 사람 17점         │
  ▼              ▼                  │
손등인가?      누가 사용자인가?       │
(외적 부호)    (얼굴 크기×선명도)     │
  │              │ ⑤ PersonLock     │
  └──────┬───────┘                  │
         ▼ ⑥ attach_detections      │
   "잠긴 사용자의 손"만 남김          │
   + 팔꿈치 보이는지 확인             │
         │                          │
         ▼ HandObservation          │
         │  ⑦ GestureFilter (FSM)   │
         │     5프레임 연속 유지?     │
         ▼                          │
    GestureEvent(next_item)         │
         │  ⑧ EventSender           │
         ▼                          │
   console 로그 / UDP JSON / 콜백 ───┘
```

**두 모델이 병렬로 도는 게 핵심**이다. 손은 MediaPipe가, 사람은 RTMPose가 본다.
서로 다른 모델이지만 **같은 프레임 좌표계**를 쓰므로 픽셀 거리로 그대로 결합할 수 있다.
