**층을 나눈 기준은 "카메라·모델 없이 테스트할 수 있는가"**다.

`postprocess/`(잠금·판정)는 **순수 로직**이라 카메라도 모델도 없이 단위 테스트가 된다.
실제로 테스트 36건 중 25건이 이 두 파일을 검증한다. 반대로 `capture/`, `inference/`는
하드웨어·모델에 묶여 있어 테스트가 어렵다.

**그래서 판정 로직을 검출기에서 최대한 떼어냈다.** 이게 이 프로젝트 구조의 설계 의도다.

`detector.py`는 **계약(contract)만 정의**하는 얇은 파일이다:

```python
@dataclass
class Detection:
    class_id: int
    class_name: str
    conf: float
    bbox: tuple            # (x1, y1, x2, y2) 원본 프레임 픽셀
    hand_side: str = None  # "left"|"right"

def create_gesture_detector(config):
    from src.inference.detector_mediapipe import MediaPipeGestureDetector  # 순환 임포트 방지
    return MediaPipeGestureDetector(config)
```

검출기가 바뀌어도 `infer(frame) -> list[Detection]`만 지키면 나머지 층은 손댈 필요가 없다.
