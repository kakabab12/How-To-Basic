`src/postprocess/person_lock.py` (전반부)

**문제**: 키오스크 앞에 여러 명이 있다. 누구 손을 들을 것인가?

**답**: **오토포커스가 맞은 사람**이 사용자다. 카메라 초점이 맞았다는 건 카메라 앞에
제대로 서 있다는 뜻이다.

#### 점수 = 얼굴 크기 × 초점 선명도

```python
def _laplacian_sharpness(frame, face_box):
    """얼굴 영역의 초점 선명도 — 라플라시안 분산. 클수록 초점이 맞은 것."""
    x1, y1, x2, y2 = face_box
    crop = frame[y1:y2, x1:x2]
    gray = cv2.cvtColor(crop, cv2.COLOR_BGR2GRAY)
    return float(cv2.Laplacian(gray, cv2.CV_64F).var())
```

**라플라시안 분산**은 고전적인 초점 측정법이다. 라플라시안은 2차 미분이라 **경계(edge)에서
크게 반응**하는데, 초점이 흐리면 경계가 뭉개져 분산이 작아진다.

```python
def _score(self, frame, person):
    face_box = _face_box_from_head(person.head_points, frame.shape)
    if face_box is None:
        return None, None
    x1, y1, x2, y2 = face_box
    area_ratio = ((x2 - x1) * (y2 - y1)) / float(frame.shape[0] * frame.shape[1])
    sharpness = self._sharpness_fn(frame, face_box)
    sharpness_norm = sharpness / (sharpness + SHARPNESS_SQUASH)   # 0~1로 압축
    weight = self._sharpness_weight
    return (1.0 - weight) * min(area_ratio * 10.0, 1.0) + weight * sharpness_norm, face_box
```

두 값의 **정규화 방식**을 눈여겨볼 것:

- **선명도**: `v / (v + K)` where `K = SHARPNESS_SQUASH = 300.0`.
  라플라시안 분산은 상한이 없어서(0~수천) 그냥 쓸 수 없다. 이 식은 **0~1로 부드럽게 압축**하며
  `v = K`일 때 정확히 0.5가 된다.
- **크기**: `min(area_ratio * 10.0, 1.0)` — 얼굴이 화면의 10%를 넘으면 만점.

그리고 `sharpness_weight=0.5`로 **가중 평균**한다(0이면 크기만, 1이면 선명도만).

#### 잠금 → 추적 → 해제 (상태 기계)

```python
def update(self, frame, persons):
    if not self.enabled:
        return None
    now_sec = self._clock()

    scored = [(score, person, face_box) for person in persons
              if (score := self._score(frame, person)[0]) is not None]

    if self.locked_person is not None:
        return self._follow_locked(scored, now_sec)      # ── 이미 잠김 → 추적

    scored.sort(key=lambda item: item[0], reverse=True)
    _, best_person, best_face_box = scored[0]
    best_center = _center(best_person.bbox)

    is_same_candidate = self._candidate_center is not None and (
        math.dist(best_center, self._candidate_center) <= self._follow_radius_px
    )
    self._candidate_count = self._candidate_count + 1 if is_same_candidate else 1
    self._candidate_center = best_center

    if self._candidate_count >= self._lock_frame_count:   # 3프레임 연속
        self.locked_person = best_person
        ...
```

**왜 3프레임 연속을 요구하나?** 한 프레임만 보고 잠그면, 지나가던 사람이 순간적으로 1등이
되는 순간 잠겨 버린다. **연속성**을 요구해 노이즈를 거른다. 이 패턴(N프레임 연속 = 확정)은
11단계의 제스처 판정에서 **똑같이 반복**된다.

```python
def _follow_locked(self, scored, now_sec):
    locked_center = _center(self.locked_person.bbox)
    best_match, best_dist = None, None
    for _, person, face_box in scored:
        dist = math.dist(_center(person.bbox), locked_center)
        if dist <= self._follow_radius_px and (best_dist is None or dist < best_dist):
            best_match, best_dist = (person, face_box), dist

    if best_match is not None:
        self.locked_person, self.locked_face_box = best_match   # 위치 갱신
        self._last_seen_sec = now_sec
        return self.locked_person

    if now_sec - self._last_seen_sec > self._release_sec:        # 2초 미검출
        logger.info("사용자 잠금 해제 — %.1f초 미검출", now_sec - self._last_seen_sec)
        self.locked_person = None
        ...
```

**추적은 "가장 가까운 사람"으로 한다** — 얼굴 인식이나 ReID 같은 무거운 기법 대신,
**반경 안에서 제일 가까운 사람 = 같은 사람**이라고 가정한다. 프레임 간 사람은 조금밖에
못 움직이므로 성립한다. 싸고 충분한 선택이다.

**해제는 즉시 하지 않는다** — 2초(`release_sec`)를 기다린다. 포즈 추정이 한두 프레임
놓쳤다고 바로 풀리면 사용 중에 잠금이 계속 끊긴다.
