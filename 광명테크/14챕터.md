**그런데 이 분류기는 실행되지 않는다.** `hand_pose_classifier`를 임포트하는 곳을 전부 찾으면:

```
scripts/collect_landmarks.py    ← 데이터 녹화용
scripts/train_classifier.py     ← 학습용
```

**이 둘뿐이다.** `realtime_loop.py`도 `detector_mediapipe.py`도 쓰지 않는다.
즉 **실행 경로는 여전히 7단계의 기하 규칙**이고, 학습 분류기는 나란히 놓인 **별도 실험**이다.
README도 이를 인정한다 — "남아있는 미사용 코드 — 정리 필요 여부 확인 요망".

#### 그래도 볼 가치가 있는 이유 — `normalize_landmarks()`

```python
def normalize_landmarks(points_px):
    """21개 (x, y) 픽셀 좌표 -> 손목 원점 + 손크기(손목~중지MCP 거리) 스케일로
    정규화한 42차원 벡터. 카메라 거리·손 크기·사람 차이를 흡수한다."""
    pts = np.asarray(points_px, dtype=np.float32)
    wrist = pts[WRIST_IDX]
    scale = max(float(np.linalg.norm(pts[MIDDLE_MCP_IDX] - wrist)), 1e-6)
    normalized = (pts - wrist) / scale
    return normalized.reshape(-1).astype(np.float32)
```

**정규화 2단계**가 교과서적이다:

1. **평행이동 제거** — 손목을 원점으로 (`pts - wrist`). 손이 화면 어디에 있든 같아진다.
2. **스케일 제거** — 손목~중지MCP 거리로 나눈다. 카메라에서 멀든 가깝든, 손이 크든 작든 같아진다.

`max(..., 1e-6)`은 0으로 나누기 방어다.

그리고 docstring에 **가장 중요한 규칙**이 적혀 있다:

> normalize_landmarks()는 데이터 녹화·학습·추론이 전부 공유하는 단일 소스다 —
> 정규화가 어긋나면 학습된 모델이 실전에서 무조건 틀어지기 때문에 절대 따로 구현하면 안 된다.

**학습/추론 전처리 불일치(train-serving skew)**는 ML에서 가장 흔하고 찾기 어려운 버그다.
그걸 "함수 하나를 세 곳이 공유"로 막았다.

#### 학습해 보려면

```bash
py scripts/collect_landmarks.py   # 웹캠으로 손 랜드마크 녹화 → data/<label>/*.npy
py scripts/train_classifier.py    # PyTorch 학습 → ONNX export (softmax 포함)
```

`data/`는 `.gitignore`에 있다 — `collect_landmarks.py가 녹화한 원본 손 랜드마크
(개인 녹화 데이터라 git에 안 올림)`. **재현하려면 처음부터 다시 녹화**해야 한다.

ONNX가 **softmax를 포함해서** export되므로 추론 출력이 곧 확률이다:

```python
def predict(self, points_px):
    feature = normalize_landmarks(points_px)[None, :]     # (1, 42)
    (probs,) = self._session.run(None, {self._input_name: feature})
    idx = int(np.argmax(probs[0]))
    return self._labels[idx], float(probs[0][idx])
```
