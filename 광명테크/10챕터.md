`src/postprocess/person_lock.py` (후반부) — **`attach_detections()`**

이제 손(MediaPipe)과 사람(RTMPose)을 **결합**한다. 이 프로젝트에서 가장 조건이 많은 함수다.

#### 거울 보정 — 여기서 좌우를 뒤집는다

```python
def user_wrists(self):
    """잠긴 사용자의 손목 좌표를 '사용자 기준' 좌/우로 돌려준다"""
    model_left  = self.locked_person.wrist(KPT_LEFT_WRIST, self._kpt_conf)
    model_right = self.locked_person.wrist(KPT_RIGHT_WRIST, self._kpt_conf)
    if self._is_mirror:
        # 거울 프레임에서 포즈 모델의 '왼손목'은 사용자의 실제 오른손이다
        return {"left": model_right, "right": model_left}
    return {"left": model_left, "right": model_right}
```

6단계에서 예고한 거울 문제의 처리 지점이다. **포즈 모델은 "화면에 보이는 해부학적 좌우"**를
말하는데, 거울 반전된 프레임에서는 사용자가 느끼는 좌우와 반대다.

#### 귀속 규칙 — 다른 사람 손을 어떻게 막나

```python
for det in detections:
    gesture = class_map.get(det.class_name)
    if gesture is None:
        continue
    det_center = _center(det.bbox)
    side_hint = getattr(det, "hand_side", None)

    if side_hint in ("left", "right"):
        wrist = wrists[side_hint]
        if wrist is not None:
            if math.dist(det_center, wrist) > self._wrist_match_px:
                continue   # 잠긴 사용자의 해당 손목 근처가 아니다 — 다른 사람 손
        elif not self._is_near_locked_person(det_center):
            continue       # 손목 키포인트 소실 — 잠긴 사람 박스 밖이면 무시
        best_side = side_hint
    else:
        # hand_side가 없으면 가장 가까운 손목으로 귀속
        ...
    if not self._is_elbow_visible(best_side):
        continue           # 손만 보이고 팔꿈치가 프레임 밖 — "손등팔등" 요건 미충족
```

**우선순위 설계가 핵심**이다:

1. **1순위는 MediaPipe의 `hand_side`** — 손 모델이 "이건 왼손"이라고 말한 걸 믿는다.
2. **손목 거리는 "소유권 검사"로만 쓴다** — 좌우를 정하는 데 쓰는 게 아니라,
   "이게 **잠긴 사용자의** 손인지" 확인하는 데만 쓴다.
3. **손목이 `None`이면 박스 근접으로 폴백** — 주석이 이유를 밝힌다:
   `(한쪽 팔 없음·가림 — 포즈 모델이 환각하기 쉬운 상황)`

> **이 설계 덕분에 한쪽 팔이 없는 사용자도 인식된다.** 포즈 모델이 손목을 못 찾아도
> MediaPipe가 손만 찾으면, 잠긴 사람 박스 근처인지로 대신 판단한다. **배리어프리가
> 목적인 프로젝트답게 접근성이 코드 구조에 반영**되어 있다.

#### 팔꿈치 게이트 — 이벤트가 안 뜨는 진짜 이유

```python
def _is_elbow_visible(self, user_side):
    """"손등팔등" 판정은 손뿐 아니라 팔꿈치까지 보여야 한다는 요건이라 —
    손만 프레임 가장자리에 걸쳐 들어온 경우(팔 전체가 안 보임)를 걸러낸다."""
    if self.locked_person is None:
        return False
    model_left_idx, model_right_idx = (
        (KPT_RIGHT_ELBOW, KPT_LEFT_ELBOW) if self._is_mirror else (KPT_LEFT_ELBOW, KPT_RIGHT_ELBOW)
    )
    model_idx = model_left_idx if user_side == "left" else model_right_idx
    return self.locked_person.keypoints[model_idx][2] >= self._kpt_conf
```

**"손등팔등"이라는 이름의 뜻이 여기 있다** — 손등**과 팔**(팔꿈치)이 둘 다 보여야 한다.

이게 **`person_lock.enabled: false`로 끄면 이벤트가 아예 안 뜨는 이유**다:

```
enabled: false → PoseEstimator를 아예 생성 안 함 (13단계)
              → 팔꿈치 키포인트가 존재하지 않음
              → 하지만 enabled=false 분기는 팔꿈치 검사를 건너뛴다
```

정확히는, `enabled=false`면 `attach_detections`의 **앞쪽 분기**로 빠져서 팔꿈치 검사 없이
화면 좌우 절반으로 귀속한다. 즉 **끄면 팔꿈치 요건이 사라진다.** 다만 잠금이 없으면
`locked_person is None`이라 `enabled=true`인데 잠금 전이면 **빈 목록**을 준다:

```python
if self.locked_person is None:
    return observations    # 잠금이 없으면 빈 목록 — 다른 사람 손을 절대 통과시키지 않는다
```

**"잠금 없으면 아무것도 통과 안 됨"이 기본값**이다. 안전한 방향으로 실패하는(fail-safe) 설계다.

#### 출력 — HandObservation

```python
@dataclass
class HandObservation:
    side: str        # "left" | "right" — 사용자 기준 좌/우
    gesture: str     # 표준 제스처 이름 (class_map 적용 후)
    conf: float
    cx_ratio: float  # 프레임 폭 대비 중심 x (0.0~1.0)
```

`cx_ratio`가 **비율**인 데 주목. 11단계의 "제자리에 있었나" 판정이 해상도와 무관해진다.
