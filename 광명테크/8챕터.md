`src/inference/pose_estimator.py`

**왜 손만으로 부족한가?** 옆 사람이 손을 흔들면 그것도 인식된다. **누구 손인지** 알아야 한다.
그러려면 사람 몸을 봐야 한다.

`rtmlib`(RTMPose, Apache-2.0)를 쓴다. 출력은 **COCO 17 키포인트** 표준:

```python
KPT_NOSE = 0
KPT_HEAD_INDICES = (0, 1, 2, 3, 4)  # 코·양눈·양귀 — 얼굴 영역 추정에 사용
KPT_LEFT_WRIST = 9
KPT_RIGHT_WRIST = 10
```

```python
class PoseEstimator:
    def __init__(self, config):
        from rtmlib import Body   # 무거운 의존 — person_lock을 끈 환경에선 임포트하지 않는다
        self._body = Body(mode=model["pose_mode"], backend="onnxruntime", device=model["device"])
```

**`import`가 함수 안에 있다.** `person_lock.enabled=false`면 `PoseEstimator` 자체를 안 만들고,
그러면 `rtmlib`을 임포트조차 하지 않는다. 무거운 라이브러리를 안 쓸 때 안 불러오는 최적화다.

```python
def infer(self, frame):
    keypoints_xy, scores = self._body(frame)        # (N,17,2), (N,17)
    persons = []
    for xy, score in zip(keypoints_xy, scores):
        keypoints = np.concatenate([xy, score[:, None]], axis=1).astype(np.float32)  # (17,3)
        bbox = _bbox_from_keypoints(keypoints, self._kpt_conf_threshold, frame.shape)
        if bbox is None:
            continue
        head_points = [(kp[0], kp[1]) for i in KPT_HEAD_INDICES
                       if keypoints[i][2] >= self._kpt_conf_threshold]
        persons.append(PersonPose(bbox=bbox, conf=float(score.mean()),
                                  keypoints=keypoints, head_points=head_points))
    return persons
```

- **사람 박스가 없다** — RTMPose는 키포인트만 준다. 그래서 `_bbox_from_keypoints()`로
  **신뢰도 통과한 키포인트를 감싸는 박스**를 직접 만든다(패딩 10%). 추적용이다.
- **`(x, y)`와 `score`를 `(17, 3)`으로 합친다** — 이후 `keypoints[i][2]`가 신뢰도가 된다.
- `PersonPose.wrist(index, min_conf)`는 신뢰도 미달이면 **`None`을 준다.** "손목이 안 보임"을
  명시적으로 표현하는 것이고, 10단계에서 이 `None` 처리가 중요해진다.

```python
def wrist(self, index, min_conf):
    x, y, conf = self.keypoints[index]
    if conf < min_conf:
        return None
    return float(x), float(y)
```

> **모델은 첫 실행 때 자동 다운로드**된다(약 40MB, `~/.cache/rtmlib`). 인터넷이 필요하고
> 첫 실행이 느린 이유가 이것이다.
