#### ⚠ 모델 파일을 직접 받아야 한다

README는 `hand_landmarker.task`가 `이미 받아뒀으므로 추가로 할 일 없다`고 하지만
**`.gitignore`에 `models/weights/*.task`가 있어서 git에 없다.** 클론하면 없다:

```bash
curl -o models/weights/hand_landmarker.task \
  https://storage.googleapis.com/mediapipe-models/hand_landmarker/hand_landmarker/float16/1/hand_landmarker.task
```

#### 실행

```bash
py scripts/run_demo.py
```

주먹 쥔 채 **손등 + 팔꿈치**를 카메라로 향하고 유지 → `>>> GESTURE: next_item (0.9x)`.
`q`로 종료. **첫 실행은 rtmlib이 포즈 모델 40MB를 받느라 느리다** (정상).

#### 알려진 불일치 — 코드와 문서가 어긋난 3곳

| # | 위치 | 문제 | 무엇이 맞나 |
|---|---|---|---|
| 1 | `README.md` 표, `is_back_of_hand` docstring | `flip_orientation` 기본값 `false`라고 함 / "실측 전 이론적 유도" | **`config.yaml`의 `true`가 맞다** (2026-07-15 실측) |
| 2 | `run_demo.py` docstring | `label은 move_left/move_right/select/go_home 등` | **`next_item` 하나뿐** |
| 3 | `detector_mediapipe.py` 클래스 docstring | `infer(frame) -> list[Detection] (fist만)` | **`손등팔등`** |

추가로:

- **`config.yaml`의 `max_infer_fps: 30`은 코드에서 안 읽는다** (죽은 설정)
- **`GestureFilter.filter_observations`의 `raised`/`raised_high` 인자는 죽어 있다**

#### 트러블슈팅

| 증상 | 해결 |
|---|---|
| `카메라(device_id=0)를 열 수 없습니다` | 다른 앱이 점유 중인지 확인 → `device_id`를 1로 → `windows_backend`를 `dshow`/`msmf`로 |
| `모델 파일이 없습니다` | 위 `hand_landmarker.task` 다운로드 |
| 첫 실행이 유독 느림 | 정상 (rtmlib 포즈 모델 최초 다운로드) |
| 손등을 보여도 인식 안 됨 | `flip_orientation` 반전 |
| 손등·팔꿈치 다 보이는데 안 됨 | `kpt_conf_threshold`(0.3)를 낮추거나, 뒤로 물러나 팔꿈치를 프레임 안에 확실히 넣기 |
| 좌/우가 반대 | `flip_handedness` 반전 |
| 가만히 있는데 오탐 | `conf_threshold`↑ 또는 `stable_frame_count`↑ |
| **이벤트가 전혀 안 뜸** | `person_lock.enabled`가 `false`인지 확인 → 반드시 `true` |
