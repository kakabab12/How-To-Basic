# 11챕터 — 7층 판정 FSM — 흔들림을 이벤트로 바꾸기

> **광명테크 제스처 인식** · 11/15챕터 · [📚 목차](README.md)

---

`src/postprocess/gesture_filter.py`

**문제**: 손등이 한 프레임 보였다고 이벤트를 내면, 손을 내리는 도중에 잠깐 손등이 보여도
오작동한다.

**답**: **N프레임 연속 + 거의 제자리**여야 확정한다.

```python
class _StableTracker:
    """정적 상태 유지 판정 — 같은 제스처 N프레임 연속 + 거의 제자리."""

    def update(self, gesture, cx_ratio):
        is_same_run = gesture == self._gesture and (
            abs(cx_ratio - self._start_cx_ratio) <= self._max_move_ratio
        )
        if is_same_run:
            self._count += 1
        else:
            self._gesture = gesture          # 리셋하고 새 런 시작
            self._count = 1
            self._start_cx_ratio = cx_ratio
        if self._count >= self._stable_frame_count:
            self.reset()
            return True
        return False
```

**두 조건을 동시에 본다:**

| 조건 | 값 | 막는 것 |
|---|---|---|
| 같은 제스처 연속 | `stable_frame_count: 5` | 순간적 오검출 |
| 거의 제자리 | `max_static_move_ratio: 0.08` | 손을 흔들며 지나가는 동작 |

**`_start_cx_ratio` 기준으로 비교**하는 게 포인트다. 직전 프레임과 비교하면 천천히 움직이는
손이 매 프레임 "제자리"로 판정되어 결국 통과해 버린다. **런의 시작점**과 비교하므로
누적 이동이 8%를 넘으면 리셋된다.

#### 쿨다운

```python
def filter_observations(self, observations, raised=None, raised_high=None):
    now_sec = self._clock()
    if self._is_in_cooldown(now_sec):
        return None                      # 확정 직후 1초는 입력 전체 무시

    for obs in observations:
        fired = self._tracker.update(obs.gesture, obs.cx_ratio)
        if fired and obs.gesture == BACK_HAND_ARM:
            return self._confirm("next_item", obs.conf, now_sec, hand_side=obs.side)
    return None
```

**입력 이름과 출력 이름이 다르다**:
`obs.gesture == "손등팔등"`(관측된 손 모양) → `class_name="next_item"`(키오스크가 받을 명령).
**물리적 관측**과 **의미적 명령**을 분리한 것이다. 나중에 같은 손짓에 다른 명령을 붙이려면
이 한 줄만 고치면 된다.

#### 죽은 인자

```python
def filter_observations(self, observations, raised=None, raised_high=None):
    """raised/raised_high는 더 이상 쓰지 않지만(예전 손 들기 판정 제거), 호출부
    시그니처 호환을 위해 인자만 받아두고 무시한다."""
```

**단순화의 화석**이다. 예전엔 "손을 얼마나 높이 들었나"도 봤다. 지금은 안 본다.
호출부(`realtime_loop.py`)는 실제로 `filter_observations(observations)`만 호출하므로
**이 두 인자는 완전히 죽어 있다.** 지워도 된다.

#### `_clock` 주입 — 테스트 가능하게 만드는 장치

```python
def __init__(self, config, clock=time.monotonic):
    self._clock = clock
```

`time.monotonic`을 **기본값 인자로 주입**받는다. 테스트에서 가짜 시계를 넣어
**1초를 실제로 기다리지 않고** 쿨다운을 검증할 수 있다. `PersonLock`도 같은 방식이다
(`clock`, `sharpness_fn` 둘 다 주입 가능). 테스트 36건이 빠르게 도는 비결이다.
