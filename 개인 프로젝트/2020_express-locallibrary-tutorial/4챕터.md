# 4챕터 — `bin/www` — 진입점

> **Express 지역 도서관 (2020)** · 4/10챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

```javascript
#!/usr/bin/env node

/**
 * Module dependencies.
 */
var app = require('../app');
var debug = require('debug')('express-locallibrary-tutorial:server');
var http = require('http');

/**
 * Get port from environment and store in Express.
 */
var port = normalizePort(process.env.PORT || '3000');
app.set('port', port);

/**
 * Create HTTP server.
 */
var server = http.createServer(app);

/**
 * Listen on provided port, on all network interfaces.
 */
server.listen(port);
server.on('error', onError);
server.on('listening', onListening);
```

**`express-generator`가 만들어 주는 표준 파일**이다. 사람이 거의 안 고친다.

#### `app.js`와 `bin/www`를 왜 나누나

**Express의 중요한 관습**이다:

| 파일 | 책임 |
|---|---|
| **`app.js`** | **앱이 무엇인가** — 라우트, 미들웨어, 뷰 엔진 |
| **`bin/www`** | **어디서 듣는가** — 포트, HTTP 서버, 에러 처리 |

**왜 나누는 게 좋은가:**

```javascript
// 테스트할 때
const app = require('./app');       // ← 앱만 가져온다
request(app).get('/').expect(200);  // ← 실제 포트를 열지 않고 테스트!
```

**서버를 띄우지 않고 앱을 테스트**할 수 있다. `app.js`가 `listen()`까지 했다면
테스트마다 실제 포트를 점유해서 병렬 테스트가 불가능하다.

> **이 구조는 라즈베리파이 프로젝트의 Flask 코드와 대조된다.**
> 거기서는 `app.run(host='0.0.0.0', port=5000)`이 파일 맨 아래에 그냥 붙어 있다.
> Express는 **프레임워크가 분리를 강제**하고, Flask는 개발자에게 맡긴다.

#### `process.env.PORT || '3000'`

**환경변수가 있으면 그걸, 없으면 3000.** 클라우드 배포(Heroku 등)는 포트를 환경변수로
주입하므로 이 한 줄이 배포 호환성을 만든다.

`normalizePort`는 `"3000"`(문자열)을 `3000`(숫자)으로 바꾸고, 명명된 파이프도 처리한다.
