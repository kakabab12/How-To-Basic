# 6챕터 — MongoDB 연결

> **Express 지역 도서관 (2020)** · 6/10챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

```javascript
//Set up mongoose connection
var mongoose = require('mongoose');
var mongoDB = 'mongodb://127.0.0.1:27017/kababdb';
mongoose.connect(mongoDB, { useNewUrlParser: true , useUnifiedTopology: true});
var db = mongoose.connection;
db.on('error', console.error.bind(console, 'MongoDB connection error:'));
```

#### 한 줄씩

| 코드 | 의미 |
|---|---|
| `mongodb://127.0.0.1:27017` | **로컬** MongoDB, 기본 포트 |
| `/kababdb` | **DB 이름** — 본인 아이디에서 따온 것 |
| `useNewUrlParser: true` | 당시 mongoose 5.x의 필수 옵션 (지금은 불필요) |
| `useUnifiedTopology: true` | 새 연결 관리 엔진 (지금은 기본값) |
| `db.on('error', ...)` | **연결 에러를 콘솔에 찍는다** |

#### `useNewUrlParser`가 시대를 말해 준다

**2020년 mongoose 5.x 시절의 옵션**이다. 이걸 안 넣으면
`DeprecationWarning: current URL string parser is deprecated` 경고가 떴다.

**mongoose 6.0(2021)부터는 둘 다 기본값이 되어 넣으면 오히려 경고가 뜬다.**
→ **이 코드가 2020년에 멈춰 있다는 증거**다(10단계에서 다시 다룬다).

#### `console.error.bind(console, '...')`

**`bind`로 첫 인자를 미리 채운 함수**를 만든다:

```javascript
db.on('error', console.error.bind(console, 'MongoDB connection error:'));
// 에러 발생 시 →  console.error('MongoDB connection error:', 실제에러)
```

`(err) => console.error('MongoDB connection error:', err)`와 같다.
**2020년 Node 코드에서 흔하던 관용구**다(화살표 함수가 보편화되기 전 습관).

#### 연결 실패해도 서버는 뜬다

**`mongoose.connect`는 비동기**다. 실패해도 예외를 던지지 않고 `db.on('error')`로
콘솔에 찍기만 한다. **서버는 정상적으로 3000번 포트에 뜬다.**

> 그래서 **MongoDB가 없어도 서버는 시작되고**, `/`에 접속하는 순간에야 문제가 터진다.
> 다만 이 프로젝트는 그 전에 다른 이유로 죽는다 — 7단계.
