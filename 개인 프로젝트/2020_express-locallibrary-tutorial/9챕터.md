# 9챕터 — `package.json` — 의존성과 스크립트

> **Express 지역 도서관 (2020)** · 9/10챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

```json
{
  "name": "express-locallibrary-tutorial",
  "version": "0.0.0",
  "private": true,
  "scripts": {
    "start": "node ./bin/www",
    "devstart": "nodemon ./bin/www",
    "serverstart": "DEBUG=express-locallibrary-tutorial:* npm run devstart"
  },
  "dependencies": {
    "cookie-parser": "~1.4.4",
    "debug": "~2.6.9",
    "express": "~4.16.1",
    "http-errors": "~1.6.3",
    "mongoose": "^5.12.13",
    "morgan": "~1.9.1",
    "pug": "2.0.0-beta11"
  }
}
```

#### 스크립트 3개

| 스크립트 | 명령 | 용도 |
|---|---|---|
| `start` | `node ./bin/www` | **일반 실행** |
| `devstart` | `nodemon ./bin/www` | **개발** — 파일이 바뀌면 자동 재시작 |
| `serverstart` | `DEBUG=...:* npm run devstart` | **디버그 로그까지** |

**`nodemon`이 `dependencies`에 없다.** `devDependencies`에도 없다.
전역 설치(`npm i -g nodemon`)를 전제한 것이고, MDN 튜토리얼이 그렇게 안내한다.

> **`serverstart`는 Windows에서 안 된다.** `DEBUG=... npm run ...`은 **유닉스 셸 문법**이다.
> Windows CMD/PowerShell에서는 `set DEBUG=...` 또는 `cross-env`가 필요하다.
> MDN 튜토리얼을 그대로 복사한 흔적이다.

#### 버전 표기 두 가지

```
"~1.4.4"   → 1.4.x 안에서 최신 (패치만)
"^5.12.13" → 5.x.x 안에서 최신 (마이너까지)
"2.0.0-beta11" → 정확히 이 버전 (범위 없음!)
```

| 기호 | 허용 범위 | 위험도 |
|---|---|---|
| `~` | 패치만 | 낮음 |
| `^` | 마이너+패치 | 중간 |
| 없음 | 고정 | 없음 |

#### `pug: 2.0.0-beta11`이 눈에 띈다

**베타 버전이 고정**되어 있다. Pug 2 정식이 2017년에 나왔고 **3.x가 현재**인데
**beta11**에 묶여 있다.

`express-generator`가 당시 그렇게 생성했던 흔적이고, **아무도 손대지 않았다**는 뜻이다.

#### `private: true`

**npm에 실수로 배포되는 걸 막는다.** `npm publish`를 하면 거부된다.
학교 과제에 적절한 설정이고, `express-generator` 기본값이다.

#### 이 의존성 목록이 2020년을 증명한다

| 패키지 | 이 프로젝트 | 현재(2026) |
|---|---|---|
| `express` | **~4.16.1** (2018) | 4.21 / 5.x |
| `mongoose` | **^5.12.13** (2021) | 8.x |
| `pug` | **2.0.0-beta11** | 3.x |
| `debug` | ~2.6.9 | 4.x |

**전부 4~6년 묵었다.** 10단계의 실행 문제로 이어진다.
