# 3챕터 — ★ '복구' 폴더 이름의 의미

> **Express 지역 도서관 (2020)** · 3/10챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

**폴더 이름이 `복구`(recovery)다.**

```
2020_express-locallibrary-tutorial/
└── 복구/          ← 왜 "복구"인가?
```

이건 단순한 이름이 아니라 **사건의 기록**이다. 7단계에서 볼 증거와 합치면 이야기가 맞는다:

#### 증거 1 — 모델이 없다

```
복구/
├── app.js
├── bin/www
├── routes/       ← 있다
├── views/        ← 있다
├── public/       ← 있다
└── models/       ← ❌ 없다!
```

**MDN 튜토리얼은 `models/` 폴더가 핵심**이다(Book, Author, Genre, BookInstance).
Express 앱에서 `routes`와 `views`만 있고 `models`가 없는 건 **비정상**이다.

#### 증거 2 — 코드가 없는 모델을 부른다

```javascript
// routes/index.js
router.get('/', function(req, res, next) {
  Board.find({}, function (err, board) {      // ← Board 가 어디에도 없다!
      res.render('index', { title: 'Express', board: board });
  });
});
```

**`Board`를 `require`하는 줄이 없다.** 정의도, 임포트도 없다.
**한때는 `models/Board.js`가 있었다는 뜻**이다.

#### 증거 3 — DB 이름이 개인화되어 있다

```javascript
var mongoDB = 'mongodb://127.0.0.1:27017/kababdb';
//                                       ^^^^^^^
```

**`kababdb`** — GitHub 아이디 `kakabab12`와 이어진다.
**MDN 튜토리얼의 기본값이 아니다**(원본은 `local_library`).
**튜토리얼을 그대로 베낀 게 아니라 자기 것으로 바꿔 가며 작업했다**는 뜻이다.

#### 재구성한 이야기

```
1. MDN 튜토리얼을 따라가며 앱을 만든다
2. Board 모델을 직접 만들어 게시판 기능을 붙인다 (튜토리얼에 없는 응용!)
3. 스크린샷 4장을 찍는다 (= 이때는 돌아갔다)
4. ❌ 파일이 날아간다 (실수로 삭제? 디스크 문제?)
5. 남은 조각을 모아 "복구" 폴더에 담는다
6. models/ 는 끝내 못 살렸다
```

> **`Board`는 MDN 튜토리얼에 없는 모델**이다. 튜토리얼은 Book/Author/Genre/BookInstance만
> 다룬다. **본인이 게시판을 추가로 만들었다**는 뜻이고, 하필 **그 부분이 날아갔다.**
