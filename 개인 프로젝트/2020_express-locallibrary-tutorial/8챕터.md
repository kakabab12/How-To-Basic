**Pug**(구 Jade)는 **들여쓰기로 HTML을 쓰는** 템플릿 엔진이다.

#### `layout.pug` — 뼈대

```pug
doctype html
html
  head
    title= title
    link(rel='stylesheet', href='/stylesheets/style.css')
  body
    block content
```

**HTML로 옮기면:**

```html
<!DOCTYPE html>
<html>
  <head>
    <title>{{title}}</title>
    <link rel="stylesheet" href="/stylesheets/style.css">
  </head>
  <body>
    <!-- block content 자리 -->
  </body>
</html>
```

| Pug 문법 | 의미 |
|---|---|
| 들여쓰기 | **중첩** (닫는 태그가 없다!) |
| `title= title` | **`=`는 변수 출력** (왼쪽은 태그, 오른쪽은 변수) |
| `link(rel='...', href='...')` | 괄호 안이 **속성** |
| **`block content`** | **자식이 채울 자리** |

#### `index.pug` — 실제 페이지

```pug
extends layout

block content
  h1= title
  p Welcome to #{title}
```

| 문법 | 의미 |
|---|---|
| **`extends layout`** | `layout.pug`를 **상속** |
| **`block content`** | 부모의 `block content` 자리를 **채운다** |
| `h1= title` | `<h1>Express</h1>` |
| `#{title}` | **문자열 안에 끼워 넣기(보간)** → `<p>Welcome to Express</p>` |

#### 렌더링 흐름

```javascript
res.render('index', { title: 'Express', board: board });
//          ───┬───   ─────────┬──────────────────────
//         views/index.pug     로컬 변수
```

```
index.pug
  ├── extends layout  →  layout.pug 를 불러온다
  ├── title = 'Express' 주입
  └── block content 를 layout 의 body 에 끼워 넣는다
        ↓
  최종 HTML 완성 → 브라우저로
```

#### `board`가 템플릿에서 쓰이지 않는다

```javascript
res.render('index', { title: 'Express', board: board });   // board 를 넘기는데
```

```pug
block content
  h1= title
  p Welcome to #{title}          ← board 를 안 쓴다!
```

**7단계의 `Board.find()`로 게시글을 가져와서 넘기는데, 템플릿은 그걸 무시한다.**

> **3단계의 "복구" 이야기와 맞아떨어진다.** 원래 `index.pug`는 게시글 목록을
> `each item in board` 같은 루프로 그렸을 것이다. **템플릿도 튜토리얼 기본형으로
> 되돌아가 있다** — 즉 `views/index.pug`도 복구 과정에서 원본으로 덮인 것으로 보인다.
>
> **결국 남은 건 라우터 한 줄뿐**이다. `Board.find(...)`가 **"여기 뭔가 있었다"는
> 유일한 흔적**이다.
