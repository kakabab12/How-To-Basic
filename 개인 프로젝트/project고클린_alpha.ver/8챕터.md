# 8챕터 — `set_working_state` — 작업 중 버튼 잠금

> **고클린 alpha.ver** · 8/15챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

7단계에서 스레드를 도입하면 **새로운 문제**가 생긴다: 사용자가 **버튼을 또 누를 수 있다.**

조각 모음이 도는 중에 게임 모드를 누르면 두 작업이 동시에 시스템을 건드린다.

```python
def set_working_state(self, is_working, status_text_en="PROCESSING"):
    lang = self.current_lang

    if self.btn_link is not None:      # 이전 작업이 만든 링크 버튼 제거
        self.btn_link.destroy()
        self.btn_link = None

    if is_working:
        display_text = f"[BUSY] {status_text_en}"
        if lang == "KO":
            if   "UPDATING" in status_text_en: display_text = "[작업 중] 업데이트 확인 중..."
            elif "CLEANING" in status_text_en: display_text = "[작업 중] 시스템 청소 중..."
            elif "GAME"     in status_text_en: display_text = "[작업 중] 게임 최적화 중..."
            elif "PING"     in status_text_en: display_text = "[작업 중] 핑 측정 중..."
            elif "SCANNING" in status_text_en: display_text = "[작업 중] 하드웨어 스캔 중..."

        self.status_title.configure(text=display_text, text_color="#FFD700")   # 금색
        self.progressbar.configure(mode="indeterminate")
        self.progressbar.start()                                                # ★
        for btn in [self.btn_update, self.btn_clean, self.btn_game, self.btn_ping, self.btn_info]:
            btn.configure(state="disabled")                                     # ★ 전부 잠금
    else:
        self.status_title.configure(text=self.ui_locales["done"][lang], text_color=COLOR_ACCENT)
        self.progressbar.configure(mode="determinate")
        self.progressbar.stop()
        self.progressbar.set(1)                                                 # 100%
        for btn in [...]:
            btn.configure(state="normal")                                       # 해제
```

#### 세 가지가 동시에 바뀐다

| 요소 | 작업 중 | 완료 |
|---|---|---|
| **상태 텍스트** | `[BUSY] PING TEST...` (금색 `#FFD700`) | `[OK] JOB FINISHED` (시안) |
| **진행 막대** | `indeterminate` + `start()` — 무한 애니메이션 | `determinate` + `set(1)` — 꽉 참 |
| **버튼 5개** | `disabled` | `normal` |

#### `indeterminate`가 정직하다

**`defrag`가 몇 % 진행됐는지 알 방법이 없다.** `subprocess`는 명령이 끝나야 출력을 준다.

이때 선택지는 둘이다:
- ❌ **가짜 퍼센트를 보여준다** (많은 프로그램이 이렇게 한다)
- ✅ **`indeterminate`** — "얼마나 걸릴지 모르지만 돌고 있다"

**모르는 걸 아는 척하지 않는다.** 좌우로 움직이는 막대가 정확히 그 의미다.

#### 다국어 처리가 여기만 다르다

11단계에서 볼 `ui_locales` 딕셔너리 대신, 여기서는 **`if "UPDATING" in status_text_en`**
같은 문자열 검사를 쓴다.

**일관성이 깨진 부분**이다. `set_working_state(True, "PING TEST...")`처럼 호출부가
**영어 문자열을 넘기기 때문에** 생긴 구조다. 키(`"ping"`)를 넘겼다면 딕셔너리로 통일할
수 있었다:

```python
# 지금
self.set_working_state(True, "PING TEST...")
# 통일한다면
self.set_working_state(True, "ping")   # → self.cmd_locales["busy_ping"][lang]
```

동작에는 문제가 없지만, **언어를 하나 더 추가할 때 두 군데를 고쳐야** 한다.
