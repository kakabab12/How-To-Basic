# 1챕터 — 이 프로그램이 푸는 문제

> **고클린 alpha.ver** · 1/15챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

**"PC가 느려졌을 때 하는 일들"이 너무 흩어져 있다.**

DNS 캐시를 지우려면 CMD를 열어 `ipconfig /flushdns`, 임시 파일은 `%TEMP%` 폴더를 찾아
직접 삭제, 드라이버는 제조사 사이트를 검색, 게임 전에는 작업관리자로 프로그램을 하나씩
종료... **전부 다른 곳에 있고 전부 명령어를 외워야 한다.**

이 프로그램은 그걸 **버튼 5개로** 묶었다.

| 버튼 | 대신 해 주는 일 |
|---|---|
| **SYSTEM UPDATE** | winget·pip 업데이트 + **GPU 제조사 감지** → 드라이버 페이지 |
| **CLEAN & BOOST** | `flushdns` + Temp 삭제 + `dism` + `winsock reset` + `defrag` |
| **GAME MODE** | 무거운 프로그램 일괄 종료 + explorer 재시작 |
| **PING TEST** | 8.8.8.8 / 1.1.1.1 지연 측정 → 등급 판정 |
| **MY PC INFO** | CPU·보드·GPU·RAM + 내부/외부 IP |

**핵심 설계 사상**: *"새로운 걸 만들지 않는다."*
전부 **Windows 내장 명령**(`winget`, `dism`, `netsh`, `defrag`, `taskkill`, `wmic`, `ping`)을
`subprocess`로 부를 뿐이다. 외부 라이브러리도, 설치할 것도 없다.

**그래서 이 프로그램의 진짜 코드는 "명령을 부르는 부분"이 아니라 그 주변**이다 —
권한 획득(4단계), 스레드(7단계), 인코딩(9단계), 창 숨기기(10단계). 시스템 도구를 GUI로
감쌀 때 실제로 어려운 건 전부 거기 있다.
