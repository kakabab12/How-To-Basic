#### ③ `start_gamemode`

본인 설명:

> 게임 프레임에 영향을 줄 수 있는 무거운 백그라운드 프로그램들(크롬, 엣지, 카카오톡,
> 디스코드, 스팀, 오피스 등)의 리스트를 만들어 `taskkill`로 일괄 강제 종료합니다.
> 종료 후 Windows 탐색기(`explorer.exe`)를 재시작하여 메모리를 반환받습니다.

```
taskkill /F /IM chrome.exe
taskkill /F /IM msedge.exe
taskkill /F /IM KakaoTalk.exe
taskkill /F /IM Discord.exe
...
taskkill /F /IM explorer.exe    →    start explorer.exe   ← 재시작
```

**`explorer.exe` 재시작이 포인트다.** 탐색기는 파일 썸네일·아이콘 캐시로 메모리를
수백 MB 쥐고 안 놓는다. 재시작하면 **작업표시줄이 잠깐 사라졌다 돌아오면서 메모리가 반환**된다.

> **⚠ alpha.ver에는 초기.ver의 Y/N 확인이 없다.**
> 배치 버전은 `set /p confirm=`으로 물었는데, GUI 버전은 **버튼을 누르면 바로 죽인다.**
> 크롬에 열어 둔 탭도, 저장 안 한 워드 문서도 그대로 날아간다.
> **`CTkInputDialog`나 확인 팝업을 넣는 게 맞다.** 초기 버전이 더 안전했던 드문 사례다.

#### ④ `start_ping` — 한/영 Windows를 모두 처리하는 정규식

```python
def task():
    def get_ms(ip):
        try:
            out = self.get_cmd_output(f"ping {ip} -n 3")
            m = re.search(r'(평균|Average)\s*=\s*(\d+)ms', out)     # ★
            return int(m.group(2)) if m else None
        except: return None

    def get_stat(ms):
        if ms is None: return "FAIL"
        if ms <= 10: return "EXCELLENT" if self.current_lang == "EN" else "매우 좋음"
        if ms <= 30: return "GOOD" if self.current_lang == "EN" else "좋음"
        val = "NORMAL" if ms <= 60 else "SLOW"
        if self.current_lang == "KO":
            val = "보통" if ms <= 60 else "느림"
        return val

    gms = get_ms("8.8.8.8")      # Google DNS
    cms = get_ms("1.1.1.1")      # Cloudflare DNS
    ...
```

**`r'(평균|Average)\s*=\s*(\d+)ms'`가 이 함수의 백미**다.

| Windows 언어 | `ping` 출력 |
|---|---|
| 한국어 | `최소 = 12ms, 최대 = 25ms, **평균 = 18ms**` |
| 영어 | `Minimum = 12ms, Maximum = 25ms, **Average = 18ms**` |

**정규식 하나로 둘 다 잡는다:**

| 조각 | 역할 |
|---|---|
| `(평균\|Average)` | **그룹 1** — 두 언어를 모두 매칭 |
| `\s*=\s*` | 등호 주변 공백 개수 차이 흡수 |
| `(\d+)` | **그룹 2** — 실제 숫자 |
| `ms` | 단위 |
| `m.group(2)` | **그룹 2**를 꺼낸다 (그룹 1은 언어 판별용) |

> **9단계의 `safe_decode`와 짝을 이룬다.** 인코딩을 뚫고 문자열을 얻어도(9단계),
> **그 문자열의 언어가 다르면** 파싱이 안 된다. 두 방어가 함께 있어야 완성된다.

#### 왜 DNS 서버 2곳을 재나

| IP | 제공 |
|---|---|
| `8.8.8.8` | Google Public DNS |
| `1.1.1.1` | Cloudflare DNS |

**원인을 구분할 수 있다:**

| 결과 | 해석 |
|---|---|
| 둘 다 빠름 | 인터넷 정상 |
| **하나만 느림** | 그 경로/ISP 라우팅 문제 |
| **둘 다 느림** | 내 인터넷 또는 공유기 문제 |

**한 곳만 쟀다면 이 구분이 불가능하다.**

#### 등급 기준

| 지연 | 등급 |
|---|---|
| ≤ 10ms | 매우 좋음 |
| ≤ 30ms | 좋음 |
| ≤ 60ms | 보통 |
| > 60ms | 느림 |

**게임 기준으로 타당한 구간**이다. FPS 게임은 30ms를 넘으면 체감되고, 60ms를 넘으면
불리해진다. `-n 3`(3번만)은 기본 4번보다 빠른 응답을 위한 선택이다.
