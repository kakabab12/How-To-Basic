# 10챕터 — `get_cmd_output` — 검은 창이 번쩍이지 않게

> **고클린 alpha.ver** · 10/15챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

```python
def get_cmd_output(self, cmd):
    try:
        startupinfo = subprocess.STARTUPINFO()
        startupinfo.dwFlags |= subprocess.STARTF_USESHOWWINDOW     # ★
        output = subprocess.check_output(cmd, shell=True, startupinfo=startupinfo)
        return self.safe_decode(output).strip()
    except:
        return ""
```

#### `STARTF_USESHOWWINDOW`가 없으면

**명령을 부를 때마다 검은 CMD 창이 0.1초씩 번쩍인다.**

이 프로그램은 `wmic`, `ping`, `ipconfig`, `powershell`을 **수십 번** 부른다.
내 PC 정보 하나 보려고 눌렀는데 **화면이 난리가 난다.** GUI 앱으로서 치명적이다.

| 필드 | 의미 |
|---|---|
| `STARTUPINFO()` | Windows 프로세스 시작 옵션 구조체 |
| `dwFlags \|= STARTF_USESHOWWINDOW` | "창 표시 방식을 지정하겠다"는 플래그 |
| (기본 `wShowWindow = SW_HIDE`) | **창을 숨긴다** |

**Windows 전용 코드**다. 리눅스/맥에는 `STARTUPINFO`가 없어서 `AttributeError`가 난다.
이 프로그램은 애초에 Windows 전용(`wmic`, `winget`, `defrag`)이라 문제없다.

#### `except: return ""`

**명령이 실패해도 빈 문자열을 준다.**

```python
gpu = self.get_cmd_output("wmic path win32_VideoController get name")
if not gpu:            # ← 호출부가 이렇게 처리할 수 있다
    ...
```

`wmic`은 Windows 11에서 **더 이상 기본 설치가 아니다**(deprecated). 없는 PC에서
`FileNotFoundError`가 나도 **프로그램은 살아 있고 해당 항목만 비어 나온다.**

> **⚠ bare `except:`의 문제**: `except:`는 `KeyboardInterrupt`와 `SystemExit`까지 삼킨다.
> 실무 기준으로는 `except Exception:`이 맞다. 다만 GUI 앱이라 Ctrl+C를 쓸 일이 없어
> 실질적 영향은 없다.

#### 9·10단계를 합치면

```python
get_cmd_output(cmd)
    ├── 창을 숨기고 실행       ← 10단계
    ├── safe_decode 로 디코딩  ← 9단계
    └── 실패해도 "" 반환       ← 10단계
```

**이 한 함수가 "Windows 명령을 GUI에서 안전하게 부르기"의 완성형**이다.
프로그램 전체가 이 함수 하나를 통해서만 시스템과 대화한다.
