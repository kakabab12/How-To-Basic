본인 설명:

> 사이드바에는 버튼을, 메인 영역에는 명령 프롬프트(CMD) 느낌이 나는 터미널 텍스트 박스를
> 배치했습니다.

#### GUI인데 왜 터미널을 흉내 내나

**진행 상황을 보여줄 방법이 필요하다.** `defrag`가 3분 도는 동안 화면이 멈춰 있으면
사용자는 프로그램이 죽은 줄 안다.

**로그를 흘려보내면**:
- 뭔가 **실제로 일어나고 있다**는 증거가 된다
- 무엇을 했는지 **사후에 확인**할 수 있다
- 에러가 나도 **그대로 보인다**

시스템 도구에는 이게 대시보드나 그래프보다 자연스럽다.

#### 자동 스크롤

```python
def log(self, message):
    self.textbox.insert("end", message + "\n")
    self.textbox.see("end")          # ★
```

**`see("end")`가 없으면** 로그가 아무리 쌓여도 화면은 맨 위에 머문다.
사용자가 직접 스크롤을 내려야 최신 내용이 보인다. **한 줄로 해결**한다.

#### 위젯을 인스턴스 변수로 잡는다

```python
self.btn_update = ...
self.btn_clean  = ...
self.btn_game   = ...
self.btn_ping   = ...
self.btn_info   = ...
self.status_title = ...
self.progressbar  = ...
self.textbox      = ...
```

**지역 변수로 만들면 나중에 접근할 수 없다.** 이 프로그램은 나중에
**버튼을 잠그고**(8단계) **텍스트를 다른 언어로 바꿔야**(11단계) 하므로
**전부 `self.`로 붙잡아 둬야** 한다.
