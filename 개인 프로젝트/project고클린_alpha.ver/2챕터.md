> **고클린 alpha.ver** · 2/15챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

**같은 기능을 두 번 만들었다.**

| | `project고클린_초기.ver/` | `project고클린_alpha.ver/` (이 폴더) |
|---|---|---|
| 파일 | `update.bat` | `alpha_test1.py` (568줄) |
| 언어 | **배치 스크립트** | **Python** |
| UI | CMD 창 + ASCII 아트 | **customtkinter 다크 GUI** |
| 실행 | 순차 (기다림) | **멀티스레딩** |
| 다국어 | ❌ | ✅ 한/영 실시간 |
| 배포 | `.bat` | **PyInstaller `.exe`** |

**배치로 먼저 "되는지" 확인하고, Python으로 "제대로" 다시 썼다.**

> 이 패턴은 이 사람의 다른 프로젝트에서도 반복된다 — 라즈베리파이 프로젝트의
> `sub/`(가짜 데이터로 UI 먼저) → `RPI-Flask-main/`(진짜 추론). **"일단 되게 만들고,
> 그다음 제대로 만든다."**

두 버전을 비교하며 읽으면 **같은 문제를 다른 도구로 어떻게 푸는지**가 보인다.
특히 **관리자 권한**(4단계)과 **한글 인코딩**(9단계)이 그렇다.

→ 배치 버전은 [`project고클린_초기.ver/`](../project고클린_초기.ver/README.md) 참고
