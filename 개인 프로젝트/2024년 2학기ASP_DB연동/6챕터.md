```asp
<%
DSNless="DRIVER={Microsoft Access Driver (*.mdb)}; "
DSNless=DSNless & "DBQ=" & server.mappath("nwind.mdb")

Set Conn = Server.CreateObject("ADODB.Connection")
Conn.Open DSNless

Set Rs = Server.CreateObject("ADODB.Recordset")
Rs.Open "Select * From tblCategories", Conn
%>

<html>
<head>
  <meta charset="UTF-8">
</head>
<body>

<table border="2">
  <tr>
   <td>레이블1</td>
   <td>레이블2</td>
   <td>레이블3</td>
  </tr>

<%
while not Rs.eof
%>
  <tr>
    <td>
      <input type="text" name="CID" value="<% = Rs("CategoryID")%>" >
    </td>
    ...
<%
  Rs.MoveNext
wend
%>
</table>
</body>
</html>
```

#### `<% %>`가 HTML 밖으로 나왔다 들어갔다 한다

**이게 클래식 ASP의 핵심 문법**이다:

```
<%  while not Rs.eof  %>        ← 서버 코드: 루프 시작
    <tr>                         ← HTML: 그대로 출력
      <td><% = Rs("CategoryID") %></td>   ← 서버 코드: 값 출력
    </tr>
<%  Rs.MoveNext                  ← 서버 코드
    wend  %>                     ← 루프 끝
```

**루프의 몸통이 HTML**이다. `while` 안에 있는 HTML이 행 수만큼 반복 출력된다.

| 문법 | 의미 |
|---|---|
| `<% ... %>` | **코드 실행** (출력 없음) |
| `<% = 값 %>` | **값 출력** (= `Response.Write 값`) |

> **PHP의 `<?php ?>`, EJS의 `<% %>`가 그대로 이 문법을 물려받았다.**
> EJS는 이름부터 "Embedded JavaScript"로 ASP를 JS로 옮긴 것이다.
>
> **Pug**([Express 8단계](../2020_express-locallibrary-tutorial/README.md))는 정반대
> 철학이다 — HTML을 코드처럼 쓴다. ASP/EJS는 코드를 HTML에 끼워 넣는다.

#### Recordset 커서 조작

```asp
while not Rs.eof     ' 끝(End Of File)이 아닌 동안
    ...
    Rs.MoveNext      ' 다음 행으로 ★
wend
```

**`Rs.MoveNext`를 빠뜨리면 무한 루프**다. 첫 행에서 영원히 멈춘다.
서버가 죽을 때까지 같은 `<tr>`을 뱉는다.

**현대 언어는 이 실수가 불가능하다:**

```python
for row in rows:      # 커서를 언어가 관리한다
    print(row)
```

**ASP는 커서를 직접 움직인다.** 이게 "추상화가 거의 없다"의 실제 의미다 —
**DB 커서라는 개념을 직접 만지게 된다.**

#### `<input type="text">`로 출력하는 이유

```asp
<input type="text" name="CID" value="<% = Rs("CategoryID")%>" >
```

**단순 표시라면 `<td><%=Rs("CategoryID")%></td>`면 된다.**
`<input>`으로 감싼 건 **수정 가능한 목록**을 만들려는 것이다 —
`read2.asp`/`read3.asp`에서 수정 폼으로 발전했을 것으로 보인다(2단계의 숫자 접미사).
