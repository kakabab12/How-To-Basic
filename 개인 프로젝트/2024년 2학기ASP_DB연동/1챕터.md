#### 클래식 ASP는 언제 기술인가

| 시기 | 기술 |
|---|---|
| **1996** | **Classic ASP** 등장 |
| 2002 | ASP.NET 등장 (사실상 후계자) |
| 2016 | ASP.NET Core |
| **2024** | **이 과제** |

**28년 된 기술**이다. 실무에서 신규 개발에 쓸 일은 사실상 없다.

#### 그런데 왜 배우나

**서버사이드 렌더링의 원리는 변하지 않았기 때문**이다.

```
클래식 ASP (1996)              현대 (2024)
──────────────────────────────────────────────
<% ... VBScript ... %>    →    <?php ?> / EJS / Pug / JSX
Request.Form("name")      →    req.body.name
Response.Redirect         →    res.redirect()
ADODB.Connection          →    mongoose / Prisma / SQLAlchemy
Response.Write            →    res.send()
```

**개념이 1:1로 대응된다.** ASP를 이해하면 PHP·Node·Django가 다 같은 그림으로 보인다.
그리고 ASP는 **추상화가 거의 없어서** 무슨 일이 일어나는지가 코드에 다 보인다 —
교육용으로는 그게 장점이다.

> **이 폴더의 진짜 가치**: [Express 프로젝트](../2020_express-locallibrary-tutorial/README.md)와
> 나란히 읽으면 **"프레임워크가 대신 해 주던 일"**이 뭔지 알게 된다.
> Express의 `app.use(express.urlencoded())` 미들웨어가 없으면 어떻게 폼을 받는가?
> → `Request.Form("txtcid")`. 그게 전부다.

#### `iisstart.htm`이 말해 주는 것

**IIS의 기본 시작 페이지**다. 이게 있다는 건 **`C:\inetpub\wwwroot`를 통째로 커밋**했다는 뜻이다.
`root/`라는 폴더 이름도 그래서 나왔다.
