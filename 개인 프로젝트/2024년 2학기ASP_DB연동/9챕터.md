# 9챕터 — `GetEmpInfo.asp` — 자기 자신에게 POST

> **ASP + Access DB 연동 (2024)** · 9/10챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

**독립 예제**다. Northwind의 **직원 검색**.

```asp
<HTML>
<HEAD><TITLE>Getting Employee Info</TITLE></HEAD>
<BODY>
<H1>Employee Info</H1>

<FORM method="post" action="GetEmpInfo.asp">                        ← ★ 자기 자신!
Enter the last name of the employee you want To search:
<BR><INPUT type=text name="empLastName" size=40
           value=<%=Request("empLastName")%> >                       ← ★ 입력 유지
<INPUT type=submit style="width=150" value="Show">
</FORM>
<HR>
<span>Sample names are: <I>Davolio, Fuller, Buchanan</I></span>      ← 친절
<HR><BR>

<%
if Request("empLastName") = "" Then Response.End                     ← ★ 가드
%>

<%
DSNless="DRIVER={Microsoft Access Driver (*.mdb)}; "
DSNless=DSNless & "DBQ=" & server.mappath("nwind.mdb")

sql1 = "select firstname, lastname, city, hiredate, photo from tblEmployees where "
sql2 = "lastname='"
sql3 = Request.Form("empLastName") & "'"
sql = sql1 & sql2 & sql3

Set oRS = Server.CreateObject("ADODB.Recordset")
oRS.CursorLocation = 3
...
```

#### 지금까지와 다른 세 가지

**① 폼과 처리가 한 파일에 있다**

```asp
<FORM method="post" action="GetEmpInfo.asp">     ← 자기 자신에게 POST
```

4단계의 `.htm` + `.asp` 쌍과 **정반대 패턴**이다.

| | 쌍 방식 (`insert.htm`+`insert.asp`) | 자기 POST (`GetEmpInfo.asp`) |
|---|---|---|
| 파일 | 2개 | **1개** |
| 결과 표시 | 리다이렉트해서 못 보여줌 | **폼 아래에 바로** |
| 재검색 | 폼으로 돌아가야 | **바로 다시 입력** |

**검색에는 자기 POST가 맞다.** 결과를 보면서 다시 검색하는 게 자연스럽다.
**삽입에는 쌍 방식이 맞다**(PRG 패턴, 4단계).

> **용도에 따라 패턴을 달리 쓴 것**이다. 그리고 이건 실습 커리큘럼이 **두 가지를 다
> 가르치려고** 배치한 것으로 보인다.

**② `value=<%=Request("empLastName")%>` — 입력값 유지**

검색 후에도 **검색어가 입력창에 남는다.** 없으면 검색할 때마다 빈 칸이 되어
오타를 고치려면 처음부터 다시 쳐야 한다.

> **⚠ 따옴표가 없다.** `value=<%=...%>`는 `value=Davolio`가 된다.
> **공백이 있는 이름을 넣으면 깨진다** (`value=Van Halen` → `value="Van"`으로 잘림).
> `value="<%=...%>"`가 맞다.

**③ `if ... Then Response.End` — 가드 절**

```asp
if Request("empLastName") = "" Then Response.End
```

**빈 검색어면 여기서 응답을 끝낸다.** 아래 DB 코드는 실행되지 않는다.

**처음 페이지를 열었을 때**(POST 전)는 검색어가 없으므로 **폼만 보이고 끝난다.**
같은 파일이 **"폼만"과 "폼+결과"** 두 상태를 처리하는 방법이다.

> **현대의 early return과 같은 개념**이다:
> ```python
> if not query:
>     return render(form)
> ```

#### `Sample names are: Davolio, Fuller, Buchanan`

**Northwind DB의 실제 직원 이름**이다. **사용자가 뭘 입력해야 할지 알려 주는 배려** —
빈 검색 결과로 헤매지 않게 한다. 작지만 좋은 UX다.

#### `oRS.CursorLocation = 3`

**`3` = `adUseClient`** — 커서를 클라이언트(웹 서버) 쪽에 둔다.

| 값 | 상수 | 의미 |
|---|---|---|
| 2 | `adUseServer` | DB 서버가 커서 관리 (기본) |
| **3** | **`adUseClient`** | **전체를 가져와 메모리에** |

**클라이언트 커서를 쓰면 `RecordCount`를 쓸 수 있고 정렬·필터가 자유롭다.**
Access 같은 파일 DB에서는 흔한 선택이다.

**`3`이라는 매직 넘버**로 쓴 게 아쉽다 — ASP에서는 `adovbs.inc`를 include하면
`adUseClient`라는 이름을 쓸 수 있었다.

#### `photo` 컬럼

```sql
select firstname, lastname, city, hiredate, photo from tblEmployees
```

**`photo`는 이미지 BLOB**이다. Northwind에는 직원 사진이 DB 안에 바이너리로 들어 있다.
ASP에서 이걸 표시하려면 별도 처리가 필요한데, 이 예제가 어디까지 했는지는
파일 뒷부분을 봐야 한다.
