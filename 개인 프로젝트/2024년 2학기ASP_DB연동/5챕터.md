# 5챕터 — `insert.asp` — 삽입, 그리고 주석 처리된 두 번째 방법

> **ASP + Access DB 연동 (2024)** · 5/10챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

```asp
<%
X1 = Request.Form("txtcid")
X2 = Request.Form("txtcname")
X3 = Request.Form("txtdesc")

DSNless="DRIVER={Microsoft Access Driver (*.mdb)}; "
DSNless=DSNless & "DBQ=" & server.mappath("nwind.mdb")

Set Conn = Server.CreateObject("ADODB.Connection")
Conn.Open DSNless

SQL = "INSERT INTO tblCategories (CategoryID, CategoryName, Description) VALUES ('"&X1&"','"&X2&"','"&X3&"')"
Conn.Execute SQL

'SQL = "SELECT * FROM tblCategories"
'Set rs = Server.CreateObject("adodb.recordset")
'rs.Open SQL, Conn
'rs.AddNew
'rs("CategoryID")= X1
'rs("CategoryName") = X2
'rs("Description")= X3
'rs.Update
'rs.Close

Conn.Close
Set Conn=nothing
Response.Redirect "insert.htm"
%>
```

#### 주석 처리된 코드가 이 파일의 백미다

**같은 일을 하는 두 가지 방법이 나란히 있다:**

| | 활성 코드 (SQL 직접) | 주석 코드 (Recordset) |
|---|---|---|
| 방법 | `Conn.Execute "INSERT INTO..."` | `rs.AddNew` → `rs("컬럼")=값` → `rs.Update` |
| 성격 | **SQL을 직접 쓴다** | **객체처럼 다룬다** |
| 속도 | **빠름** | 느림 (테이블 전체를 연다) |
| 가독성 | SQL을 알아야 함 | **직관적** |
| 현대의 후예 | Raw SQL / `db.execute()` | **ORM** (mongoose, Prisma...) |

**`rs.AddNew` 방식이 사실상 ORM의 조상**이다:

```asp
rs.AddNew                        ' 새 행 만들기
rs("CategoryID") = X1            ' 필드에 값 대입
rs("CategoryName") = X2
rs.Update                        ' 저장
```

#### 왜 SQL 직접 방식을 택했나

**주석 코드는 `SELECT * FROM tblCategories`로 테이블 전체를 연다.** 행 하나 넣으려고
전부 읽는 셈이라 **비효율적**이다. 실무에서도 `Conn.Execute`가 정석이었다.

> **실습 과정에서 두 방법을 다 해 보고 하나를 남긴 흔적**이다. 지우지 않고 주석으로
> 둔 덕분에 **비교 학습 자료**가 됐다.

#### `'`가 VBScript의 주석이다

`//`도 `#`도 아닌 **작은따옴표**다. 그래서 SQL 문자열 안의 `'`와 헷갈리기 쉽다.
