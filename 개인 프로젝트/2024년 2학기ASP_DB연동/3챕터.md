# 3챕터 — ★ DSN-less 연결 — 모든 파일의 공통 4줄

> **ASP + Access DB 연동 (2024)** · 3/10챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

**거의 모든 `.asp` 파일이 이 4줄로 시작한다:**

```asp
DSNless="DRIVER={Microsoft Access Driver (*.mdb)}; "
DSNless=DSNless & "DBQ=" & server.mappath("nwind.mdb")

Set Conn = Server.CreateObject("ADODB.Connection")
Conn.Open DSNless
```

#### "DSN-less"가 무슨 뜻인가

**DSN**(Data Source Name)은 Windows **ODBC 관리자에 등록하는 DB 연결 설정**이다.

| 방식 | 방법 | 문제 |
|---|---|---|
| **DSN 사용** | 제어판 → ODBC → DSN 등록 → 코드에서 이름만 호출 | **서버마다 등록해야 한다** |
| **DSN-less** | **코드에 연결 문자열을 직접** 쓴다 | 없음 (그래서 표준이 됨) |

**DSN-less가 현대적 방식**이다. 지금의 `mongodb://127.0.0.1:27017/kababdb`나
`postgresql://user:pass@host/db` 같은 **연결 문자열(connection string)**이 정확히 이것이다.

> [Express 프로젝트 6단계](../2020_express-locallibrary-tutorial/README.md)의
> `var mongoDB = 'mongodb://127.0.0.1:27017/kababdb';` 와 **완전히 같은 개념**이다.
> 형식만 다르다.

#### 한 줄씩

| 코드 | 의미 |
|---|---|
| `DRIVER={Microsoft Access Driver (*.mdb)}` | **어떤 DB인지** — Access용 ODBC 드라이버 |
| `&` | VBScript의 **문자열 연결** (JS의 `+`) |
| **`server.mappath("nwind.mdb")`** | **가상 경로 → 실제 물리 경로** 변환 |
| `Server.CreateObject("ADODB.Connection")` | COM 객체 생성 |
| `Conn.Open DSNless` | 연결 |

#### `server.mappath()`가 핵심이다

```
server.mappath("nwind.mdb")
        ↓
"C:\inetpub\wwwroot\nwind.mdb"      ← 실제 디스크 경로
```

**Access 드라이버는 파일 경로를 요구한다.** `.mdb`는 서버가 아니라 **파일**이니까.
하지만 웹 애플리케이션은 자기가 디스크 어디에 있는지 모른다.

`server.mappath`가 **"현재 스크립트 기준 상대 경로 → 절대 경로"**를 만들어 준다.
**하드코딩하지 않아서** 서버를 옮겨도 동작한다.

> **광명테크 프로젝트의 `config_loader.py`가 하는 일과 같다:**
> ```python
> root_dir = os.path.dirname(os.path.dirname(os.path.abspath(config_path)))
> config["model"]["mediapipe"]["hand_landmarker_path"] = os.path.join(root_dir, ...)
> ```
> **"상대 경로를 절대 경로로"** — 28년 차이인데 같은 문제를 푼다.

#### 반복이 문제다

**이 4줄이 20개 파일에 그대로 복사되어 있다.**

```asp
' insert.asp
DSNless="DRIVER={Microsoft Access Driver (*.mdb)}; "
DSNless=DSNless & "DBQ=" & server.mappath("nwind.mdb")
' read.asp — 똑같음
' delete.asp — 똑같음
' ... 20번 반복
```

**DB 이름을 바꾸려면 20개 파일을 다 고쳐야 한다.**

**ASP에도 해결책이 있었다** — `#include`:

```asp
<!--#include file="db.inc"-->
```

`db.inc`에 연결 코드를 한 번만 두고 모든 파일이 포함하는 방식이다.
**실습 예제라 각 파일을 독립적으로 읽히게 하려고 일부러 반복했을 수도 있다.**

#### 리소스 해제

```asp
Conn.Close
Set Conn = nothing
```

**`Set Conn = nothing`이 중요하다.** VBScript에는 GC가 있지만 **COM 객체는 명시적으로
놓아 줘야** 확실히 해제된다. 안 하면 Access 파일이 잠긴 채 남을 수 있다.

> `Conn.Close`만 하고 `Set ... = nothing`을 빠뜨린 파일도 있다(`delete.asp`).
> 실습 코드라 큰 문제는 없지만 일관성은 없다.
