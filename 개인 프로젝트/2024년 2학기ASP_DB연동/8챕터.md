# 8챕터 — ★ `c` 시리즈 — 3초마다 새로고침하는 게시판

> **ASP + Access DB 연동 (2024)** · 8/10챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

**`c`로 시작하는 6개 파일은 완전히 별개의 프로젝트**다.

| 파일 | 역할 |
|---|---|
| `cwrite.htm` → `cwrite.asp` | 글쓰기 |
| **`cread.asp`** | **읽기 ★** |
| `cupdate2.asp` / `cupdate3.asp` | 수정 |
| `cdelete.asp` | 삭제 |
| `cframe.htm` | **프레임으로 묶기** |

**DB부터 다르다:**

```asp
DSNless=DSNless & "DBQ=" & server.mappath("onchat.mdb")    ← nwind.mdb 가 아니다!
```

#### `cwrite.asp` — 아이디어 하나만 받는다

```asp
<%
X = Request.Form("txtIdea")

DSNless="DRIVER={Microsoft Access Driver (*.mdb)}; "
DSNless=DSNless & "DBQ=" & server.mappath("onchat.mdb")

Set Conn = Server.CreateObject("ADODB.Connection")
Conn.Open DSNless

SQL = "INSERT INTO IdeaTime (Idea) VALUES ('"& X &"')"
Conn.Execute SQL

Conn.Close
Set Conn=nothing
Response.Redirect "cwrite.htm"
%>
```

**테이블 `IdeaTime`, 컬럼 `Idea` 하나뿐**이다. 제목도 작성자도 없다.
**`onchat`**이라는 DB 이름과 합치면 정체가 분명하다 — **채팅/아이디어 보드**다.

#### `cread.asp` — 이 폴더에서 가장 흥미로운 파일

```asp
<%
DSNless=DSNless & "DBQ=" & server.mappath("onchat.mdb")
Set Conn = Server.CreateObject("ADODB.Connection")
Conn.Open DSNless

Set Rs = Server.CreateObject("ADODB.Recordset")
Rs.Open "Select * From IdeaTime Order By id DESC", Conn      ← ① 최신순
%>

<html>
<head>
  <meta charset="UTF-8">
  <meta http-equiv="refresh" content="3">                     ← ② ★ 3초마다 자동 새로고침!
</head>
<body>

<table border="0" width="800">
<%
while not Rs.eof
%>
  <tr <% If Rs("id") Mod 2 = 0 Then %> bgcolor="lightblue"    ← ③ 얼룩말 무늬
      <% Else %> bgcolor="beige" <% End If %> >
  ...
```

**세 가지가 눈에 띈다.**

#### ① `Order By id DESC` — 최신 글이 위로

**채팅/게시판의 기본**이다. `id`가 auto-increment라면 큰 값이 최신이다.

#### ② `<meta http-equiv="refresh" content="3">` — 가난한 자의 실시간

**3초마다 페이지 전체를 다시 불러온다.**

```
0초: 페이지 로드 → 글 목록 표시
3초: 자동 새로고침 → 새 글 있으면 나타남
6초: 자동 새로고침
...
```

**AJAX도 WebSocket도 없던 시절의 "실시간"**이다. 그리고 **자바스크립트 한 줄 없이**
HTML 태그 하나로 구현된다.

| 시대 | 실시간 방법 |
|---|---|
| **1996~** | **`<meta refresh>`** ← 이 코드 |
| 2005~ | AJAX 폴링 (`setInterval` + `XMLHttpRequest`) |
| 2011~ | WebSocket |
| 현재 | WebSocket / SSE |

**단점이 명확하다**: 페이지 전체(HTML+CSS+이미지)를 3초마다 다시 받는다. 스크롤 위치도
초기화되고, 글을 쓰는 도중에 새로고침되면 **입력하던 게 날아간다.**
→ 그래서 **`cframe.htm`이 필요하다.**

#### `cframe.htm` — 프레임으로 문제를 우회한다

```
┌─────────────────────────────┐
│  cread.asp                  │  ← 3초마다 새로고침 (이 프레임만)
│  (글 목록)                   │
│                             │
├─────────────────────────────┤
│  cwrite.htm                 │  ← 새로고침 안 됨 (입력 유지!)
│  (입력 폼)                   │
└─────────────────────────────┘
```

**`<frameset>`으로 화면을 둘로 나누면 위쪽만 새로고침된다.**
아래 입력 폼은 그대로 남아서 타이핑하던 게 안 날아간다.

> **이게 프레임의 원래 존재 이유**다. "페이지 일부만 갱신"을 하려면 프레임밖에 없었다.
> **AJAX가 등장하면서 프레임은 사라졌다** — AJAX가 정확히 같은 일을 더 잘하니까.
>
> `<frameset>`은 HTML5에서 **폐기(deprecated)**됐다. 이 코드는 그 시절의 화석이다.

#### ③ `Rs("id") Mod 2 = 0` — 얼룩말 무늬

```asp
<tr <% If Rs("id") Mod 2 = 0 Then %> bgcolor="lightblue"
    <% Else %> bgcolor="beige" <% End If %> >
```

**짝수 id는 하늘색, 홀수는 베이지.** 행을 번갈아 칠해 **읽기 쉽게** 하는 기법이다.

**지금은 CSS 한 줄이면 된다:**

```css
tr:nth-child(even) { background: lightblue; }
```

**`:nth-child`가 2011년에야 널리 지원**되기 시작했다. 그전엔 **서버가 행 번호를 세어서
색을 직접 넣는** 수밖에 없었다.

> **`id Mod 2`는 `nth-child(even)`과 미묘하게 다르다.** 중간 글이 삭제되면 id에 구멍이
> 나서 **같은 색이 연속으로** 나올 수 있다. 화면상 행 번호가 아니라 **DB의 id**를 쓰기 때문이다.
