#### 모든 파일이 SQL 인젝션에 취약하다

**예외 없이 전부 문자열 연결로 SQL을 만든다:**

```asp
' insert.asp
SQL = "INSERT INTO tblCategories (...) VALUES ('"&X1&"','"&X2&"','"&X3&"')"

' delete.asp
sql = "DELETE * FROM tblCategories WHERE CategoryID = '"& X1 &"';"

' cwrite.asp
SQL = "INSERT INTO IdeaTime (Idea) VALUES ('"& X &"')"

' GetEmpInfo.asp
sql3 = Request.Form("empLastName") & "'"
sql = sql1 & sql2 & sql3
```

#### 어떻게 뚫리나

`delete.asp`의 `txtcid`에 이걸 넣으면:

```
' OR '1'='1
```

**만들어지는 SQL:**

```sql
DELETE * FROM tblCategories WHERE CategoryID = '' OR '1'='1';
                                                ─────────────
                                                항상 참!
```

**`WHERE`가 무력화되어 테이블 전체가 지워진다.** 7단계에서 "WHERE가 있어 다행"이라 했지만
**인젝션으로 우회된다.**

`GetEmpInfo.asp`도 마찬가지다:

```
' OR '1'='1
→  select ... from tblEmployees where lastname='' OR '1'='1'
→  전 직원 정보가 나온다
```

#### 올바른 방법 — 파라미터 바인딩

**ASP에도 해결책이 있었다** (`ADODB.Command`):

```asp
Set Cmd = Server.CreateObject("ADODB.Command")
Cmd.ActiveConnection = Conn
Cmd.CommandText = "DELETE FROM tblCategories WHERE CategoryID = ?"    ← 값을 ? 로
Cmd.Parameters.Append Cmd.CreateParameter("cid", 200, 1, 50, X1)      ← 따로 전달
Cmd.Execute
```

**`?` 자리에 값을 "데이터로" 넣으면** DB가 그걸 **절대 SQL로 해석하지 않는다.**
`' OR '1'='1`을 넣어도 그냥 **그런 이름의 CategoryID를 찾을 뿐**이다.

**현대 언어도 전부 같은 원리다:**

```python
cursor.execute("DELETE FROM t WHERE id = %s", (user_input,))   # Python
```
```javascript
await Board.find({ title: userInput });    // mongoose — 자동으로 안전
```

> **이 코드를 비난할 일은 아니다.** 2024년 학교 실습이고, **로컬 IIS에서 혼자 돌리는
> 예제**다. 인터넷에 노출되지 않는다. 당시 교재가 그렇게 가르쳤을 가능성도 높다.
>
> **다만 이 폴더를 읽는 사람은 반드시 알아야 한다**: 이 패턴을 실제 서비스에
> 그대로 옮기면 **즉시 털린다.** SQL 인젝션은 OWASP Top 10에 20년 넘게 올라 있는
> 1급 취약점이다.
>
> **그리고 이게 ORM이 존재하는 이유**다. [Express 프로젝트](../2020_express-locallibrary-tutorial/README.md)의
> mongoose는 이런 걸 신경 쓸 필요가 없다 — 기본적으로 안전하다.

#### 실행법

**❌ 가장 어려운 프로젝트다.** 최신 Windows에서 클래식 ASP + Access를 살리려면:

**1. IIS 설치 + ASP 활성화**

```
제어판 → 프로그램 → Windows 기능 켜기/끄기
  → 인터넷 정보 서비스
    → World Wide Web 서비스
      → 응용 프로그램 개발 기능
        → ☑ ASP          ← 기본 꺼져 있다!
        → ☑ ISAPI 확장
```

**2. 파일 배치**

```
root/ 의 내용을 C:\inetpub\wwwroot\ 에 복사
또는 IIS 관리자에서 가상 디렉터리로 등록
```

**3. Access 드라이버**

```
Microsoft Access Database Engine 2010 Redistributable 설치
(또는 Office가 설치되어 있으면 이미 있을 수 있음)
```

**4. ⚠ 32비트 문제**

```
IIS 관리자 → 응용 프로그램 풀 → DefaultAppPool → 고급 설정
  → 32비트 응용 프로그램 사용: True     ← Access 드라이버가 32비트면 필수
```

**5. ⚠ 쓰기 권한**

```
.mdb 파일에 IIS_IUSRS 쓰기 권한 부여
(INSERT/DELETE 하려면 필요 — Access는 파일에 직접 쓴다)
```

**6. 접속**

```
http://localhost/insert.htm
http://localhost/cframe.htm
http://localhost/GetEmpInfo.asp
```

> **코드를 읽는 게 목적이라면 IIS를 설치하지 말고 그냥 텍스트로 열어 보길 권한다.**
> `.asp`는 그냥 텍스트 파일이다. 이 문서가 주요 파일을 다 인용해 뒀다.
