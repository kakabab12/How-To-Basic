# 4챕터 — `.htm` + `.asp` 쌍 — 폼과 처리의 분리

> **ASP + Access DB 연동 (2024)** · 4/10챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

#### `insert.htm` — 순수 HTML 폼

```html
<html>
<head>
  <meta charset="UTF-8">
</head>
<body>

<form method="post" action="insert.asp">
 <table border="2">
   <tr>
     <td> Category ID </td>
     <td> <input type="text" name="txtcid" size="20"> </td>
   </tr>
   <tr>
     <td> Category Name </td>
     <td> <input type="text" name="txtcname" size="20"> </td>
   </tr>
   <tr>
     <td> Description </td>
     <td> <input type="text" name="txtdesc" size="20"> </td>
   </tr>
 </table>
 ...
</form>
</body>
</html>
```

**`.asp`가 아니라 `.htm`이다** — 서버 코드가 한 줄도 없다. 순수 정적 파일.

#### 왜 나누나

| 파일 | 확장자 | 서버가 하는 일 |
|---|---|---|
| `insert.htm` | `.htm` | **그냥 파일을 보낸다** (빠름) |
| `insert.asp` | `.asp` | **VBScript를 실행**한다 (느림) |

**IIS는 확장자로 처리 방식을 정한다.** 폼은 동적일 게 없으니 `.htm`으로 두면
서버가 스크립트 엔진을 안 거치고 바로 보낸다.

#### 폼 → 처리 흐름

```
사용자가 insert.htm 을 연다
    ↓  입력하고 [전송]
POST /insert.asp
  txtcid=1&txtcname=음료&txtdesc=...
    ↓
insert.asp 실행
  X1 = Request.Form("txtcid")     ← name 속성과 정확히 일치해야 한다
  ...INSERT...
    ↓
Response.Redirect "insert.htm"    ← 폼으로 되돌린다
```

**`name="txtcid"` ↔ `Request.Form("txtcid")`가 계약**이다. 하나만 오타 나도 조용히
빈 값이 들어간다.

#### `Response.Redirect`로 폼에 되돌린다

```asp
Conn.Close
Set Conn=nothing
Response.Redirect "insert.htm"
```

**결과 페이지를 안 보여주고 폼으로 돌아간다.** 연속 입력에 편하다.

> **이건 PRG 패턴(Post-Redirect-Get)의 초기 형태**다. POST 후 리다이렉트하지 않으면,
> 사용자가 F5를 눌렀을 때 **"양식을 다시 제출하시겠습니까?"**가 뜨고 **중복 INSERT**가 된다.
> 리다이렉트하면 F5는 그냥 GET이라 안전하다.
>
> 의도한 것인지 우연인지는 알 수 없지만 **결과적으로 옳다.**

**단점**: 성공/실패를 알려주지 않는다. INSERT가 실패해도 그냥 폼으로 돌아간다.
