# 2챕터 — 파일 이름에서 구조 읽기

> **ASP + Access DB 연동 (2024)** · 2/10챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

**코드를 열기 전에 파일명만으로 구조가 보인다.**

```
root/
├─ [기본 CRUD — nwind.mdb 의 tblCategories]
│   insert.htm  → insert.asp        삽입
│   insert2.htm → insert2.asp       삽입 (개선판)
│   read.asp / read2.asp / read3.asp   조회 (3단계 발전)
│   delete.htm  → delete.asp → delete2.asp   삭제
│   form.htm / frame.htm / test1.htm / test2.htm / test2.asp / reg.htm
│
├─ ['c' 시리즈 — onchat.mdb 의 IdeaTime]  ← 8단계
│   cwrite.htm → cwrite.asp         글쓰기
│   cread.asp                       읽기 (자동 새로고침!)
│   cupdate2.asp / cupdate3.asp     수정
│   cdelete.asp                     삭제
│   cframe.htm                      프레임으로 묶기
│
├─ [독립 예제]
│   GetEmpInfo.asp                  직원 검색 (9단계)
│   random.asp                      난수
│
└─ [DB]
    nwind.mdb / Northwind.mdb       Microsoft 예제 DB
    onchat.mdb                      직접 만든 DB
```

#### 규칙 3가지

**① 숫자 접미사 = 개선 버전**

```
read.asp → read2.asp → read3.asp
insert.asp → insert2.asp
cupdate2.asp → cupdate3.asp
delete.asp → delete2.asp
```

**같은 기능을 단계적으로 발전**시키는 실습 방식이다.
`read.asp`로 목록을 띄우고, `read2.asp`에서 수정 폼을 붙이고... 하는 식.

> 이 습관은 **6년 뒤에도 그대로다.** 메디컬 프로젝트의 `server_old2~5.py`,
> `index_old~old10.html`, 젯슨의 `jetson_main1(2)~(9).py`.
> **git 브랜치 대신 파일 복사**로 버전을 관리하는 방식이 이때부터 있었다.

**② `c` 접두사 = 별개의 세트**

`c`로 시작하는 6개는 **다른 DB**(`onchat.mdb`)를 쓰는 **독립된 게시판**이다(8단계).
`c`는 `chat`의 c로 보인다.

**③ `.htm` + `.asp` 쌍 = 폼과 처리**

```
insert.htm   ← 사용자가 보는 입력 폼 (정적)
insert.asp   ← 폼을 받아 DB에 넣는 로직 (동적)
```

4단계에서 자세히 본다.

#### Northwind가 뭔가

**`nwind.mdb` / `Northwind.mdb`는 Microsoft의 표준 예제 DB**다.
가상의 식품 무역회사 데이터로, 1990년대부터 Access/SQL Server 교육에 쓰였다.

| 테이블 | 내용 |
|---|---|
| `tblCategories` | 상품 분류 (음료, 유제품...) ← **이 과제가 주로 쓰는 것** |
| `tblEmployees` | 직원 (Davolio, Fuller, Buchanan...) ← 9단계 |
| `tblProducts` | 상품 |
