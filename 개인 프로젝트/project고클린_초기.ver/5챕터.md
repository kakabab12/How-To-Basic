# 5챕터 — 기능 ① 업데이트 — `UsoClient`와 하드코딩된 NVIDIA

> **고클린 초기.ver** · 5/8챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

```batch
:UPDATE
echo [1/6] 프로그램 업데이트 (Winget)
winget upgrade --all --include-unknown --accept-package-agreements --accept-source-agreements

echo [2/6] 파이썬 PIP 업데이트
python -m pip install --upgrade pip

echo [3/6] 리눅스 서브시스템(WSL) 업데이트
wsl --update

echo [4/6] 윈도우 + 드라이버 검색
UsoClient StartScan
UsoClient StartDownload
UsoClient StartInstall

echo [5/6] MS 스토어 업데이트
start ms-windows-store://downloadsandupdates

echo [6/6] 그래픽카드 드라이버 (GeForce)
winget upgrade Nvidia.GeForceExperience
```

#### `[1/6]` 진행 표시

**모든 기능이 `[n/총]` 형식으로 진행을 알린다.** 배치에는 진행 막대가 없으니
**숫자로 대신**한다. alpha.ver의 `progressbar`(8단계)가 이걸 대체한다.

#### `winget` 플래그가 실전적이다

| 플래그 | 왜 |
|---|---|
| `--all` | 전부 업그레이드 |
| `--include-unknown` | **버전을 모르는 패키지도 포함** — 없으면 상당수가 건너뛰어진다 |
| `--accept-package-agreements` | **약관 동의 프롬프트 자동 수락** |
| `--accept-source-agreements` | 소스 약관 자동 수락 |

**뒤의 두 개가 없으면 스크립트가 프롬프트에서 멈춘다.** 자동화의 필수 조건이다.

#### `UsoClient` — 문서화되지 않은 명령

본인 설명:

> 사용자에게 윈도우 설정 창을 띄우지 않고 **백그라운드에서 조용히** 윈도우 업데이트 스캔과
> 설치를 강제하는 강력한 명령어입니다.

**Microsoft가 공식 문서화하지 않은 내부 명령**이다(Update Session Orchestrator).

```
UsoClient StartScan      → 업데이트 검색
UsoClient StartDownload  → 다운로드
UsoClient StartInstall   → 설치
```

**찾아서 쓴 것 자체가 조사의 결과다.** 보통은 `start ms-settings:windowsupdate`로
설정 창을 띄우고 사용자가 누르게 하는데, 그걸 넘어섰다.

> **⚠ `UsoClient`는 최신 Windows 11에서 동작하지 않을 수 있다.** MS가 조용히 막았다.
> 대안은 PowerShell의 `PSWindowsUpdate` 모듈이지만 별도 설치가 필요하다.

#### `[6/6]`이 이 스크립트의 가장 큰 한계다

```batch
echo [6/6] 그래픽카드 드라이버 (GeForce)
winget upgrade Nvidia.GeForceExperience
```

**NVIDIA가 하드코딩되어 있다.** AMD나 Intel GPU를 쓰는 사람에게는 **아무 의미가 없다**
(그냥 "패키지를 찾을 수 없음" 에러가 뜬다).

**alpha.ver은 정확히 이 문제를 고쳤다:**

```python
# alpha.ver — wmic 으로 GPU 제조사를 감지해서 맞는 페이지를 연다
"NVIDIA" in name  →  NVIDIA 드라이버 페이지
"AMD"    in name  →  AMD 페이지
"Intel"  in name  →  Intel 페이지
```

> **버전 간 개선을 가장 뚜렷하게 볼 수 있는 지점**이다. 배치로는 GPU 감지 + 분기 +
> 브라우저 열기를 짜기가 번거롭지만, Python에서는 `wmic` 출력을 파싱해 `if`로 나누면 끝이다.
> **언어를 바꾼 이유 중 하나**가 여기 있다.

> **⚠ `python -m pip`도 문제다.** 이 PC에서 `python`은 Microsoft Store 스텁이라
> 아무것도 하지 않는다. `py -m pip`가 맞다.
