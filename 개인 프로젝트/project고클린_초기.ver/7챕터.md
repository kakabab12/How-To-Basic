# 7챕터 — 기능 ③ 게임 모드 — ★ 안전장치

> **고클린 초기.ver** · 7/8챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

```batch
:KILLPROCESS
echo    [주의] 현재 작업 중인 인터넷 창, 카톡, 게임, 문서가 강제로 종료됩니다.
echo    정말 진행하시겠습니까? (메모리 확보 및 렉 제거)
echo.
set /p confirm="                             진행하려면 'Y', 취소하려면 'N' 입력: "

if /i "%confirm%" neq "Y" goto MENU        ← ★ Y가 아니면 무조건 메뉴로
```

#### 이 스크립트에서 가장 중요한 3줄

본인 설명:

> 작업 중인 데이터가 날아갈 위험이 있는 프로세스 강제 종료(게임 모드) 기능에서는
> `set /p confirm=`을 사용해 사전에 진행 여부(Y/N)를 묻는 **안전장치**를 두었습니다.

| 조각 | 의미 |
|---|---|
| `/i` | **대소문자 무시** — `y`도 `Y`도 통과 |
| `neq "Y"` | **`Y`가 아니면** (not equal) |
| `goto MENU` | 취소 |

**"Y가 아니면 취소"가 올바른 방향**이다. `if "%confirm%"=="N" goto MENU`였다면
사용자가 엔터만 쳐도(빈 값) **그대로 진행**된다. **위험한 쪽을 기본값으로 두지 않았다.**

**경고 문구도 구체적이다** — "인터넷 창, 카톡, 게임, 문서"라고 **실제로 뭐가 죽는지**
말한다. "정말 진행하시겠습니까?"만 있는 것보다 훨씬 낫다.

> **⚠ alpha.ver에는 이 확인이 없다.** GUI 버튼을 누르면 **바로 죽인다.**
> **초기 버전이 더 안전했던 드문 사례**다.

#### 종료 목록이 분류되어 있다

```batch
:: 브라우저
taskkill /F /IM chrome.exe /T >nul 2>&1
taskkill /F /IM msedge.exe /T >nul 2>&1
taskkill /F /IM whale.exe /T >nul 2>&1       ← 네이버 웨일
taskkill /F /IM firefox.exe /T >nul 2>&1

:: 메신저
taskkill /F /IM KakaoTalk.exe /T >nul 2>&1
taskkill /F /IM Discord.exe /T >nul 2>&1
taskkill /F /IM Skype.exe /T >nul 2>&1
taskkill /F /IM Teams.exe /T >nul 2>&1
taskkill /F /IM Zoom.exe /T >nul 2>&1

:: 게임 런처
taskkill /F /IM steam.exe /T >nul 2>&1
taskkill /F /IM EpicGamesLauncher.exe /T >nul 2>&1
taskkill /F /IM Battle.net.exe /T >nul 2>&1
taskkill /F /IM RiotClientServices.exe /T >nul 2>&1
taskkill /F /IM nxsteam.exe /T >nul 2>&1      ← 넥슨

:: 기타 앱
taskkill /F /IM notepad++.exe /T >nul 2>&1
taskkill /F /IM Hwp.exe /T >nul 2>&1          ← 한글
taskkill /F /IM EXCEL.EXE /T >nul 2>&1
taskkill /F /IM WINWORD.EXE /T >nul 2>&1
taskkill /F /IM POWERPNT.EXE /T >nul 2>&1
```

**주석으로 4개 그룹**(브라우저/메신저/게임 런처/기타 앱)으로 나눴다.
반복되는 명령이지만 **읽기 쉽고, 추가할 때 어디에 넣을지 명확**하다.

#### 플래그 세 개가 각각 일한다

| 플래그 | 역할 |
|---|---|
| `/F` | **강제** 종료 (정상 종료 요청이 아님) |
| `/IM` | **이미지 이름**(프로세스명)으로 지정 |
| **`/T`** | **자식 프로세스까지** 함께 종료 ← 크롬처럼 프로세스를 수십 개 띄우는 앱에 필수 |
| `>nul 2>&1` | **없는 프로그램에 대한 에러 숨김** — 스팀 안 깔린 PC에서도 조용히 넘어간다 |

**`/T`가 없으면 크롬 탭 프로세스들이 그대로 살아남는다.** 크롬은 탭마다 프로세스를
따로 띄우므로 부모만 죽여선 소용이 없다.

#### 목록이 "한국인 PC"에 맞춰져 있다

`whale.exe`(네이버 웨일), `KakaoTalk.exe`, `Hwp.exe`(한글), `nxsteam.exe`(넥슨) —
**해외 도구 목록에는 절대 없을 항목들**이다. 실제 자기 PC를 보고 만든 목록이라는 증거다.

#### 탐색기 재시작

```batch
echo [2/3] 윈도우 탐색기(Explorer) 재시작
taskkill /f /im explorer.exe >nul 2>&1
start explorer.exe
```

**탐색기는 썸네일·아이콘 캐시로 메모리를 수백 MB 쥐고 안 놓는다.**
재시작하면 작업표시줄이 잠깐 사라졌다 돌아오면서 **메모리가 반환**된다.

`:CLEAN`의 `[8/8]`과 같은 처리인데, 거기서는 `&`로 한 줄에 붙였다:

```batch
taskkill /f /im explorer.exe & start explorer.exe
```

**`&`는 "성공/실패 무관하게 다음 실행"**이다. 탐색기를 죽인 뒤 반드시 다시 켜야 하므로
(`&&`가 아니라) `&`가 맞다 — 죽이기가 실패해도 켜는 건 시도해야 한다.
