# 6챕터 — 기능 ② 청소 — 8단계

> **고클린 초기.ver** · 6/8챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

```batch
:CLEAN
echo [1/8] DNS 및 ARP 캐시 초기화
ipconfig /flushdns
arp -d *

echo [2/8] 임시 파일 및 시스템 찌꺼기 제거
del /s /q /f "%TEMP%\*.*" > nul 2>&1
del /s /q /f "C:\Windows\Temp\*.*" > nul 2>&1

echo [3/8] 윈도우 업데이트 파일 정리
dism /online /Cleanup-Image /StartComponentCleanup

echo [4/8] 윈도우 이미지 원본 복구
dism /online /Cleanup-Image /RestoreHealth

echo [5/8] 시스템 파일 무결성 검사
sfc /scannow

echo [6/8] 네트워크 소켓 초기화
netsh winsock reset

echo [7/8] 모든 디스크 최적화
defrag /C /O

echo [8/8] 탐색기 리프레시
taskkill /f /im explorer.exe & start explorer.exe
```

#### 순서에 논리가 있다

```
1. 캐시 (가벼움, 즉시)
2. 파일 삭제 (가벼움)
3~4. 시스템 이미지 (무거움, 수 분)
5. 무결성 검사 (매우 무거움, 10~30분)
6. 네트워크 (즉시, 재부팅 필요)
7. 디스크 (매우 무거움)
8. 탐색기 (즉시, 메모리 반환)
```

**가벼운 것부터 무거운 것 순**이다. 사용자가 중간에 Ctrl+C로 끊어도 **빠른 효과는
이미 봤다.**

#### 각 명령의 플래그

| 명령 | 플래그 | 의미 |
|---|---|---|
| `del` | `/s /q /f` | 하위폴더 포함(`/s`) + 확인 없이(`/q`) + 읽기전용도 강제(`/f`) |
| `del` | `> nul 2>&1` | **사용 중인 파일은 삭제 실패한다** — 그 에러를 전부 숨긴다 |
| `dism` | `/StartComponentCleanup` | 오래된 업데이트 백업 제거 |
| `dism` | `/RestoreHealth` | 손상된 시스템 이미지 복구 |
| `defrag` | `/C /O` | 모든 드라이브(`/C`) + 미디어 타입별 최적화(`/O`) |
| `taskkill` | `/f /im` | 강제(`/f`) + 이미지 이름으로(`/im`) |

**`defrag /O`가 중요하다** — SSD에는 조각 모음이 아니라 **TRIM**을 실행한다.
`/O`(optimize)는 드라이브 종류를 알아서 판단한다. **SSD에 조각 모음을 돌리면 수명만
깎이는데** 그걸 피한 것이다.

**`arp -d *`**는 alpha.ver에 없는 것이다 — ARP 캐시(IP↔MAC 매핑) 삭제로,
로컬 네트워크 이상 시 도움이 된다.

#### 8단계 → 5단계로 줄어든 이유

| | 초기.ver | alpha.ver |
|---|---|---|
| 단계 | **8** | **5** |
| 빠진 것 | — | `arp -d`, `dism /RestoreHealth`, **`sfc /scannow`** |

**`sfc /scannow`가 빠진 게 결정적**이다. 10~30분이 걸린다.

```
배치: 콘솔에 진행률이 실시간으로 찍힌다 → 30분도 참을 만하다
GUI:  진행률을 못 가져온다(subprocess는 끝나야 출력을 준다)
      → 30분 동안 indeterminate 막대만 돈다 → 사용자가 죽은 줄 안다
```

> **GUI로 옮기면서 기능을 뺐다.** 도구를 바꾸면 얻는 것도 있고 잃는 것도 있다는
> 좋은 사례다. **`sfc`는 배치 버전이 더 낫다.**

> **⚠ `netsh winsock reset`은 재부팅이 필요하다.** 두 버전 모두 그 안내가 없다.
