# 4챕터 — 메뉴 루프 — `goto`로 만드는 상태 기계

> **고클린 초기.ver** · 4/8챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

```batch
:MENU
cls
echo.
:: --- 메인 아스키 아트 ---
echo                  -.            -.           .,.            ,,     ...
   ... (15줄) ...
echo                            =========================================
echo                                   [ SYSTEM MASTER CONTROL ]
echo                            =========================================
echo.
echo                             1. 시스템 업데이트 (Update All)
echo                             2. 시스템 청소 (Clean + Boost)
echo                             3. 프로세스 초기화 (Game Mode)
echo                             4. 인터넷 핑 테스트 (Ping Check)
echo                             5. 내 IP 및 PC 정보 확인 (My Info)
echo                             6. 종료 (Exit)
echo.
set /p choice="                             원하는 기능의 번호 입력 (1~6): "

if "%choice%"=="1" goto UPDATE
if "%choice%"=="2" goto CLEAN
if "%choice%"=="3" goto KILLPROCESS
if "%choice%"=="4" goto PINGTEST
if "%choice%"=="5" goto MYINFO
if "%choice%"=="6" goto EXIT
goto MENU
```

#### `goto MENU`가 마지막 줄인 이유

**잘못된 입력에 대한 처리**다.

```
사용자가 "9" 입력
    ↓
if 문 6개 전부 불일치
    ↓
goto MENU      ← 메뉴를 다시 그린다 (무한 루프)
```

**에러 메시지도 없고 검증도 없다.** 그냥 **메뉴가 다시 뜬다.**
투박하지만 배치에서는 이게 가장 단순하고 확실한 방법이다.

#### 각 기능의 끝도 `goto MENU`다

```batch
:UPDATE
...작업...
echo 아무 키나 누르면 메뉴로 돌아갑니다.
pause > nul
goto MENU        ← 메뉴로 복귀
```

**`pause > nul`**이 중요하다. 없으면 작업이 끝나자마자 `cls`로 화면이 지워져서
**결과를 볼 시간이 없다.** `> nul`은 "계속하려면 아무 키나 누르십시오..." 기본 메시지를
숨기고, 대신 한글 안내를 직접 출력한다.

#### 이게 곧 상태 기계다

```
        ┌──────────────────────────────┐
        ▼                              │
    ┌───────┐  1  ┌────────┐  pause    │
    │ :MENU │────▶│ :UPDATE│───────────┤
    │       │  2  ├────────┤           │
    │       │────▶│ :CLEAN │───────────┤
    │       │  3  ├────────┤           │
    │       │────▶│:KILLPRO│───────────┤
    │       │ ... └────────┘           │
    │       │  6  ┌────────┐           │
    │       │────▶│ :EXIT  │           │
    └───────┘     └────────┘           │
        ▲   기타 입력                   │
        └──────────────────────────────┘
```

**라벨(`:MENU`)이 상태, `goto`가 전이**다. alpha.ver의 GUI 버튼과 이벤트 핸들러가
정확히 이 구조를 대체한다 — 다만 GUI는 **메뉴로 "돌아갈" 필요가 없다.**
창이 항상 거기 있으니까.

> **배치와 GUI의 근본적 차이**: 배치는 **한 번에 하나**만 할 수 있어서 메뉴 ↔ 작업을
> 오간다. GUI는 **항상 모든 버튼이 보이고**, 그래서 7단계(alpha.ver)의
> "작업 중 버튼 잠금"이 필요해진다.
