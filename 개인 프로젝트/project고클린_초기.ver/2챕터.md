# 2챕터 — ★ `fltmc` 관리자 권한 트릭

> **고클린 초기.ver** · 2/8챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

**이 파일에서 가장 영리한 5줄**이다.

```batch
:: ============================================================
:: [자동 관리자 권한 획득]
:: ============================================================
fltmc >nul 2>&1 || (
    echo [!] 관리자 권한을 요청합니다...
    powershell Start-Process -FilePath '%0' -Verb RunAs
    exit /b
)
```

#### 문제

**배치 스크립트에는 `is_admin()` 같은 함수가 없다.** 관리자인지 물어볼 API가 없다.
그런데 이 스크립트가 하는 일(`del %TEMP%`, `netsh winsock reset`, `sfc`, `defrag`)은
**전부 관리자 전용**이다.

#### 답 — 관리자만 되는 명령을 일부러 실행해 본다

**`fltmc`**(파일시스템 필터 드라이버 관리자)는 **관리자가 아니면 에러를 낸다.**
그 성질을 이용한다:

```
fltmc >nul 2>&1  ||  ( ... )
  │       │  │      └── 앞이 실패하면(||) 이걸 실행
  │       │  └── 표준에러도 버림 (2>&1)
  │       └── 표준출력 버림 (>nul)  — 화면에 아무것도 안 보이게
  └── 관리자면 성공(exit 0), 아니면 실패(exit != 0)
```

| 조각 | 역할 |
|---|---|
| `>nul 2>&1` | **출력과 에러를 모두 버린다** — 우리는 성공/실패만 알면 된다 |
| `\|\|` | **앞 명령이 실패했을 때만** 뒤를 실행 (배치의 OR 연산자) |
| `%0` | **자기 자신의 경로** |
| `-Verb RunAs` | **UAC 창을 띄운다** |
| `exit /b` | 현재(권한 없는) 인스턴스 종료 |

#### 흐름

```
실행
 │
 ▼
fltmc 성공? ── 예 ──▶ 관리자다. 계속 진행
 │
 아니오
 │
 ▼
powershell Start-Process '%0' -Verb RunAs   ← UAC 창 + 새 인스턴스
 │
 ▼
exit /b     ← 현재 인스턴스는 죽는다 (창 두 개 방지)
```

**`exit /b`가 없으면 권한 없는 창과 있는 창이 둘 다 뜬다** — alpha.ver의 `sys.exit()`와
정확히 같은 이유다.

#### 왜 `powershell`을 부르나

**배치에는 UAC를 띄우는 방법이 없다.** `-Verb RunAs`는 PowerShell(정확히는 .NET의
`Process.Start`)의 기능이다. 그래서 **배치가 PowerShell을 불러서** 자신을 다시 띄운다.

> **간접 증거로 상태를 알아내는 발상** — 배치 스크립트의 대표적 관용구다.
> `fltmc` 말고 `net session`, `openfiles`도 같은 목적으로 쓰인다.
>
> **alpha.ver은 이 트릭이 필요 없다.** Python은 `ctypes.windll.shell32.IsUserAnAdmin()`으로
> **정식 API를 직접 물어본다**. → [alpha.ver 4단계](../project고클린_alpha.ver/4챕터.md)
