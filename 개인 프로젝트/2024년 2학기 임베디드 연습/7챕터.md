# 7챕터 — ★ 주야간 검증 — 영상 3개의 의미

> **임베디드 리눅스 프로젝트 (2024)** · 7/8챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

```
바디캠.mp4                        14.8 MB
배달감지 카메라-평상시.mp4          13.2 MB
배달 감지 카메라- 어두울때.mp4       6.1 MB    ← ★
```

**파일명이 이 프로젝트에서 가장 중요한 정보**다.

#### 왜 조건을 나눠 찍었나

**보고서를 다시 보면:**

> LED는 작동 상태를 표시하는 신호 역할을 하며, **어두운 곳에서는 조명 역할**도 합니다.

**"어두운 곳에서는 조명 역할"을 주장했으면 증명해야 한다.**

```
배달감지 카메라-평상시.mp4      →  "정상 조건에서 동작한다"
배달 감지 카메라- 어두울때.mp4   →  "★ LED 조명으로 어두워도 보인다"
```

**시연 영상 2개가 곧 두 개의 테스트 케이스**다.

#### 이게 왜 중요한가

**비전 시스템에서 조명은 전부다.**

| 조건 | 카메라가 보는 것 |
|---|---|
| 밝음 | 물체가 선명 |
| 어두움 | **노이즈 덩어리** — 아무것도 안 보인다 |

**"낮에는 잘 되는데 밤에는 안 돼요"**는 비전 프로젝트의 가장 흔한 실패다.
**그걸 알고 조건을 나눠 검증**했다.

> **보고서에도 그렇게 적혀 있다:**
> ```
> 3. 시연 영상
>    평상시 동영상입니다.
>    어두워서 안보일 때 동영상입니다.
> ```
> **"어두워서 안보일 때"** — 문제를 명시하고 해결(LED)을 보여 준다.

#### 파일 크기가 말해 주는 것

```
평상시:    13.2 MB
어두울때:   6.1 MB    ← 절반 이하!
```

**어두운 영상이 훨씬 작다.** 비디오 압축은 **변화가 적을수록 작아진다** —
어두운 화면은 대부분 검정이라 압축이 잘 된다.

**즉 파일 크기 자체가 "정말 어두운 데서 찍었다"는 증거**다.

#### 이 습관이 이후에 이어진다

| 프로젝트 | 검증 방식 |
|---|---|
| **2024 임베디드** | **평상시 / 어두울 때** 영상 분리 |
| 2025 라즈베리파이 | `sub/`에서 **가짜 데이터로 UI 먼저** 검증 |
| 2026 젯슨 | `cam_check.py` — **메인 서버 켜기 전 듀얼 카메라 점검** |
| 2026 광명테크 | **단위 테스트 36건** (카메라·모델 없이) |

**"주장했으면 증명한다"가 일관된다.** 방법만 점점 정교해진다 —
영상 → 가짜 데이터 → 진단 스크립트 → 자동화된 단위 테스트.
