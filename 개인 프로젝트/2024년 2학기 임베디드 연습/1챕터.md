```
2024년 2학기 임베디드 연습/
├── 20119160 이지용 - 임베디드리눅스 프로젝트.hwpx     5.5 MB   ← ★ 여기 코드가 있다
├── 20119160 이지용.pptx                              58 KB
├── 바디캠.mp4                                       14.8 MB
├── 배달감지 카메라-평상시.mp4                        13.2 MB
└── 배달 감지 카메라- 어두울때.mp4                     6.1 MB
```

**`.py` 파일이 하나도 없다.** 처음 보면 "산출물만 있고 코드는 없는 폴더"로 보인다.

#### 그런데 `.hwpx`를 열면 코드가 통째로 들어 있다

**`.hwpx`는 사실 ZIP 파일**이다(한글 2014+ 의 XML 기반 포맷, ODF 계열).

```
20119160 이지용 - 임베디드리눅스 프로젝트.hwpx  (= ZIP)
├── mimetype
├── version.xml
├── Contents/header.xml
├── Contents/section0.xml        ← ★ 본문 텍스트가 여기 (코드 포함!)
├── Contents/content.hpf
├── BinData/image2.bmp           ← 삽입된 사진들
├── BinData/image4.bmp
├── BinData/image6.bmp
├── Preview/PrvText.txt          ← 미리보기 텍스트
├── Preview/PrvImage.png
├── settings.xml
└── META-INF/container.rdf
```

**보고서 본문에 "2. 작성한 코드" 섹션이 있고, 거기 전체 소스가 붙여넣어져 있다.**

#### 꺼내는 법

```python
import zipfile, re

z = zipfile.ZipFile('20119160 이지용 - 임베디드리눅스 프로젝트.hwpx')
raw = z.read('Contents/section0.xml').decode('utf-8')
txt = re.sub(r'<[^>]+>', ' ', raw)      # XML 태그 제거
txt = re.sub(r'\s+', ' ', txt).strip()
print(txt)
```

**또는 그냥 한글(HWP)로 열면 된다.** 이 문서는 **코드를 읽으려고** 파이썬으로 뽑았다.

> **`.pptx`도 ZIP이다**(OOXML). `.docx`, `.xlsx`도 마찬가지.
> **"문서 파일이 사실은 압축 파일"**이라는 건 알아 두면 두고두고 쓸모 있다 —
> 파일이 깨졌을 때 안의 이미지만 건져 내거나, 이렇게 텍스트를 추출할 수 있다.

#### 보고서 첫 줄

```
임베디드 리눅스 스마트 홈 프로젝트
작성자 20119160 소프트웨어융합학과 이지용
```

**"스마트 홈"이 주제**다. 그 아래 프로젝트 두 개가 들어 있다.

> **왜 `.py`를 따로 커밋 안 했나**: 학교 제출은 **보고서 하나로 끝**이었을 것이다.
> 라즈베리파이 안에 있던 `.py`는 그대로 두고 보고서에 복사해 넣은 뒤,
> 나중에 폴더를 정리할 때 **제출물만 남긴** 것으로 보인다.
>
> **[Express 프로젝트](../2020_express-locallibrary-tutorial/README.md)의 "복구" 사건과 비슷하지만
> 결과는 정반대다** — 거기선 코드가 사라졌는데, 여기선 **보고서가 코드를 살려 놨다.**
