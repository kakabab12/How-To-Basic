# 5챕터 — ★ 초음파 센서의 원리 — `17150`은 어디서 왔나

> **임베디드 리눅스 프로젝트 (2024)** · 5/8챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

```python
# 초음파 센서 핀 설정
TRIG_PIN = 23
ECHO_PIN = 24

def get_distance():
    GPIO.output(TRIG_PIN, GPIO.HIGH)
    time.sleep(0.00001)                    # 10 마이크로초
    GPIO.output(TRIG_PIN, GPIO.LOW)

    pulse_start = time.time()
    while GPIO.input(ECHO_PIN) == GPIO.LOW:
        pulse_start = time.time()

    pulse_end = time.time()
    while GPIO.input(ECHO_PIN) == GPIO.HIGH:
        pulse_end = time.time()

    pulse_duration = pulse_end - pulse_start
    distance = pulse_duration * 17150       # ★ 센서에서의 거리 계산 (cm)
    return distance
```

**HC-SR04 초음파 센서의 표준 코드**다.

#### 동작 원리 — 박쥐와 같다

```
①  TRIG 핀에 10μs 펄스        →  센서가 40kHz 초음파 8발 발사
                                        │
②                                       ▼  ~~~~~~~~▶  📦
                                                      물체
③  ECHO 핀이 HIGH가 됨        ◀────────  ~~~~~~~~
    (소리가 돌아올 때까지 유지)
④  ECHO가 LOW로 떨어짐
    → HIGH였던 시간 = 왕복 시간
```

#### `17150`의 정체

**마법의 숫자가 아니다. 유도해 보면:**

```
공기 중 음속 = 343 m/s  (섭씨 20도 기준)
            = 34,300 cm/s

거리 = 속도 × 시간
     = 34300 × pulse_duration      ← 이건 "왕복" 거리다!

물체까지의 거리 = 왕복 / 2
               = 34300 / 2 × pulse_duration
               = 17150 × pulse_duration   ★
```

**`17150 = 34300 ÷ 2`** — 음속의 절반이다.
**나누기 2가 "소리가 갔다가 돌아왔다"**는 뜻이다.

> **주석이 `# 센서에서의 거리 계산 (cm)`라고만 되어 있다.** `17150`이 어디서 왔는지는
> 안 적혀 있다. HC-SR04 예제 코드에서 그대로 가져온 것으로 보인다.
> **알고 쓰면 온도 보정도 할 수 있다** — 음속은 온도에 따라 변한다
> (`331.5 + 0.6 × 섭씨온도` m/s).

#### ⚠ 이 함수의 위험 — 무한 루프

```python
while GPIO.input(ECHO_PIN) == GPIO.LOW:
    pulse_start = time.time()
```

**타임아웃이 없다.**

**센서가 고장 났거나, 선이 빠졌거나, 물체가 너무 멀어서 반향이 안 돌아오면**
`ECHO`가 영원히 `LOW`에 머물고 **이 루프가 무한히 돈다.**

```
정상:  ECHO LOW → (곧) HIGH → (곧) LOW → 거리 계산
고장:  ECHO LOW → LOW → LOW → LOW → ... 영원히
       프로그램이 멈춘다 (Ctrl+C 도 안 먹는다)
```

**고치려면:**

```python
timeout = time.time() + 0.05        # 50ms 제한 (약 8m)
while GPIO.input(ECHO_PIN) == GPIO.LOW:
    pulse_start = time.time()
    if pulse_start > timeout:
        return -1                   # 측정 실패
```

> **HC-SR04 예제 코드에 흔한 문제**다. 인터넷의 대부분 예제가 타임아웃이 없다.
> 실습에서는 잘 돌아가지만 **실제 제품이라면 반드시 넣어야 한다.**

#### 바쁜 대기(busy-wait)

```python
while GPIO.input(ECHO_PIN) == GPIO.HIGH:
    pulse_end = time.time()
```

**CPU를 100% 쓰면서 기다린다.** `time.sleep()`이 없다.

**그런데 이건 맞다.** 초음파 왕복은 **수 밀리초**라서, `sleep(0.001)`을 넣으면
**측정 정밀도가 1ms(≈17cm) 단위로 뭉개진다.** 정밀한 시간 측정에는 바쁜 대기가 필요하다.
