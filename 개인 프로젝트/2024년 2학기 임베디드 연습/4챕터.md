#### 주제

> **주제 : 배달 감지 센서 카메라**

> 이 프로젝트는 **카메라 모듈과 초음파 센서 모듈**을 활용하여 배달 물품이 도착했을 때
> 이를 감지하고 사용자에게 보여주는 카메라를 설계하고 구현하는 것을 목표로 합니다.

#### 선정 이유가 이 프로젝트의 백미다

> 현재의 현관 초인종은 **바닥을 제외한 정면에 보이는 것만** 확인할 수 있지만,
> **바닥 부분이 보이지 않는다**는 단점이 있습니다.
>
> **ex) 배달기사가 초인종을 안 누르고 문 앞에 배달음식을 두고 간 상황.**

**실제 불편에서 출발한 문제 정의**다.

```
    기존 초인종 카메라              이 프로젝트
    ┌─────────┐                    ┌─────────┐
    │  👁      │ ← 정면만 본다      │  👁      │
    │         │                    │    ↓     │ ← 아래도 본다
    │         │                    │  📦      │
    └─────────┘                    └─────────┘
       📦  ← 바닥의 배달음식을         + 초음파 센서로 "있다/없다" 판정
           못 본다
```

**"초인종을 안 누르고 두고 가면 온 줄도 모른다"** — 누구나 겪는 일이고,
그걸 **기존 제품의 구조적 한계**(정면만 본다)로 정확히 짚었다.

> **서술형 상황을 요구사항으로 바꾸는 습관**이 여기서부터 보인다 —
> "배달기사가 문 앞에 두고 간다"(상황) → "바닥을 볼 수 있어야 한다"(요구사항).
> 문제 정의가 먼저고 기술은 그다음이다.

#### 기능 명세

> `sudo python3 fooddetect.py`로 실행을 하면 Live Video가 켜집니다.
> **초음파 센서가 물건을 감지하면 화면에 "배달이 감지되었습니다"라는 메시지가 표시**되며,
> 물건이 없으면 "아무것도 감지되지 않았습니다"라고 표시됩니다.
> LED는 작동 상태를 표시하는 신호 역할을 하며, **어두운 곳에서는 조명 역할**도 합니다.
> 또한 **물건이 감지되면 LED가 켜지도록** 설계되었습니다.

**바디캠과 LED 로직이 다르다:**

| | 바디캠 | 배달 감지 |
|---|---|---|
| LED 켜지는 조건 | **카메라가 켜지면** | **물건이 감지되면** |
| 의미 | "녹화 중" | **"배달 왔다"** |

**같은 부품(LED)을 다른 의미로** 쓴다. 배달 감지에서는 LED가 **알림**이자 **조명**이다.

#### 왜 카메라만으로 안 하고 초음파를 쓰나

**2024년 시점에서 옳은 선택**이다:

| 방법 | 필요한 것 |
|---|---|
| **초음파 센서** | 센서 하나 + 거리 비교 한 줄 |
| 영상으로 물체 감지 | **객체 검출 모델 + 학습 데이터 + 연산 능력** |

**라즈베리파이에서 YOLO를 돌리는 건 1년 뒤(2025년 프로젝트)에 가서야 한다.**
지금은 **초음파로 "있다/없다"만 판정하고, 카메라는 "보여주기"만** 한다.

> **역할 분담이 명확하다**: 센서 = 판정, 카메라 = 확인.
> **적정 기술의 좋은 예**다.
