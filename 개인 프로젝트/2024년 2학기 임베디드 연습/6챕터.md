**보고서에서 추출한 전체 코드:**

```python
import time
import RPi.GPIO as GPIO
import cv2
from picamera2 import Picamera2

# 초음파 센서 핀 설정
TRIG_PIN = 23
ECHO_PIN = 24
# LED 핀 설정
LED_PIN = 25

GPIO.setmode(GPIO.BCM)
GPIO.setup(TRIG_PIN, GPIO.OUT)
GPIO.setup(ECHO_PIN, GPIO.IN)
GPIO.setup(LED_PIN, GPIO.OUT)

def get_distance():
    ...  # 5단계

# 배달 감지 함수
def detect_food():
    distance = get_distance()
    if distance < 10:                       # 음식이 감지되었을 때 (예: 10cm 이하)
        GPIO.output(LED_PIN, GPIO.HIGH)     # LED 켜기
        return "Baedal object detect!!"
    else:
        GPIO.output(LED_PIN, GPIO.LOW)      # LED 끄기
        return "There is nothing."

# 실시간 영상 스트리밍 함수
def video_stream():
    picam2 = Picamera2()
    picam2.start()

    while True:
        frame = picam2.capture_array()
        frame_resized = cv2.resize(frame, (640, 480))

        # 초음파 센서로 음식 감지 및 상태 출력
        status = detect_food()

        # 상태 텍스트를 이미지에 추가
        font = cv2.FONT_HERSHEY_SIMPLEX
        cv2.putText(frame_resized, status, (10, 30), font, 1, (0, 255, 0), 2, cv2.LINE_AA)

        cv2.imshow("Live Video", frame_resized)

        if cv2.waitKey(1) & 0xFF == ord('q'):
            break

    picam2.stop()
    cv2.destroyAllWindows()

if __name__ == '__main__':
    try:
        video_stream()
    except KeyboardInterrupt:
        GPIO.cleanup()
        cv2.destroyAllWindows()
```

#### 핀 3개를 쓴다

| 핀 | 방향 | 역할 |
|---|---|---|
| `TRIG_PIN = 23` | **OUT** | 초음파 발사 신호 |
| `ECHO_PIN = 24` | **IN** | 반향 수신 |
| `LED_PIN = 25` | **OUT** | 알림 + 조명 |

**`GPIO.setup(ECHO_PIN, GPIO.IN)`만 입력**이다. 나머지는 출력.
**방향을 잘못 설정하면 센서가 안 읽히거나 최악의 경우 핀이 탄다.**

#### `detect_food()` — 판정과 부작용이 섞여 있다

```python
def detect_food():
    distance = get_distance()
    if distance < 10:
        GPIO.output(LED_PIN, GPIO.HIGH)     # ← 부작용 (LED 제어)
        return "Baedal object detect!!"     # ← 반환값 (문자열)
```

**함수 하나가 두 가지를 한다**: LED를 켜고, 문자열을 반환한다.

**더 깔끔하게 하려면:**

```python
def is_detected():
    return get_distance() < 10           # 판정만

# 호출부
detected = is_detected()
GPIO.output(LED_PIN, GPIO.HIGH if detected else GPIO.LOW)
status = "Baedal object detect!!" if detected else "There is nothing."
```

**다만 이 규모에서는 문제가 아니다.** 함수 하나로 끝나는 게 오히려 읽기 쉽다.

#### `cv2.putText` — 영상에 글자 새기기

```python
font = cv2.FONT_HERSHEY_SIMPLEX
cv2.putText(frame_resized, status, (10, 30), font, 1, (0, 255, 0), 2, cv2.LINE_AA)
#            ─────┬─────  ──┬──   ──┬──  ─┬─ ┬  ─────┬────  ┬  ──────┬─────
#             대상 이미지  텍스트  위치  폰트 크기   색(BGR!)  두께  안티앨리어싱
```

| 인자 | 값 | 의미 |
|---|---|---|
| `(10, 30)` | 좌상단 | 좌표 |
| `1` | 배율 | 폰트 크기 |
| **`(0, 255, 0)`** | **초록** | **BGR 순서!** (RGB가 아니다) |
| `2` | 두께 | |
| `cv2.LINE_AA` | 안티앨리어싱 | 글자가 부드러워진다 |

**OpenCV는 색을 BGR 순서로 받는다.** `(0, 255, 0)`은 RGB로도 BGR로도 초록이라 안 걸리지만,
빨강을 넣으려고 `(255, 0, 0)`을 쓰면 **파랑이 나온다.** 흔한 함정이다.

> **한글은 `cv2.putText`로 못 찍는다.** 그래서 `"Baedal object detect!!"`처럼
> **영어(+로마자 "Baedal")**로 썼다. 보고서 본문에는 "배달이 감지되었습니다"라고
> 되어 있지만 **실제 화면에는 영어**가 나온다.
>
> 한글을 넣으려면 PIL로 그려서 합성해야 한다 — 번거로워서 다들 영어로 우회한다.

#### ⚠ 매 프레임마다 초음파를 쏜다

```python
while True:
    frame = picam2.capture_array()
    ...
    status = detect_food()          # ← 프레임마다 초음파 측정!
```

**초음파 측정은 수 밀리초가 걸리고 바쁜 대기를 한다**(5단계).
**30fps면 초당 30번** 쏘는 셈이다.

**문제:**
- 프레임 레이트가 떨어진다
- 초음파가 너무 자주 발사되면 **이전 반향이 아직 남아 있어 오측정**될 수 있다
  (HC-SR04 데이터시트는 **60ms 이상 간격**을 권장한다)

**고치려면** 측정 주기를 분리한다:

```python
last_check = 0
status = "There is nothing."
while True:
    frame = picam2.capture_array()
    now = time.time()
    if now - last_check > 0.1:       # 100ms마다만
        status = detect_food()
        last_check = now
    ...
```

#### ⚠ `bodycam.py`보다 정리가 허술하다

```python
def video_stream():
    ...
    picam2.stop()                    # ← finally 가 아니다!
    cv2.destroyAllWindows()

if __name__ == '__main__':
    try:
        video_stream()
    except KeyboardInterrupt:        # ← KeyboardInterrupt 만 잡는다
        GPIO.cleanup()
        cv2.destroyAllWindows()
```

**`bodycam.py`는 `try/except/finally`로 감쌌는데**(3단계), **여기는 `finally`가 없다.**

| 상황 | `bodycam.py` | `fooddetect.py` |
|---|---|---|
| `q` 로 정상 종료 | ✅ 정리됨 | ⚠ `GPIO.cleanup()` **안 됨** |
| `Ctrl+C` | ✅ 정리됨 | ✅ 정리됨 |
| 다른 예외 (센서 오류 등) | ✅ 정리됨 | ❌ **정리 안 됨** |

**`q`로 끄면 `GPIO.cleanup()`이 호출되지 않는다** — `break` 후 `picam2.stop()`은 하지만
GPIO는 안 놓는다. **LED가 켜진 채 남을 수 있다.**

> **같은 보고서 안의 두 코드인데 품질이 다르다.** `bodycam.py`를 먼저 만들면서
> `finally`를 배웠는데, `fooddetect.py`를 쓸 때 그걸 적용하지 않은 것으로 보인다.
> **배운 걸 다음 코드에 적용하는 게 생각보다 어렵다**는 흔한 사례다.
