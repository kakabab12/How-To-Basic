# 5챕터 — ★ `style2.css` — 유틸리티 클래스

> **XAMPP 웹 자기소개서 (2025)** · 5/8챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

**26줄 전체:**

```css
.color-red   { color: red; }
.color-blue  { color: blue; }

.bg-yellow   { background-color: yellow; }
.bg-light    { background-color: #e0f7fa; }

.small       { font-size: 12px; }
.large       { font-size: 24px; }

.search-links { margin-top: 30px; }
```

#### CSS 두 개는 "두 가지 버전"이 아니다

**처음엔 style1과 style2가 대안(A안/B안)인 줄 알기 쉽다. 아니다.**

```html
<link rel="stylesheet" href="style1.css">
<link rel="stylesheet" href="style2.css">     ← 둘 다 적용된다!
```

**둘 다 로드되어 함께 적용**된다. **역할이 나뉘어 있다:**

| 파일 | 역할 | 선택자 |
|---|---|---|
| **`style1.css`** | **요소 스타일** — 레이아웃·타이포 | `body`, `table`, `.title` |
| **`style2.css`** | **유틸리티 클래스** — 색·크기 | `.color-red`, `.bg-yellow`, `.small` |

#### 이게 유틸리티 우선(utility-first) CSS다

```html
<p class="color-red bg-yellow">저는 <strong>개발자</strong>가 되고 싶습니다!</p>
   ────────────────────────
   클래스를 조합해서 스타일링
```

**한 클래스 = 한 속성**이다. 조합해서 쓴다.

**이건 Tailwind CSS의 철학과 같다:**

```html
<!-- 이 과제 (2025) -->
<p class="color-red bg-yellow">

<!-- Tailwind -->
<p class="text-red-500 bg-yellow-300">
```

> **본인이 Tailwind를 알고 짰을 리는 없다.** 과제 요구사항이
> **"글자 색상 2종 이상, 배경 색상 2종 이상"**이라 자연스럽게 그렇게 됐을 것이다.
> **요구사항이 구조를 만든 것**이다.
>
> 그래도 **파일을 나눈 판단은 좋다.** "요소 스타일"과 "재사용 유틸"을 분리하는 건
> 규모가 커져도 유효한 원칙이다.

#### `.search-links`가 style2에 있는 게 어색하다

```css
.search-links { margin-top: 30px; }     /* ← 유틸리티가 아니라 레이아웃 */
```

**나머지는 전부 "색/크기" 유틸인데 이것만 레이아웃**이다. `style1.css`에 있어야 맞다.
**막판에 여백이 필요해서 열려 있던 파일에 넣은** 것으로 보인다.

> **파일 분리 규칙은 만들기는 쉬워도 지키기는 어렵다**는 작은 사례다.

#### `red`, `yellow` 같은 색 이름

```css
.color-red  { color: red; }
.bg-yellow  { background-color: yellow; }
.bg-light   { background-color: #e0f7fa; }    ← 이것만 hex
```

**CSS 색 이름은 140개**가 표준으로 정의되어 있다. 동작에는 문제없다.

다만 **`red`와 `yellow`는 매우 쨍한 원색**이라 함께 쓰면 눈이 아프다.
`.bg-light`(`#e0f7fa`, 연한 하늘색)만 hex로 골랐는데 **이게 가장 보기 좋다** —
색을 직접 고르면 결과가 낫다는 걸 보여 준다.
