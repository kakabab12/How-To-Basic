**63줄 전체 구조:**

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <title>이지용의 개인 홈페이지</title>
  <link rel="stylesheet" href="style1.css">
  <link rel="stylesheet" href="style2.css">        ← ★ CSS 2개 (5단계)
</head>
<body>

  <h1 class="title"> 안녕하세요! 이지용입니다</h1>

  <!-- 1. 본인 사진 -->
  <img src="myphoto.jpg" alt="본인 사진" width="200" />
  <img src="myphoto.jpg" alt="친구 사진" width="200" />       ← ★ 6단계
  <img src="myphoto.jpg" alt="카자루 사진" width="200" />

  <!-- 2. 테이블로 자기소개 -->
  <table border="1">
    <tr><th>항목</th><th>내용</th></tr>
    <tr><td>이름</td><td><i>이지용</i></td></tr>              <!-- 5. 이탤릭 -->
    <tr><td>전공</td><td>컴퓨터공학</td></tr>
    <tr><td>취미</td><td>운동, 프로그래밍</td></tr>
  </table>

  <!-- 3. 글자 색상 2종 이상, 4. 배경 색상 2종 이상 -->
  <p class="color-red bg-yellow">저는 <strong>개발자</strong>가 되고 싶습니다!</p>
  <p class="color-blue bg-light">지금은 <em>Node.js</em>와 <em>MongoDB</em>를 배우고 있어요.</p>

  <!-- 6. 글씨 크기 다르게 -->
  <p class="small">작은 글씨입니다</p>
  <p class="large">큰 글씨입니다</p>

  <!-- 8. 검색엔진 이미지 링크 -->
  <div class="search-links">
    <a href="https://www.naver.com" target="_blank">
      <img src="naver.png" alt="네이버" width="80">
    </a>
    <a href="https://www.google.com" target="_blank">
      <img src="google.png" alt="구글" width="80">
    </a>
  </div>

  <!-- 추가점수: 좋아하는 유튜브 영상 삽입 -->
  <h3> 좋아하는 유튜브 영상</h3>
  <iframe width="560" height="315" src="https://www.youtube.com/watch?v=..." ...></iframe>

</body>
</html>
```

#### 잘한 것들

| 항목 | 코드 | 왜 좋은가 |
|---|---|---|
| **`<!DOCTYPE html>`** | 첫 줄 | HTML5 표준 모드 (없으면 쿼크 모드) |
| **`lang="ko"`** | `<html lang="ko">` | 스크린 리더·번역기가 한국어로 인식 |
| **`<meta charset="UTF-8">`** | `<head>` 첫 줄 | **한글 깨짐 방지** |
| **`alt` 속성** | 모든 `<img>` | 접근성 + 이미지 깨졌을 때 대체 텍스트 |
| **`<th>` 사용** | 테이블 헤더 | `<td>`가 아니라 `<th>` — 의미가 맞다 |
| **`target="_blank"`** | 외부 링크 | 새 탭으로 |

**`<meta charset="UTF-8">`이 `<head>`의 첫 줄에 있는 것도 맞다.**
브라우저는 인코딩을 알기 전까지 바이트를 못 읽으므로 **최대한 앞**에 와야 한다.

#### `<i>` vs `<em>`, `<strong>` vs `<b>`

**둘 다 쓰고 있다:**

```html
<td><i>이지용</i></td>                          <!-- 5. 이탤릭 -->
<p>...<strong>개발자</strong>가 되고 싶습니다!</p>
<p>지금은 <em>Node.js</em>와 <em>MongoDB</em>를 배우고 있어요.</p>
```

| 태그 | 의미 | 용도 |
|---|---|---|
| `<i>` | **모양만** 이탤릭 | 이름, 학명, 외래어 |
| `<em>` | **강조** (의미) | 스크린 리더가 억양을 바꾼다 |
| `<b>` | **모양만** 굵게 | — |
| `<strong>` | **중요** (의미) | 스크린 리더가 강조한다 |

**HTML5에서 `<i>`와 `<b>`는 폐기되지 않았다.** "모양 전용"으로 의미가 재정의됐다.
이름에 `<i>`를 쓴 건 **오히려 적절**하다(강조가 아니라 그냥 이탤릭체).

> **의도한 것인지 우연인지는 모르지만 결과적으로 맞게 썼다.**

#### `border="1"`은 옛날 방식이다

```html
<table border="1">
```

**HTML 속성으로 스타일을 지정**하는 건 CSS 이전 방식이다. 지금은:

```css
table { border: 1px solid black; border-collapse: collapse; }
```

**`style1.css`에 `border-collapse: collapse`가 있는 걸 보면**(4단계) CSS도 쓰고 있다.
**속성과 CSS를 섞어 쓰는 중**이다 — 배우는 과정에서 흔한 과도기다.
