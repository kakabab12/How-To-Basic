# 2챕터 — ★ 주석이 채점 기준표다

> **XAMPP 웹 자기소개서 (2025)** · 2/8챕터 · [📚 목차](README.md) · [개인 프로젝트 전체](../README.md)

---

**이 파일에서 가장 흥미로운 발견**이다. `index.html`의 주석을 모아 보면:

```html
<!-- 1. 본인 사진 -->
<!-- 2. 테이블로 자기소개 -->
<!-- 3. 글자 색상 2종 이상, 4. 배경 색상 2종 이상 -->
<!-- 5. 이탤릭 -->
<!-- 6. 글씨 크기 다르게 -->
<!-- 8. 검색엔진 이미지 링크 -->
<!-- 추가점수: 좋아하는 유튜브 영상 삽입 -->
```

**번호가 붙어 있다.** 이건 코드 설명 주석이 아니라 **과제 요구사항 목록**이다.

#### 재구성한 채점 기준

| # | 요구사항 | 구현 |
|---|---|---|
| 1 | 본인 사진 | `<img src="myphoto.jpg">` |
| 2 | 테이블로 자기소개 | `<table border="1">` |
| 3 | 글자 색상 2종 이상 | `.color-red`, `.color-blue` |
| 4 | 배경 색상 2종 이상 | `.bg-yellow`, `.bg-light` |
| 5 | 이탤릭 | `<i>이지용</i>` |
| 6 | 글씨 크기 다르게 | `.small`, `.large` |
| **7** | **?** | **주석에 없다** |
| 8 | 검색엔진 이미지 링크 | 네이버·구글 |
| + | **추가점수** | **YouTube 영상** |

**7번 주석이 없다.** 두 가지 가능성:
- 7번은 주석 없이 구현됐다 (예: `<h1>` 제목, `<strong>` 굵게)
- 7번을 빠뜨렸다 (그런데 15점 만점이니 아닐 것)

> `<strong>개발자</strong>`와 `<em>Node.js</em>`가 주석 없이 들어가 있다.
> **7번이 "굵게/강조"였을 가능성**이 높다.

#### 이 습관이 왜 좋은가

**요구사항 번호를 코드에 남기면:**

| 이점 | 설명 |
|---|---|
| **빠뜨림 방지** | 1~8을 훑으면 누락이 바로 보인다 |
| **채점자가 편하다** | "3번 어디 있지?" 하고 찾을 필요가 없다 |
| **자기 검증** | 제출 전에 체크리스트가 된다 |

**15점 만점의 이유가 이것일 수도 있다.** 요구사항을 하나하나 짚어서 보여 줬다.

> **실무에도 같은 습관이 있다.** 코드에 티켓 번호를 다는 것:
> ```python
> # JIRA-1234: 재시도 로직 추가
> ```
> **"왜 이 코드가 있는가"를 추적 가능하게** 만드는 것 — 근본은 같다.
