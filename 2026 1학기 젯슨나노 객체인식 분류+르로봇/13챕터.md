# 13챕터 — 쿨다운과 안전 정지

> **젯슨 나노 객체인식 분류 + 르로봇** · 13/15챕터 · [📚 목차](README.md)

---

#### 쿨다운 — 연타 방지

```python
COMMAND_COOLDOWN = 5.0

now = time.time()
if system_running and (now - last_cmd_t2 > COMMAND_COOLDOWN):
    ...
    last_cmd_t2 = now
```

**같은 박스가 30프레임 연속 보이면 명령을 30번 보내게 된다.** 로봇팔이 미쳐 버린다.
5초 쿨다운으로 막았다.

**타이머가 카메라별로 따로**다 (`last_cmd_t1`, `last_cmd_t2`) — 저울 쪽과 박스 쪽이
서로의 쿨다운에 걸리면 안 되니 맞는 설계다.

> 라즈베리파이 편의 `threading.Timer(3.0, release_target)`(락 방식)과 비교하면,
> 젯슨은 **타임스탬프 비교**로 더 단순하게 갔다. 스레드를 만들지 않아 가볍다.

#### `system_running` — 소프트웨어 비상 정지

```python
if over and system_running and (now - last_cmd_t1 > COMMAND_COOLDOWN):
```

**모든 명령 전송이 `system_running`을 확인한다.** 웹 대시보드의 STOP 버튼이
`/control`로 이걸 `False`로 바꾸면 **추론과 스트리밍은 계속되지만 로봇은 안 움직인다.**

전시회에서 **"화면은 보여 주되 로봇은 세우고 싶은"** 상황을 위한 설계다.
`process_frame`은 상태를 화면에도 찍는다:

```python
status_color = (0,255,0) if system_running else (0,0,255)
cv2.putText(annotated, f"SYS: {'RUNNING' if system_running else 'STOPPED'}", ...)
```

#### CAM1은 아직 로봇에 연결 안 됨

```python
if over and system_running and (now - last_cmd_t1 > COMMAND_COOLDOWN):
    # [수정됨] 나중에 르로봇과 HTTP 통신을 연동할 자리입니다. (현재는 기록만 남김)
    action_log_1.append({"action": f"초과 중량 감지 ({weight}g)", "result": "기록됨 (HTTP 전송 대기)"})
    last_cmd_t1 = now
```

**무게 초과를 감지해도 로그만 남긴다.** 르로봇(LeRobot ACT) 연동이 **미완**이라는 뜻이다.
주석이 솔직하게 그 자리를 표시해 뒀다.

즉 **현재 실제로 로봇을 움직이는 건 CAM2(박스)뿐**이고, CAM1(무게)은 감지·표시까지만 한다.
