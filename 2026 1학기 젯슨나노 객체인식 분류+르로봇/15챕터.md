#### `lerobot/act_floor1_test.py` — 본인이 쓴 유일한 lerobot 파일

```python
import os, subprocess

# GPU / CPU 부하 제한
os.environ["CUDA_VISIBLE_DEVICES"] = "0"
os.environ["OMP_NUM_THREADS"] = "1"
os.environ["OPENBLAS_NUM_THREADS"] = "1"
os.environ["MKL_NUM_THREADS"] = "1"
os.environ["NUMEXPR_NUM_THREADS"] = "1"

cmd = [
    "lerobot-record",
    "--robot.type=so101_follower",
    "--robot.port=/dev/ttyACM0",
    "--robot.id=my_follower",
    # 카메라 1개 + 더 낮은 해상도/FPS
    "--robot.cameras={ front: {type: opencv, index_or_path: 2, width: 160, height: 120, fps: 10}}",
    # ACT 모델 경로
    "--policy.path=/home/user/project/lerobot/outputs/train/act_floor1_stack/checkpoints/080000/pretrained_model",
    # 평가 데이터셋
    "--dataset.root=/home/user/project/lerobot/eval_floor1",
    "--dataset.single_task=stack_box_floor1",
    "--dataset.num_episodes=1",
]
```

**읽어낼 사실:**

| 항목 | 값 |
|---|---|
| 로봇 | **SO-101 팔로워 암**, `/dev/ttyACM0` |
| 과제 | **`stack_box_floor1`** — 바닥의 박스 쌓기 |
| 정책 | **ACT** (Action Chunking Transformer) |
| 학습량 | **80,000 스텝** 체크포인트 |
| 카메라 | **160×120 @ 10fps** ← 극단적으로 낮다 |

**이 파일 전체가 "젯슨 나노의 자원 한계와 싸운 기록"**이다:

- 스레드 수를 **전부 1로** 묶었다 (`OMP`, `OPENBLAS`, `MKL`, `NUMEXPR`)
- 카메라를 **1개**로 줄이고 **160×120 @ 10fps**로 낮췄다 (주석: `카메라 1개 + 더 낮은 해상도/FPS`)
- 에피소드 **1개**만

> **ACT는 Transformer 정책**이라 무겁다. 데스크톱 GPU 기준으로 만들어진 걸 젯슨 나노에
> 욱여넣으려니 이렇게까지 깎아야 했다.

#### `로봇팔.txt` — 실행 순서 메모

```
conda activate lerobot312    # 객체인식용 환경
conda activate lerobot       # ACT 실행용 환경

export PYTHONPATH=/home/user/project/lerobot/src:$PYTHONPATH
export OMP_NUM_THREADS=1
export CUDA_MODULE_LOADING=LAZY
export PYTORCH_CUDA_ALLOC_CONF=max_split_size_mb:128
```

- **conda 환경을 2개로 분리**했다 — `lerobot312`(객체인식) / `lerobot`(ACT).
  두 작업의 의존성이 충돌해서 나눈 것으로 보인다.
- **`CUDA_MODULE_LOADING=LAZY`** — CUDA 커널을 필요할 때만 로드해 초기 메모리를 아낀다.
- **`PYTORCH_CUDA_ALLOC_CONF=max_split_size_mb:128`** — 메모리 조각화로 인한 **OOM 방지**.
  젯슨은 CPU와 GPU가 메모리를 공유해서 특히 빠듯하다.

> **재현 불가**: `outputs/train/.../checkpoints/080000/pretrained_model` 체크포인트가
> **저장소에 없다**(`Fix: Large files removed from history` 커밋으로 제거됨).
> 이 스크립트는 그대로는 못 돌린다. LeRobot 공식 문서대로 직접 학습해야 한다.

#### 젯슨 전용 의존성

```bash
pip install onnxruntime_gpu-1.17.0-cp310-cp310-linux_aarch64.whl
```

젯슨은 **aarch64**라 pip 기본 저장소에 GPU용 onnxruntime이 없다. NVIDIA가 배포하는
휠을 직접 받아 넣어야 해서 저장소에 함께 커밋해 둔 것이다. (`cp310` = **Python 3.10 전용**)
