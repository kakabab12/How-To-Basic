# 9챕터 — 듀얼 카메라 — RealSense + USB 웹캠

> **젯슨 나노 객체인식 분류 + 르로봇** · 9/15챕터 · [📚 목차](README.md)

---

```python
import pyrealsense2 as rs      # CAM1
import cv2                     # CAM2
CAM2_INDEX = 6
```

**서로 다른 API를 쓰는 카메라 2대**를 한 프로그램에서 돌린다.

| | CAM1 | CAM2 |
|---|---|---|
| 장치 | **Intel RealSense** | 일반 USB 웹캠 (**다이소**) |
| API | `pyrealsense2` | `cv2.VideoCapture(6)` |
| 대상 | 저울 화면의 무게 숫자 | 컨베이어 박스 |
| 모델 | `number.engine` | `box.pt` |

#### 왜 이런 조합이 됐나

**원래 계획(Plan A)은 CSI 카메라(IMX219) 2대**였다. GStreamer `nvarguscamerasrc`로
하드웨어 인코딩 영상을 받는 고성능 설계였다. **CSI 포트가 고장 나서** 포기했다.

그래서 **손에 있는 걸로 대체**했다: 연구실에 있던 RealSense + 다이소에서 산 웹캠.
`코드설명.md`의 표현이 생생하다 — *"고장 난 CSI 포트를 버리고"*.

> **`CAM2_INDEX = 6`은 마법의 숫자다.** 리눅스에서 USB 카메라는 `/dev/video0`, `/dev/video2`...
> 로 잡히는데, RealSense가 여러 개(컬러/깊이/적외선)를 차지하면서 웹캠이 6번으로 밀린 것이다.
> **다른 기기에서는 반드시 `ls /dev/video*`로 다시 확인해야 한다.**
