# 6챕터 — ⚠ 모델의 진짜 클래스 vs 코드의 라벨 맵 (버그 발견)

> **젯슨 나노 객체인식 분류 + 르로봇** · 6/15챕터 · [📚 목차](README.md)

---

**모델 파일의 메타데이터를 직접 열어서** 실제 클래스를 확인한 결과다.

#### 진짜 클래스 (모델이 말하는 것)

```
box.onnx / box.pt  :  {0: 'big_box', 1: 'small_box'}
number.onnx        :  {0: '-', 1: '.', 2: '0', 3: '1', 4: '2', 5: '3', 6: '4',
                       7: '5', 8: '6', 9: '7', 10: '8', 11: '9', 12: 'screen'}
```

#### 숫자 모델 — 코드와 완벽히 일치 ✅

```python
DIGIT_CLASS_MAP = {
    2: "0", 3: "1", 4: "2", 5: "3", 6: "4",
    7: "5", 8: "6", 9: "7", 10: "8", 11: "9"
}
```

**정확하다.** 그리고 이 맵이 **`0:'-'`와 `1:'.'`을 일부러 뺐다**는 점이 중요하다 —
저울의 마이너스 기호와 소수점을 무시하겠다는 뜻이다. `12: 'screen'`도 빠져 있는데,
이건 10단계에서 **다른 용도**로 쓴다.

#### 박스 모델 — 코드가 뒤집혀 있다 ❌

```python
BOX_LABEL_MAP = {
    0: "small_box",     # ← 모델은 0을 'big_box'라고 한다!
    1: "samll_box"      # ← 오타(samll). 모델은 1을 'small_box'라고 한다
}
```

그리고 명령 분기가 **이 잘못된 맵을 따라간다**:

```python
if cls_id == 0:
    cmd_str = "SORT:1\n"      # 1번: 소형 경로 로직
    log_msg = "소형 박스 픽업"   # ← 실제로는 big_box인데 "소형"이라고 보낸다
elif cls_id == 1:
    cmd_str = "SORT:3\n"      # 3번: 대형 경로 로직
    log_msg = "대형 박스 픽업"   # ← 실제로는 small_box인데 "대형"이라고 보낸다
```

**결과: 큰 박스가 소형 라인으로, 작은 박스가 대형 라인으로 간다.**

#### 계보를 비교하면 언제 깨졌는지가 보인다

| 파일 | `BOX_LABEL_MAP` | 명령 분기 | 판정 |
|---|---|---|---|
| `jetson_main1(9).py` (Plan A) | `{0: "big_box", 1: "samll_box"}` | — | **0=big ✅ 맞다** |
| `jetson_USB.py` | `{0: "small_box", 1: "samll_box"}` ❌ | `0 → MOVE_BIG "대형 박스 픽업"` ✅ | **라벨만 틀림** (동작은 정상) |
| `jetson_USB2.py` (최종) | `{0: "small_box", 1: "samll_box"}` ❌ | `0 → SORT:1 "소형 박스 픽업"` ❌ | **둘 다 뒤집힘** |

**사고 경위 추정**:
1. `jetson_main1(9).py`는 `0: "big_box"`로 **맞게** 적었다.
2. 어느 시점에 라벨 맵이 `0: "small_box"`로 잘못 바뀌었다(오타로 보인다 — `samll`도 같이 있다).
3. `jetson_USB.py`는 라벨이 틀렸지만 **명령은 여전히 맞았다** (`0 → MOVE_BIG`).
4. `jetson_USB2.py`에서 **"라벨과 명령이 안 맞네"** 하고 **명령을 라벨에 맞춰 고쳤다.**
   → 틀린 쪽에 맞춘 것이라 **동작이 뒤집혔다.**

> **교훈**: 잘못된 이름을 믿고 코드를 고치면 멀쩡하던 동작이 깨진다.
> `samll_box` 같은 오타는 단순 미관 문제가 아니라 — **오타가 있는 곳은 검증되지 않았다는 신호**다.

#### 고치는 법

```python
BOX_LABEL_MAP = {
    0: "big_box",       # box.pt 메타데이터 기준
    1: "small_box"
}
...
if cls_id == 0:
    cmd_str = "SORT:3\n"      # big → 대형 경로
    log_msg = "대형 박스 픽업"
elif cls_id == 1:
    cmd_str = "SORT:1\n"      # small → 소형 경로
    log_msg = "소형 박스 픽업"
```

**실행 전에 반드시 실물로 확인할 것.** 위 표는 모델 메타데이터 기준이고, 아두이노 쪽
`SORT:1`/`SORT:3`이 실제로 어느 칸인지는 아두이노 스케치를 봐야 한다(저장소에 없다).
