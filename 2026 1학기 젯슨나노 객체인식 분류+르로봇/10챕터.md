**이 프로젝트에서 가장 영리한 부분**이다.

#### 문제

저울 화면의 숫자는 **프레임 전체에서 아주 작다**. 640×640으로 리사이즈되면 숫자가
몇 픽셀밖에 안 남아서 YOLO가 못 읽는다.

#### 답 — 화면을 먼저 찾고, 거기를 확대해서 다시 본다

```python
if cam_id == 1:   # CAM1 (숫자 인식부)
    boxes1, scores1, cls1 = run_inference(model, frame, CONF_CAM1, max_det=10)   # ① 1차 추론
    zoomed, offset = get_screen_crop(frame, boxes1, scores1, cls1, FW, FH)       # ② 화면 크롭

    if zoomed is not None:
        boxes2, scores2, cls2 = run_inference(model, zoomed, CONF_CAM1, max_det=10)  # ③ 2차 추론
        raw_dets = extract_digits_from(boxes2, scores2, cls2, offset)                # ④ 좌표 역변환
    else:
        raw_dets = extract_digits_from(boxes1, scores1, cls1)   # 폴백: 1차 결과 그대로
```

**같은 모델을 두 번 돌린다.** 1차는 `screen`을 찾으려고, 2차는 그 안의 숫자를 읽으려고.

#### `get_screen_crop` — 12번 클래스의 정체

```python
def get_screen_crop(frame, boxes, scores, class_ids, fw, fh):
    best_box = None; best_conf = 0.0
    for (cx,cy,w,h), score, cls_id in zip(boxes, scores, class_ids):
        if cls_id == 12 and score > best_conf:        # ★ 12 = 'screen'
            best_conf = score; best_box = (cx,cy,w,h)

    if best_box is None: return None, None
    cx,cy,w,h = best_box
    x1=max(0,  int(cx-w/2) - SCREEN_PADDING)          # 패딩 15px
    y1=max(0,  int(cy-h/2) - SCREEN_PADDING)
    x2=min(fw, int(cx+w/2) + SCREEN_PADDING)
    y2=min(fh, int(cy+h/2) + SCREEN_PADDING)
    cropped = frame[y1:y2, x1:x2]
    if cropped.size == 0: return None, None

    zoomed = cv2.resize(cropped, (640, 640))          # ★ 640x640으로 확대
    offset = (x1, y1, (x2-x1)/640, (y2-y1)/640)       # ★ 역변환 정보
    return zoomed, offset
```

**6단계에서 `DIGIT_CLASS_MAP`이 12번을 뺀 이유가 여기 있다.** `screen`은 숫자가 아니라
**"어디를 확대할지 알려주는 앵커"**다. 학습 데이터에 저울 화면 자체를 클래스로 넣은 것이
설계의 출발점이다.

- **`max(0, ...)`, `min(fw, ...)`** — 패딩을 더하다 프레임 밖으로 나가는 걸 막는다
- **`if cropped.size == 0`** — 크롭이 빈 배열이면 `cv2.resize`가 죽는다. 방어.
- **가장 신뢰도 높은 screen 하나만** 쓴다 (`score > best_conf`)

#### 좌표 역변환 — 확대한 곳의 좌표를 원본으로 되돌리기

확대 이미지에서 찾은 숫자는 **확대 좌표계**에 있다. 화면에 그리려면 원본 좌표로 되돌려야 한다.

```python
offset = (x1, y1, (x2-x1)/640, (y2-y1)/640)
#         ─┬─  ─┬─  ─────┬─────  ─────┬─────
#        크롭 원점      가로 축소비    세로 축소비

def extract_digits_from(boxes, scores, class_ids, offset=None):
    for (cx,cy,w,h), score, cls_id in zip(boxes, scores, class_ids):
        digit = DIGIT_CLASS_MAP.get(cls_id)
        if not digit: continue          # ★ 숫자 아닌 것(screen, '-', '.')은 버린다

        if offset:
            ox,oy,csx,csy = offset
            bx1=int((cx-w/2)*640*csx)+ox     # 확대 좌표 → 원본 좌표
            by1=int((cy-h/2)*640*csy)+oy
            bx2=int((cx+w/2)*640*csx)+ox
            by2=int((cy+h/2)*640*csy)+oy
            bcx=int(cx*640*csx)+ox
        else:
            bx1=int(cx-w/2); by1=int(cy-h/2)
            bx2=int(cx+w/2); by2=int(cy+h/2)
            bcx=int(cx)
        raw.append({"digit":digit,"cx":bcx,"conf":score,"box":(bx1,by1,bx2,by2)})
    return raw
```

**`* 640 * csx + ox`** = (정규좌표 → 확대픽셀) → (확대픽셀 → 원본픽셀) → (크롭 원점 더하기).

**`DIGIT_CLASS_MAP.get(cls_id)`가 `None`이면 `continue`** — 이 한 줄이 `screen`(12),
`-`(0), `.`(1)을 자동으로 걸러낸다. **맵에 없으면 무시**라는 단순한 규칙이다.

> **이 기법의 이름**: 일반적으로 **2단계 검출(coarse-to-fine)** 또는 **ROI 줌**이라고 한다.
> 작은 객체 검출의 표준 해법이고, 이 코드는 그걸 **모델 하나로**(screen 클래스를 같이
> 학습시켜서) 구현했다. 별도 검출기를 두 개 만들지 않은 게 영리하다.
