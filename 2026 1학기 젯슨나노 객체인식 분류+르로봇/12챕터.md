#### 라즈베리파이 편과의 결정적 차이

| | 라즈베리파이 `sucess.py` | 젯슨 `jetson_USB2.py` |
|---|---|---|
| 크기 판정 | **마스크 면적으로 직접 계산** | **모델 클래스를 그대로 사용** |
| 불량 판정 | **R값 기하 규칙** | 없음 |
| 안정화 | **7프레임 투표** | 없음 (쿨다운만) |
| 모델 task | **segment** | **detect** |

**젯슨은 모델을 믿는다.** 라즈베리파이 편에서 "YOLO를 믿지 않고 기하로 재계산"했던 것과
정반대다. 모델을 `big_box`/`small_box` **2클래스로 단순화**하고 판정을 학습에 맡겼다.

> 그리고 **바로 그것이 6단계 버그가 치명적인 이유**다. 기하 규칙으로 재계산하는 구조였다면
> 라벨 맵이 틀려도 실제 크기로 바로잡혔을 텐데, 모델 클래스를 그대로 믿는 구조라
> **라벨 맵이 곧 진실**이 된다.

#### `detection_log_2.pop(0)` — 수동 링버퍼

```python
detection_log_2.append({...})
if len(detection_log_2) > 200: detection_log_2.pop(0)
```

라즈베리파이 편은 `deque(maxlen=20)`으로 이걸 **공짜로** 했다. 젯슨은 리스트 + 수동
`pop(0)`이다. 동작은 같지만 `list.pop(0)`은 **O(n)**(모든 원소를 한 칸씩 당김)이라
`deque.popleft()`(**O(1)**)보다 느리다. 200개 정도면 실용상 문제는 없다.
