# 11챕터 — 숫자 조합 — top-k 후 x정렬

> **젯슨 나노 객체인식 분류 + 르로봇** · 11/15챕터 · [📚 목차](README.md)

```python
def get_top_digits(dets, max_digits=3):
    if not dets: return []
    dets.sort(key=lambda x: x['conf'], reverse=True)   # ① 신뢰도 순
    top_dets = dets[:max_digits]                        # ② 상위 3개만
    top_dets.sort(key=lambda x: x['cx'])                # ③ 왼쪽→오른쪽
    return top_dets

def combine_digits(dets):
    if not dets: return None
    try: return int(''.join(d['digit'] for d in dets))
    except: return None
```

**정렬을 두 번 하는 게 핵심**이다:

1. **신뢰도로 정렬 → 상위 3개** (`MAX_DIGITS = 3`): 저울은 최대 3자리라고 가정.
   7단계에서 `CONF_CAM1 = 0.20`으로 **낮게** 잡아 오검출이 많이 들어오는데, 여기서 **상위
   3개만 남겨** 걸러낸다.
2. **x좌표로 정렬**: `"128"`을 만들려면 **왼쪽부터** 읽어야 한다. 신뢰도 순으로 붙이면
   `"812"` 같은 게 나온다.

**낮은 임계값 + 강한 후처리**의 조합이다. 7단계에서 말한 "임계값은 뒤에서 얼마나 걸러
주는가로 정한다"의 실행 지점이다.

#### 한계

- **자릿수가 3개 미만이면?** 저울이 `98`을 표시하면 숫자 2개만 검출된다 → `top_dets`는
  2개 → `"98"` → 정상 동작. ✅
- **오검출이 진짜 숫자보다 신뢰도가 높으면?** 잘못된 값이 나온다. 투표 버퍼 같은
  시간축 안정화가 **없다** — 라즈베리파이 편의 7프레임 투표(12단계)에 해당하는 장치가
  젯슨에는 없다. 대신 `COMMAND_COOLDOWN = 5.0`으로 **명령 연타만** 막는다.
