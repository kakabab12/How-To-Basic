# 5챕터 — TensorRT 변환 — `turnonnx.py`

> **젯슨 나노 객체인식 분류 + 르로봇** · 5/15챕터 · [📚 목차](README.md)

---

프로젝트 전체에서 **가장 짧지만 성능에 가장 큰 영향을 준 파일**이다.

```python
from ultralytics import YOLO

# 1. ONNX 모델 불러오기
model = YOLO("box.onnx")

# 2. TensorRT(.engine)로 변환
# half=True: 젯슨 GPU의 FP16 가속을 써서 속도를 2배 높입니다.
model.export(format="engine", device=0, half=True)


# from ultralytics import YOLO
# # 1. 학습한 모델 로드 (파일명 확인!)
# model = YOLO('box.pt')
# # 2. ONNX 포맷으로 변환
# # simplify=True 옵션을 주면 젯슨에서 TensorRT로 바꿀 때 훨씬 안정적입니다.
# success = model.export(format='onnx', simplify=True)
```

**주석 처리된 아래쪽이 1단계(`.pt`→`.onnx`), 활성 코드가 2단계(`.onnx`→`.engine`)**다.
두 단계를 번갈아 쓰느라 한쪽을 주석 처리해 둔 것이다.

#### `half=True` — FP16이 뭘 하나

기본 부동소수점은 **FP32**(32비트)다. `half=True`는 **FP16**(16비트)으로 바꾼다.

| 효과 | 이유 |
|---|---|
| **속도 ~2배** | 젯슨 GPU에 FP16 전용 연산 유닛이 있다 |
| **메모리 절반** | 가중치가 절반 크기 |
| **정확도 거의 그대로** | 객체 검출은 소수점 정밀도에 둔감하다 |

**젯슨처럼 자원이 빠듯한 엣지 보드에서는 사실상 필수**다. 본인 설명도 이걸 "핵심 최적화
코드"라고 부른다.

#### `simplify=True`

ONNX 그래프에서 불필요한 연산(상수 폴딩 가능한 것들)을 미리 접어 둔다.
주석대로 **"젯슨에서 TensorRT로 바꿀 때 훨씬 안정적"**이다 — TensorRT가 해석하지 못하는
복잡한 그래프 패턴을 미리 없애기 때문이다.
