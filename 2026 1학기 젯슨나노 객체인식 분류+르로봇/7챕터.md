`jetson_USB2.py` — 설정이 전부 파일 상단에 모여 있다. **좋은 습관**이다.

```python
INDEX_HTML     = "/home/user/project/index.html"
MODEL_CAM1     = "/home/user/project/number.engine"     # TensorRT (빠름)
MODEL_CAM2     = "/home/user/project/box.pt"            # PyTorch  (느림)

DIGIT_CLASS_MAP = {2:"0", 3:"1", ..., 11:"9"}
BOX_LABEL_MAP   = {0:"small_box", 1:"samll_box"}        # ⚠ 6단계 참고

CAM2_INDEX     = 6                     # USB 웹캠 주소
ARDUINO_PORT   = "/dev/ttyACM0"

CLAUDE_API_KEY = "YOUR_CLAUDE_API_KEY_HERE"             # ⚠ 플레이스홀더
CLAUDE_MODEL   = "claude-sonnet-4-20250514"

STREAM_FPS     = 15
DEVICE         = 'cuda'

CONF_CAM1      = 0.20                  # 숫자는 낮게
CONF_CAM2      = 0.60                  # 박스는 높게

WEIGHT_THRESHOLD = 98                  # 이 무게 초과 시 동작
COMMAND_COOLDOWN = 5.0
MAX_DIGITS       = 3
SCREEN_PADDING   = 15
```

#### 읽어낼 설계 판단

**① 두 카메라의 신뢰도 임계값이 다르다**

```python
CONF_CAM1 = 0.20   # 숫자
CONF_CAM2 = 0.60   # 박스
```

숫자는 **작고 흐릿해서** 모델이 자신 없어 한다 → 낮게 잡아 놓친 걸 줄인다.
대신 오검출이 늘지만, 11단계의 **top-3 + 조합** 로직이 걸러 준다.
박스는 **크고 명확**하니 높게 잡아 오검출을 막는다.

> **임계값은 "모델 성능"이 아니라 "그 클래스가 얼마나 잡기 쉬운가 + 뒤에서 얼마나 걸러
> 주는가"로 정한다.** 이 코드가 그걸 보여 준다.

**② 모델 포맷이 서로 다르다**

```python
MODEL_CAM1 = ".../number.engine"    # TensorRT
MODEL_CAM2 = ".../box.pt"           # PyTorch
```

CAM1(숫자)만 TensorRT로 가속했다. **숫자 쪽이 2단계 추론(10단계)이라 프레임당 추론을
두 번** 하기 때문에 가속이 절실했던 것으로 보인다. 박스는 한 번이라 `.pt`로 버텼다.
(`box.engine`이 있는데도 `.pt`를 쓰는 건, 엔진 재생성이 필요했거나 검증 중이었을 수 있다.)

**③ `/home/user/project/`로 하드코딩** — 그 젯슨 보드의 실제 경로다. 15단계 참고.
