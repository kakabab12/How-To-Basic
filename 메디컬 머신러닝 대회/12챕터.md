# 12챕터 — 서버 — 로딩과 방어적 설계

---

```python
app = Flask(__name__)
CORS(app)                     # 프론트가 다른 포트에서 떠도 호출 가능하게

model_cls = joblib.load('model_cls.pkl')      # ← 없으면 그냥 죽는다 (필수)
model_reg = joblib.load('model_reg.pkl')
encoders  = joblib.load('encoders.pkl')
medians   = joblib.load('medians.pkl')
features  = joblib.load('features.pkl')

try:
    resid_std = float(joblib.load('resid_std.pkl'))
except:
    resid_std = 45.0                          # ← 없어도 기본값으로 계속

try:
    pcs_lookup = joblib.load('icd10_pcs_lookup.pkl')
    print(f"✅ 수술 코드 로드: {len(pcs_lookup)}개")
except Exception as e:
    pcs_lookup = None                         # ← 없어도 계속 (기능만 비활성)
    print(f"⚠ icd10_pcs_lookup.pkl 없음: {e}")

try:
    lab_feature_map = joblib.load('lab_feature_map.pkl')
except:
    lab_feature_map = {}
    print("⚠ lab_feature_map.pkl 없음")
```

#### 필수와 선택을 구분했다

| 구분 | 파일 | 처리 |
|---|---|---|
| **필수** | `model_cls`, `model_reg`, `encoders`, `medians`, `features` | `try` 없음 → **없으면 즉시 죽는다** |
| **선택** | `resid_std`, `pcs_lookup`, `lab_feature_map` | `try/except` → **경고만 내고 계속** |

**좋은 설계다.** 모델 없이 뜬 서버는 위험하다(조용히 틀린 답을 낼 바에야 죽는 게 낫다).
반면 수술코드 조회표가 없다고 서버 전체가 못 뜰 이유는 없다 — 드롭다운만 비면 된다.

**✅/⚠ 이모지로 로딩 상태를 출력**하는 것도 실용적이다. 서버를 띄우자마자 뭐가 빠졌는지
한눈에 보인다.

#### 라우트

```python
@app.route('/')            # index.html
@app.route('/surgeries')   # 수술코드 드롭다운 (진료과별 필터링)
@app.route('/predict', methods=['POST'])   # ★ 본편
```
