```python
diag = pd.read_csv('diagnosis.csv')

# 환자별 진단 수
diag_count = diag.groupby('subject_id')['icd10_cm'].count().reset_index()
diag_count.columns = ['subject_id', 'diag_count']

# 주요 질환군 (ICD-10 첫 글자 기반)
diag['icd_group'] = diag['icd10_cm'].apply(
    lambda c: str(c)[0].upper() if pd.notna(c) and len(str(c)) > 0 else 'Z'
)

disease_map = {
    'diag_circulatory':     ['I'],
    'diag_neoplasm':        ['C', 'D'],
    'diag_endocrine':       ['E'],
    'diag_respiratory':     ['J'],
    ...
}
```

#### ICD-10 첫 글자의 의미

ICD-10 코드는 **첫 글자가 질환 대분류**다. 이건 임의의 규칙이 아니라 **국제 표준**이다:

| 첫 글자 | 질환군 | 코드 변수 |
|---|---|---|
| **I** | 순환기 (심장·혈관) | `diag_circulatory` |
| **C, D** | 신생물 (암) | `diag_neoplasm` |
| **E** | 내분비 (당뇨 등) | `diag_endocrine` |
| **J** | 호흡기 | `diag_respiratory` |
| **N** | 비뇨생식기 | `diag_genitourinary` |
| **G** | 신경계 | `diag_nervous` |

#### 왜 첫 글자만 쓰나 — 차원 축소

ICD-10 전체 코드는 **수만 개**다. 그대로 원-핫 인코딩하면 피처가 수만 개가 되어
학습이 불가능하다(차원의 저주).

**첫 글자로 묶으면 6~20개**로 줄면서도 **임상적 의미는 보존**된다. `I21`(심근경색)과
`I50`(심부전)은 다른 병이지만 **둘 다 순환기 질환이고 수술 위험을 높인다**는 점에서 같다.

**`'Z'` 폴백**도 의도적이다 — ICD-10에서 Z는 "특별한 목적의 코드"라 결측을 넣어도
의미가 왜곡되지 않는다.

**`diag_count`(진단 개수)**는 단순하지만 강력한 피처다 — **동반질환이 많을수록 위험**하다는
임상 상식을 숫자 하나로 표현한다.

> 주석에 근거도 적어 뒀다: `환자별 진단 수 (수술과 무관하게 전체 - 입원 진단이므로
> 수술 전으로 간주)`. **누수 여부를 항목마다 따졌다**는 뜻이다.
