`train.py`는 **7단계로 명확히 구획**되어 있다. 주석이 그대로 목차다:

```python
# ── 1. Operations 로드 & 기본 정제 ──
# ── 2. Diagnosis 로드 & 진단명 피처 ──
# ── 3. 범주형 인코딩 ──
# ── 4. Labs (혈액검사) ──
# ── 5. Ward Vitals (병동 생체징후) ──
# ── 6. 피처 최종 구성 ──
# ── 7. 모델 학습 ──
```

#### 1단계 코드

```python
ops = pd.read_csv('operations.csv')

# 수술 시간 계산 (분)
ops['surgery_duration'] = ops['opend_time'] - ops['opstart_time']

# 이상 수술 시간 제거 (5분 미만, 1440분=24시간 초과)
ops = ops[(ops['surgery_duration'] >= 5) & (ops['surgery_duration'] <= 1440)]

# ── 핵심: 환자별 첫 번째 수술만 기준점으로 사용 (미래 데이터 차단) ──
ops = ops.sort_values(['subject_id', 'opstart_time'])
ops['surgery_rank'] = ops.groupby('subject_id').cumcount() + 1  # 몇 번째 수술인지

# 각 수술의 cutoff = 해당 수술의 opstart_time
ops['cutoff_time'] = ops['opstart_time']
```

#### 왜 이게 이 프로젝트에서 가장 중요한 코드인가

**데이터 누수(data leakage)**란, 예측 시점에 알 수 없는 정보가 학습에 섞이는 것이다.

> **예시**: "수술 시간"을 예측하는데 "수술 종료 시각"을 피처로 넣으면 AUROC 0.99가 나온다.
> **그리고 실전에서는 완전히 쓸모없다** — 수술이 끝나야 알 수 있는 값이니까.

이 코드는 **`cutoff_time = opstart_time`**을 정의해 **"수술 시작 이후 데이터는 전부 차단"**을
선언한다. 이후 4·5단계(labs, vitals)가 이 기준으로 필터링한다.

`surgery_rank`(몇 번째 수술인지)도 영리하다 — **재수술 환자는 위험도가 높다**는 임상
지식을 피처로 만든 것이고, 이건 **수술 전에 알 수 있는 정보**라 누수가 아니다.

**이상치 제거 기준(5분~1440분)**도 임상적으로 타당하다. 5분 미만은 취소된 수술이거나
기록 오류, 24시간 초과는 데이터 오류일 가능성이 크다.

#### ⚠ 주의 깊게 볼 지점 — cutoff가 수술별이 아니라 환자별이다

7단계(labs)에서 이 `cutoff_time`을 **실제로는 쓰지 않는다**:

```python
op_cutoff = dict(zip(ops['opid'], ops['cutoff_time'])) if 'opid' in ops.columns else None   # ← 만들어 놓고
op_start_by_subject = dict(zip(ops['subject_id'], ops['opstart_time']))                     # ← 이걸 쓴다

for chunk in pd.read_csv('labs.csv', chunksize=500000):
    chunk['cutoff'] = chunk['subject_id'].map(op_start_by_subject)   # ← 환자 기준
    chunk = chunk[chunk['chart_time'] < chunk['cutoff']]
```

**`op_cutoff`(수술별 cutoff)를 만들어 놓고 한 번도 쓰지 않는다.** 죽은 변수다.
대신 `op_start_by_subject`(환자별)를 쓴다.

여기에 미묘한 문제가 있다. `ops`는 `opstart_time` **오름차순 정렬**되어 있으므로
`dict(zip(...))`은 **환자당 마지막(= 가장 늦은) 수술 시각**을 남긴다.

> **결과**: 수술을 3번 받은 환자의 경우, **1번째 수술의 예측에도 3번째 수술 직전까지의
> 혈액검사**가 피처로 들어간다. 1·2번 수술 이후에 측정된 검사가 "수술 전 데이터"로
> 섞이는 것이다. **수술이 1번뿐인 환자에게는 문제가 없다.**
>
> 죽은 `op_cutoff` 변수가 **원래는 수술별로 자르려 했다는 증거**다. 구현이 거기까지
> 가지 못했다. 고치려면 `subject_id`가 아니라 `opid` 단위로 병합해야 하는데, 그러면
> 환자별 피벗(`lab_pivot`) 구조를 통째로 바꿔야 해서 작업량이 크다.

`train.py`가 `첫 수술 비율`을 출력하는 걸 보면 **본인도 재수술 문제를 인지**하고 있었다:

```python
print(f"  첫 수술 비율: {(ops['surgery_rank']==1).mean()*100:.1f}%")
```
