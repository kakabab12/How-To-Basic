# 6챕터 — 학습 3/7 — 범주형 인코딩

```python
encoders = {}
cat_cols = ['sex', 'race', 'department', 'antype', 'icd10_pcs']
for col in cat_cols:
    le = LabelEncoder()
    df[col] = le.fit_transform(df[col].astype(str))
    encoders[col] = le
    print(f"  {col}: {len(le.classes_)}개 클래스")
```

**`encoders` 딕셔너리를 저장하는 게 핵심**이다. 나중에 `encoders.pkl`로 저장되고,
서버가 로드해서 **똑같은 매핑**을 쓴다.

> **왜 중요한가**: `LabelEncoder`는 알파벳 순으로 0,1,2...를 붙인다. 서버에서 새로
> `fit`하면 **입력 데이터에 따라 번호가 달라진다.** `Male`이 학습 때 1이었는데
> 서버에서 0이 되면 모델이 성별을 반대로 읽는다. **학습 때 만든 인코더를 그대로
> 재사용해야만** 한다.

`.astype(str)`은 결측(NaN)을 `'nan'` 문자열로 만들어 **하나의 카테고리로 취급**한다.
결측 자체가 정보일 수 있으니 나쁘지 않은 선택이다.

#### LightGBM은 원-핫이 필요 없다

`sex`를 0/1로, `department`를 0~20으로 바꾸면 **선형 모델은 "20번 과가 1번 과의 20배"**로
오해한다. 그래서 보통 원-핫 인코딩을 한다.

**트리 모델(LightGBM)은 다르다.** `department <= 7.5` 같은 분기를 반복하면 어떤 조합이든
분리해 낼 수 있어서 **라벨 인코딩만으로 충분**하다. 피처 수도 안 늘어난다.
