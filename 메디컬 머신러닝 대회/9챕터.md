# 9챕터 — 학습 6/7 — 피처 조립·결측·이상치

```python
base_features = [
    'age', 'sex', 'weight', 'height', 'asa', 'emop',
    'department', 'antype', 'icd10_pcs',
    'surgery_rank',   # 몇 번째 수술인지
]
disease_features = list(disease_map.keys())
features = base_features + ['diag_count'] + disease_features + lab_cols + vital_cols

# 존재하는 컬럼만
features = [f for f in features if f in df.columns]

# 결측치: 중앙값으로
df[features] = df[features].fillna(df[features].median())

# 이상치 클리핑 (1-99 퍼센타일)
for col in lab_cols + vital_cols:
    if col in df.columns:
        lo = df[col].quantile(0.01)
        hi = df[col].quantile(0.99)
        df[col] = df[col].clip(lo, hi)

medians = df[features].median().to_dict()
```

#### 세 가지 방어

**① `features = [f for f in features if f in df.columns]`**
데이터셋 버전이 바뀌어 컬럼이 없어져도 죽지 않는다.

**② 결측 → 중앙값**
평균이 아니라 **중앙값**인 이유: 의료 데이터는 **한쪽으로 길게 늘어진 분포**가 많다
(대부분 정상, 소수가 극단값). 평균은 극단값에 끌려가지만 중앙값은 안정적이다.

**③ 이상치 클리핑 (1~99 퍼센타일)**
**제거가 아니라 클리핑(자르기)**이다. 크레아티닌 999 같은 입력 오류를 99퍼센타일 값으로
**눌러 담는다**. 행을 지우면 그 환자의 다른 멀쩡한 정보까지 잃으니, **극단값만 눌러
샘플은 살린다**.

> **`base_features`가 곧 '수술 전에 알 수 있는 것' 목록**이다:
> 나이·성별·체중·키·**ASA 등급**·**응급 여부(emop)**·진료과·마취종류·수술코드·재수술 차수.
> 전부 수술 시작 전에 확정되는 값이다. 4단계의 원칙이 여기서 지켜진다.

**`medians`를 저장**하는 것도 중요하다 — 서버에서 사용자가 혈액검사를 입력 안 하면
**학습 때의 중앙값**으로 채워야 학습/서빙이 일치한다(11·12단계).
