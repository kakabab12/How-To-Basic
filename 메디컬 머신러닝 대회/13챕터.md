`clinical_calibrate()` — **이 프로젝트에서 가장 독창적인 함수**다.

#### 문제

LightGBM이 "ICU 확률 30%"라고 한다. 그런데 이 환자는 **ASA 5등급**(빈사 상태)이고
**응급 수술**이다. 의사가 보면 명백히 더 위험하다.

모델이 이걸 못 잡는 이유: 학습 데이터에 ASA 5 사례가 **몇 건 없어서** 모델이 그 패턴을
충분히 배우지 못했다. 희귀하지만 **결정적인** 요인이 과소평가된다.

#### 답 — 의료 지식을 로그 오즈로 더한다

```python
def clinical_calibrate(base_prob, d, asa_val):
    boost = 0.0

    if asa_val >= 5:   boost += 2.2        # 빈사 상태
    elif asa_val >= 4: boost += 1.1        # 생명 위협
    elif asa_val >= 3: boost += 0.35       # 중증 전신질환

    emop_val = float(d.get('emop', 0) or 0)
    if emop_val >= 1:
        boost += 0.8                       # 응급 수술
        if asa_val >= 4:
            boost += 0.5                   # ★ 응급 + 중증 = 상호작용

    if int(d.get('diag_circulatory', 0) or 0): boost += 0.5   # 순환기
    if int(d.get('diag_neoplasm',    0) or 0): boost += 0.4   # 암
    ...

    creat = d.get('lab_creatinine')        # 신장
    if creat:
        c = float(creat)
        if c > 5.0:   boost += 0.8
        elif c > 2.0: boost += 0.4
        elif c > 1.5: boost += 0.15

    alb = d.get('lab_albumin')             # 영양
    if alb:
        a = float(alb)
        if a < 2.5:   boost += 0.6
        elif a < 3.0: boost += 0.3
        elif a < 3.5: boost += 0.1
    ...
    sbp = d.get('lab_sbp')                 # 수축기 혈압
    if sbp:
        s = float(sbp)
        if s > 180:   boost += 0.35
        elif s > 160: boost += 0.15
        elif s < 90:  boost += 0.5         # ★ 저혈압이 고혈압보다 위험
    ...

    base_prob = max(0.5, min(99.5, base_prob))          # ① 클리핑
    logit = math.log(base_prob / (100 - base_prob))     # ② 확률 → 로짓
    return 100.0 / (1 + math.exp(-(logit + boost)))     # ③ 보정 후 다시 확률
```

#### 마지막 세 줄이 이 함수의 핵심이다

**확률에 그냥 더하면 안 된다.** `0.9 + 0.5 = 1.4` → 140% 확률은 말이 안 된다.

**로짓(로그 오즈) 공간으로 옮겨서 더한다:**

```
로짓 = ln( p / (1-p) )        확률 0~1  →  로짓 -∞~+∞
시그모이드 = 1/(1+e^-x)        로짓 -∞~+∞ →  확률 0~1
```

로짓 공간에서 **덧셈은 오즈의 곱셈**이다:

| boost | 오즈 배율 | 의미 |
|---|---|---|
| +0.35 | ×1.4 | ASA 3 |
| +0.8 | ×2.2 | 응급 수술 |
| +1.1 | ×3.0 | ASA 4 |
| **+2.2** | **×9.0** | **ASA 5** |

**이건 로지스틱 회귀의 계수와 정확히 같은 의미**다. 즉 `boost` 값들은 **의료 지식으로
직접 써 넣은 회귀 계수**인 셈이다.

> **그리고 확률이 절대 0~100%를 벗어나지 않는다.** 시그모이드가 자동으로 보장한다.
> `max(0.5, min(99.5, base_prob))`는 `log(0)` = -∞을 막는 방어다.

#### 임상 지식이 코드에 박혀 있다

| 코드 | 임상적 의미 |
|---|---|
| `if s < 90: boost += 0.5` vs `if s > 180: boost += 0.35` | **저혈압(쇼크)이 고혈압보다 급성 위험** |
| `if w > 15 or w < 2: boost += 0.3` | 백혈구는 **너무 많아도(감염) 너무 적어도(면역저하)** 위험 |
| `if a < 2.5: boost += 0.6` | 알부민 = **영양 상태**. 낮으면 회복력 저하 |
| `emop >= 1 and asa >= 4: boost += 0.5` | **상호작용 항** — 응급 + 중증은 단순 합보다 위험 |

**`w > 15 or w < 2`(U자형)는 트리 모델이 배우기 어려운 패턴**이다. 양극단이 같은 방향으로
위험한 관계라 데이터가 많아야 잡힌다. 그걸 **규칙으로 직접 주입**했다.

> **이 접근의 이름**: 통상 **하이브리드 모델** 또는 **지식 주입(knowledge injection)**이라 한다.
> 데이터가 부족한 희귀 케이스를 도메인 지식으로 메우는 정석적 방법이다.
> 의료처럼 **틀리면 사람이 다치는 분야**에서 특히 정당화된다.
>
> **단, 이 방식은 검증이 필요하다.** `boost` 값들은 실측이 아니라 **문헌·경험 기반 추정**이다.
> 실제 임상 배포 전에는 보정 전후 성능을 비교해야 한다.

`get_asa()`도 실용적이다 — 사용자가 `"ASA 3"`, `"3등급"`, `"3"` 뭘 넣든 숫자를 뽑는다:

```python
def get_asa(d):
    raw = str(d.get('asa', '') or '')
    m = re.search(r'\d+', raw)
    return float(m.group()) if m else float(medians.get('asa', 2))   # ← 없으면 학습 중앙값
```
