**INSPIRE** = *a publicly available research dataset for perioperative medicine* (v1.3).
서울대병원이 PhysioNet에 공개한 **수술 전후 의료 데이터셋**이다.

#### ⚠ CSV가 저장소에 하나도 없다

저장소 루트 `.gitignore`에 **`*.csv`**가 있기 때문이다:

```
yolov8_cls_env/
venv/
__pycache__/
*.pyc
.DS_Store
.venv/
*.csv      ← 이것 때문에 원본 데이터가 통째로 빠졌다
*.zip
*.sh
*.dll
```

**의료 데이터라 일부러 뺀 것으로 보인다** — 환자 데이터를 공개 저장소에 올리면 안 된다.
올바른 판단이다.

#### 그런데 무엇이 있었는지는 알 수 있다

`inspire-a-publicly-available-research-dataset-for-perioperative-medicine-1.3/SHA256SUMS.txt`에
**원본 파일 11개의 해시**가 남아 있다:

| 파일 | 내용 |
|---|---|
| `operations.csv` | **수술 기록** (핵심 테이블) |
| `diagnosis.csv` | 진단 (ICD-10) |
| `labs.csv` | **혈액검사** (대용량) |
| `ward_vitals.csv` | **병동 생체징후** (대용량) |
| `vitals.csv` | 수술 중 생체징후 |
| `medications.csv` | 투약 |
| `department.csv` | 진료과 |
| `parameters.csv`, `schema.csv`, `icd10_excluded.csv` | 메타데이터 |

**`train.py`가 실제로 쓰는 건 4개**다:

```python
ops   = pd.read_csv('operations.csv')
diag  = pd.read_csv('diagnosis.csv')
for chunk in pd.read_csv('labs.csv', chunksize=500000):
for chunk in pd.read_csv('ward_vitals.csv', chunksize=500000):
```

**`vitals.csv`(수술 중 생체징후)를 안 쓴 게 중요하다** — 수술 중 데이터는 예측 시점(수술
시작 직전)에 존재하지 않는다. 4단계의 데이터 누수 차단 원칙과 일관된다.

> **구하는 법**: PhysioNet에서 INSPIRE 1.3을 신청·다운로드해 `AI/` 폴더에 CSV를 넣는다.
> (자격 인증이 필요한 데이터셋이다.)
