#### `test.py` — SHAP 변수 중요도

본인 설명:

> 분류 모델과 회귀 모델 각각에 대해 어떤 요인(나이, 심박수, 혈압 등)이 예측에 가장 큰
> 영향을 미쳤는지 Summary Plot 이미지로 뽑아냅니다.

결과물이 저장소에 있다:

| 이미지 | 내용 |
|---|---|
| `shap_classification.png` | ICU 위험도에 영향을 준 변수 순위 |
| `shap_regression.png` | 수술 시간에 영향을 준 변수 순위 |
| `regression_performance.png` | 회귀 성능 |

**SHAP**(SHapley Additive exPlanations)은 게임이론의 섀플리 값을 ML에 적용한 것이다.
**"이 피처가 예측을 얼마나 밀었나"**를 공정하게 배분한다.

> **의료 AI에서 XAI는 선택이 아니라 필수다.** 의사는 "AI가 위험하대요"를 근거로 수술
> 계획을 바꾸지 않는다. **"왜"**가 있어야 한다.

#### `visualize_metrics.py` — 한글 폰트 방어

본인 설명:

> 운영체제(Windows, Mac)를 자동 감지하여 한글 폰트 깨짐 현상을 방지하는 디테일이
> 들어있습니다.

matplotlib은 기본 폰트에 한글이 없어 **네모(□□□)**로 나온다. 발표 자료를 만들다 겪는
고전적 문제다.

#### `visualize_regression.py` — 환자별 SHAP 폭포수

> "이 환자는 왜 수술 시간이 오래 걸릴까?"를 의료진에게 설명하기 위해 SHAP 폭포수
> (Waterfall) 차트를 생성합니다. 고령, ASA 4등급 등 특정 가상 환자의 데이터를 입력하면,
> 해당 환자의 수술 시간을 늘리거나 줄인 결정적 요인들을 시각적으로 분석해 줍니다.

**Summary Plot(전역)과 Waterfall(개별)의 차이**가 중요하다:

| | 대상 | 질문 |
|---|---|---|
| Summary Plot (`test.py`) | **모델 전체** | "일반적으로 뭐가 중요한가" |
| **Waterfall** (`visualize_regression.py`) | **환자 1명** | "**이 환자는** 왜 이런 예측인가" |

의사에게 필요한 건 후자다.

#### `index.html` — 이미지 없는 대시보드

> 별도의 이미지 파일 없이 CSS와 SVG만으로 원형 게이지(위험도)와 애니메이션 효과를
> 매우 가볍고 세련되게 구현했습니다.
>
> 단순히 '위험함/안전함'만 알려주는 것이 아니라, 최단/최장 수술 시간, 그리고 AI가 이
> 결과를 도출한 '결정적 요인(막대 바 형태)'과 '종합 임상 소견'을 텍스트로 자세히
> 브리핑해 줍니다.

`/predict` 응답 구조가 그대로 화면 구성이다:

```python
return jsonify({
    'surgery_fast':   round(surgery_fast),      # 최단 (resid_std 활용)
    'surgery_slow':   round(surgery_slow),      # 최장
    'risk_factors':   make_factors(imp_cls, p80c, p50c),   # 위험도 결정 요인
    'time_factors':   make_factors(imp_reg, p80r, p50r),   # 시간 결정 요인
    'recommendation': rec,                      # 종합 임상 소견
    'lab_count':      len(lab_inputs_used),     # 몇 개 입력했나
    'risk_level':     risk_level,               # 등급
})
```

**`lab_count`를 돌려주는 게 정직하다** — 혈액검사를 3개만 넣었으면 나머지는 중앙값으로
채워진 것이고, 그만큼 예측이 덜 개인화된다. 사용자가 그걸 알아야 한다.

`feat_to_kr()`, `fmt_val()` 같은 헬퍼가 **피처명을 한국어로, 값을 단위와 함께** 보여 준다
(`question.txt`의 2번 요구 반영).
