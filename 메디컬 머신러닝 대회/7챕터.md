```python
lab_chunks = []
for chunk in pd.read_csv('labs.csv', chunksize=500000):    # ★ 50만 행씩
    chunk['cutoff'] = chunk['subject_id'].map(op_start_by_subject)
    chunk = chunk[chunk['chart_time'] < chunk['cutoff']]   # ★ 수술 전만
    lab_chunks.append(chunk)

labs = pd.concat(lab_chunks, ignore_index=True)
```

#### `chunksize`가 필요한 이유

`labs.csv`는 **환자 수 × 검사 항목 수 × 측정 횟수**라 수천만 행이 된다. 통째로
`read_csv`하면 **메모리가 터진다**.

**청크 전략이 영리한 점**: 읽자마자 **곧바로 필터링**해서 필요한 것만 `lab_chunks`에
쌓는다. 수술 후 데이터는 애초에 메모리에 남기지 않는다.

```
50만 행 읽기 → 수술 전만 남기기(대부분 버림) → 리스트에 추가
        ↑                                            │
        └────────── 반복 ───────────────────────────┘
                                                     ▼
                                        전부 모아서 concat
```

#### 4가지 집계 — `last`가 왜 있나

```python
lab_pivot = labs.pivot_table(
    index='subject_id', columns='item_name',
    values='value', aggfunc=['mean', 'min', 'max', 'last']
)
lab_pivot.columns = ['lab_' + agg + '_' + col for agg, col in lab_pivot.columns]
```

주석이 설명한다: `검사항목별 mean/min/max (최근 값이 가장 중요하므로 last도 추가)`

| 집계 | 의미 |
|---|---|
| `mean` | 평소 상태 |
| `min` / `max` | **극단값** — 한 번이라도 위험 수치였나 |
| **`last`** | **수술 직전 상태** ← 임상적으로 가장 중요 |

**한 환자가 크레아티닌을 20번 쟀다면**, 평균은 만성 상태를, 최댓값은 최악의 순간을,
마지막 값은 **지금 상태**를 말한다. 셋 다 다른 정보다.

`pivot_table`은 **긴 형식(long) → 넓은 형식(wide)** 변환이다:

```
before:  subject_id | item_name  | value        after:  subject_id | lab_mean_Hb | lab_last_Hb | ...
         1          | Hb         | 12.3                 1          | 12.5        | 12.7        | ...
         1          | Hb         | 12.7
         1          | Creatinine | 1.1
```

머신러닝 모델은 **한 행 = 한 샘플**을 요구하므로 이 변환이 필수다.
