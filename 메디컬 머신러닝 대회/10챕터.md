```python
df['icu_admission'] = df['icuin_time'].notna().astype(int)   # ★ 라벨 정의
y_cls = df['icu_admission']
print(f"  ICU 입원: {y_cls.sum():,}명 ({y_cls.mean()*100:.1f}%)")

X_train, X_test, y_train, y_test = train_test_split(
    X, y_cls, test_size=0.2, random_state=42, stratify=y_cls    # ★ stratify
)

model_cls = lgb.LGBMClassifier(
    n_estimators=2000,
    learning_rate=0.02,
    num_leaves=63,
    min_child_samples=50,
    subsample=0.8,
    colsample_bytree=0.8,
    reg_alpha=0.1,
    reg_lambda=0.1,
    class_weight='balanced',      # ★ 불균형 대응
    random_state=42,

)
model_cls.fit(..., callbacks=[lgb.early_stopping(100, verbose=False), ...])
```

**라벨을 `icuin_time`의 존재 여부로 만든 게 영리하다** — "ICU 입실 시각이 기록되어 있으면
ICU에 간 것"이다. 별도 라벨 컬럼이 없어도 만들 수 있다.

#### 하이퍼파라미터 하나하나가 의도적이다

| 파라미터 | 값 | 왜 |
|---|---|---|
| `n_estimators=2000` + `learning_rate=0.02` | 많이·천천히 | **낮은 학습률 + 많은 트리**는 부스팅의 정석. early stopping이 알아서 멈춘다 |
| `early_stopping(100)` | 100라운드 개선 없으면 정지 | 2000그루를 다 안 쓴다. **과적합 방지 + 시간 절약** |
| `num_leaves=63` | 2⁶-1 | LightGBM은 leaf-wise 성장이라 이 값이 복잡도를 결정 |
| `min_child_samples=50` | 리프당 최소 50샘플 | **과적합 방지** — 환자 3명짜리 규칙을 만들지 않는다 |
| `subsample=0.8`, `colsample_bytree=0.8` | 행·열 80% 샘플링 | 트리마다 다른 데이터를 봐서 **다양성↑** |
| `reg_alpha/lambda=0.1` | L1/L2 정규화 | 추가 과적합 방지 |
| **`class_weight='balanced'`** | 클래스 가중 | ★ 아래 |

#### `class_weight='balanced'`가 필수인 이유

ICU에 가는 환자는 **소수**다(코드가 비율을 출력한다). 예를 들어 5%라면:

> **"전부 ICU 안 감"이라고 답하는 모델의 정확도 = 95%**

정확도만 보면 훌륭하지만 **완전히 쓸모없다**. `class_weight='balanced'`는 소수 클래스에
**가중치를 크게** 줘서 모델이 이 함정에 빠지지 않게 한다.

같은 이유로 `stratify=y_cls`를 쓴다 — train/test에 **ICU 비율을 동일하게** 유지한다.
그리고 평가 지표로 **AUROC**를 쓴다(정확도가 아니라).

#### 회귀 — 수술 시간 예측

```python
model_reg = lgb.LGBMRegressor(
    n_estimators=2000, learning_rate=0.02, num_leaves=63,
    min_child_samples=50, subsample=0.8, colsample_bytree=0.8,
    reg_alpha=0.1, reg_lambda=0.1, random_state=42, verbose=-1,
)
```

**분류와 하이퍼파라미터가 동일**하다. `class_weight`만 빠졌다(회귀엔 클래스가 없으니).
`stratify`도 없다(연속값이라 불가).

> **같은 피처로 두 문제를 푼다.** 피처 엔지니어링(4~9단계)의 결과물을 **두 번 재사용**하는
> 구조라 효율적이다.
