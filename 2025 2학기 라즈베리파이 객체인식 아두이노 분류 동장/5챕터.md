`RPI-Flask-main/best.onnx`(47MB)의 **ONNX 메타데이터를 직접 열어 본 결과**다:

```
task  : segment
imgsz : [640, 640]
names : {0: 'big_abnormal_box',    1: 'big_normal_box',
         2: 'medium_abnormal_box', 3: 'medium_normal_box',
         4: 'small_abnormal_box',  5: 'small_normal_box'}
```

`sub/1000장/best.onnx`, `sub/1500장/best.onnx`도 **같은 6클래스 세그멘테이션 모델**이다
(파일 내용은 서로 다름 = 다른 학습 결과).

#### 여기서 이 프로젝트의 가장 중요한 질문이 나온다

**모델이 이미 `big_normal_box`라고 6종으로 말해 주는데, 왜 코드가 다시 계산하나?**

```python
# sucess.py — 모델이 준 이름을 받아서...
original_class_name = model.names[cls_id]     # 예: "big_normal_box"

# ...크기는 면적으로 다시 판정하고
size_prefix = "medium"
if poly_area < AREA_LIMIT_SMALL: size_prefix = "small"
elif poly_area > AREA_LIMIT_BIG: size_prefix = "big"

# ...불량은 R값으로 다시 판정한다
is_abnormal = False
if "abnormal" in original_class_name:              # ← 모델 의견은 '참고'만
    is_abnormal = True
if rectangularity < RECTANGULARITY_THRESHOLD:      # ← 기하 규칙이 '덮어쓴다'
    is_abnormal = True

final_class_name = f"{size_prefix}_{state_suffix}"  # 재조합
```

**답**: 딥러닝 모델은 "이 박스가 큰지"를 **학습한 예시로부터 추측**한다. 카메라 각도가
바뀌거나 조명이 달라지면 틀린다. 반면 **마스크 면적(픽셀 수)은 물리적 측정값**이라
카메라가 고정된 공정 라인에서는 훨씬 믿을 만하다.

**즉 딥러닝은 "박스가 여기 있다(마스크)"까지만 시키고, 판정은 기하학에 맡겼다.**
세그멘테이션 모델을 쓴 이유가 바로 이것이다 — **detect(사각형)로는 면적도 R값도 못 구한다.**

주목할 점: `"abnormal" in original_class_name`은 **불량을 추가하기만 하고 취소하지 않는다.**
모델이 불량이라 하면 불량, R값이 낮아도 불량 — **둘 중 하나만 걸려도 불량**이다.
안전한 쪽(불량으로 빼는 쪽)으로 실패하게 만든 설계다.
