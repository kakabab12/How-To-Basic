# 3챕터 — ⚠ 데이터셋이 두 종류다 (가장 헷갈리는 부분)

> **라즈베리파이 객체인식 + 아두이노 분류** · 3/15챕터 · [📚 목차](README.md)

---

이 폴더를 처음 보면 **"견과류 데이터셋이 있으니 견과류를 분류하는구나"**라고 오해하기 쉽다.
**아니다.** 실제 배포된 앱은 **박스**를 분류한다.

| | 견과류 (`Classfication_fruit.v1i.yolov8/`) | 박스 (실제 배포) |
|---|---|---|
| 클래스 | `nuts-bad0.25`, `nuts-bad0.5`, `nuts-good` (3종) | `big/medium/small` × `normal/abnormal` (6종) |
| task | **detect** (박스만) | **segment** (마스크까지) |
| 모델 크기 | 12 MB | **47 MB** |
| 데이터셋 | ✅ 저장소에 있음 | ❌ **저장소에 없음** |
| 학습 스크립트 | ✅ `train.py` | ❌ 없음 |
| 결과 모델 | `runs/.../weights/best.onnx` | `RPI-Flask-main/best.onnx` |

**즉 저장소에 있는 데이터셋(견과류)은 실제 앱이 쓰는 모델(박스)과 무관하다.**
견과류 쪽은 YOLO 학습을 연습한 **곁가지**이고, 박스 세그멘테이션 모델은 **결과물(.onnx)만
있고 학습 재료가 없다.**

#### 함정 — `RPI-Flask-main/dataset.yaml`을 믿지 말 것

```yaml
train: ./dataset/images/train
val: ./dataset/images/val
nc: 1
names: ['box']
```

**이 파일은 실제 모델과 맞지 않는다.** `nc: 1, names: ['box']`라고 되어 있지만
같은 폴더의 `best.onnx`는 **6클래스 세그멘테이션 모델**이다(5단계에서 증명).
가리키는 `./dataset/` 폴더도 저장소에 없다. **낡은 잔재 파일**이다.
