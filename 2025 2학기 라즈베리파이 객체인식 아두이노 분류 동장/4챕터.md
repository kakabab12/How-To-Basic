# 4챕터 — 학습 스크립트 읽기

> **라즈베리파이 객체인식 + 아두이노 분류** · 4/15챕터 · [📚 목차](README.md)

---

`Classfication_fruit.v1i.yolov8/train.py` — **저장소에 남은 유일한 학습 코드**다.

```python
DATA_YAML = r"./data.yaml"
PROJECT = "runs_detect"
NAME = "nuts_detect_train"
EPOCHS = 50
IMG_SIZE = 640
BATCH = 16

def main():
    model = YOLO("yolov8n.pt")           # ① 사전학습 가중치에서 출발
    model.train(data=DATA_YAML, epochs=EPOCHS, imgsz=IMG_SIZE, batch=BATCH,
                project=PROJECT, name=NAME, device=0, pretrained=True)

    best_path = os.path.join("runs", "detect", PROJECT, NAME, "weights", "best.pt")
    best_model = YOLO(best_path)

    metrics = best_model.val()            # ② 검증

    export_path = best_model.export(      # ③ ONNX 변환
        format="onnx", imgsz=IMG_SIZE, opset=12, simplify=True
    )
```

**학습 → 검증 → 배포 포맷 변환을 한 스크립트에서** 끝낸다. 배포 대상(라즈베리파이)이
PyTorch를 무겁게 돌릴 수 없으니 ONNX가 필수라, 학습 파이프라인의 마지막 단계로 못박았다.

| 인자 | 값 | 왜 |
|---|---|---|
| `yolov8n.pt` | nano | 라즈베리파이 CPU에서 돌아야 하니 가장 작은 모델 |
| `device=0` | GPU 고정 | **GPU 없는 PC에선 `device='cpu'`로 바꿔야 함** |
| `opset=12` | 낮은 opset | 구버전 ONNX Runtime 호환 |
| `simplify=True` | 그래프 단순화 | 임베디드에서 안정성↑ |
| `epochs=50` | — | nano 모델 + 소규모 데이터셋 기준 무난 |

#### 데이터셋의 출처

`data.yaml`이 전부 말해 준다:

```yaml
nc: 3
names: ['nuts-bad0.25', 'nuts-bad0.5', 'nuts-good']
roboflow:
  workspace: test-gbuqg
  project: classfication_fruit
  version: 1
  license: Public Domain
  url: https://universe.roboflow.com/test-gbuqg/classfication_fruit/dataset/1
```

**어떻게 만들었나**: 견과류를 영상 촬영 → 프레임 단위로 쪼갬(파일명이 `frame_0001_jpg.rf.<해시>.jpg`
— `frame_XXXX`가 그 증거) → Roboflow 업로드 → **손상 정도별 3단계**로 라벨링 → YOLOv8 포맷 export.

**`bad0.25` / `bad0.5`는 "25% 손상 / 50% 손상"**이다. 불량을 이진(정상/불량)이 아니라
**정도**로 나눈 게 특징이다.

#### 학습 결과는 이미 저장소에 있다

```
runs/detect/runs_detect/nuts_detect_train/
├── weights/best.pt, best.onnx, last.pt
├── confusion_matrix.png            ← 어떤 클래스를 헷갈렸나
├── BoxF1_curve.png / BoxPR_curve.png / BoxP_curve.png / BoxR_curve.png
├── results.png                     ← 학습 곡선 (loss·mAP)
├── labels.jpg                      ← 라벨 분포
└── train_batch*.jpg, val_batch*_pred.jpg   ← 실제 입력·예측 시각화
```

**다시 학습하지 않고도 성능을 볼 수 있다.** `confusion_matrix.png`와 `results.png`부터 보면 된다.
