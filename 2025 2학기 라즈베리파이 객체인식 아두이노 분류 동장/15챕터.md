#### ⚠ 함정 ① 모델 경로가 없는 폴더를 가리킨다

`sucess.py`, `app_ck.py`, `app_base.py`, `prototype.py` **네 파일 전부**:

```python
model = YOLO('yolov5/best.onnx', task='segment')
```

**그런데 저장소의 파일은 `RPI-Flask-main/best.onnx`다. `yolov5/` 폴더는 존재하지 않는다.**

그대로 실행하면 `❌ 모델 로드 실패`가 찍히고, 바로 다음 줄에서 죽는다:

```python
colors = np.random.uniform(0, 255, size=(len(model.names), 3)).astype(int)
#                                            ^^^^^^^^^^^ model이 None이라 AttributeError
```

**모델 로드는 `try/except`로 감쌌는데, 바로 다음 줄이 `model.names`를 무방비로 쓴다.**
예외 처리가 반쪽이다. 해결:

```bash
mkdir yolov5 && cp best.onnx yolov5/best.onnx     # 방법 1: 경로를 맞춘다
# 방법 2: 코드에서 YOLO('best.onnx', task='segment')로 수정
```

#### ⚠ 함정 ② 라즈베리파이가 아니면 카메라가 안 뜬다

`camera.py`는 `from picamera2 import Picamera2`로 시작한다. 일반 PC에는 없다.
PC에서 테스트하려면 `FastAPI_GPU버전-수정본/`을 쓰는 게 낫다 — 거기엔 **더미 카메라 폴백**이 있다:

```python
try:
    from camera import Camera
    cam = Camera()
except Exception as e:
    print(f"⚠️ 카메라 로드 실패({e}). 더미 카메라 생성.")
    class DummyCamera:
        def get_frame(self): return np.zeros((480, 640, 3), dtype=np.uint8)
    cam = DummyCamera()
```

#### ⚠ 함정 ③ 하드코딩된 IP

```python
NOTEBOOK_SERVER_URL = "http://172.16.11.200:5000"   # sucess.py, prototype.py
# send_test.py 는 192.168.0.8
```

그 네트워크가 아니면 매초 타임아웃이 난다(무해하지만 로그가 지저분해진다).

#### ⚠ 함정 ④ 튜닝 상수는 그 카메라 전용

`AREA_LIMIT_SMALL = 52100`, `AREA_LIMIT_BIG = 260000`은 **픽셀 면적**이라
카메라 높이·해상도가 바뀌면 전부 다시 재야 한다. `RECTANGULARITY_THRESHOLD = 0.88`은
비율이라 상대적으로 안전하다.

#### 세 버전 비교

| | `RPI-Flask-main/` | `FastAPI_GPU버전-수정본/` | `sub/` |
|---|---|---|---|
| 프레임워크 | Flask | **FastAPI** | Flask |
| 실행 | `py sucess.py` | `py app_ck.py` | `py app.py` |
| 포트 | 5000 | 5000 | 5000 |
| 카메라 | Picamera2 필수 | **더미 폴백 있음** | Picamera2 |
| 추론 | ONNX CPU | GPU | — (**가짜 데이터**) |
| 특이점 | 최종 완성본 | **Gemini 챗봇** (`google.genai`) | `simulate_robot_data()` |

세 폴더 모두 `port=5000`이라 **동시에 못 띄운다.**
