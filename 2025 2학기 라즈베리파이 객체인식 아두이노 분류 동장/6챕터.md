#### 실행 시작점

```python
if __name__ == '__main__':
    threading.Thread(target=run_yolo_detection, daemon=True).start()   # ① 추론
    threading.Thread(target=read_from_arduino,  daemon=True).start()   # ② 아두이노
    threading.Thread(target=update_system_data, daemon=True).start()   # ③ 시스템 상태
    app.run(host='0.0.0.0', port=5000, debug=True, use_reloader=False)
```

**스레드 3개 + 메인(Flask)** 구조다.

| 스레드 | 하는 일 | 주기 |
|---|---|---|
| `run_yolo_detection` | 프레임 읽고 추론·판정·아두이노 전송 | 계속 (`sleep(0.01)`) |
| `read_from_arduino` | 시리얼 연결 유지·수신 | 1초 |
| `update_system_data` | CPU/메모리/FPS 수집 + 노트북 서버로 POST | 1초 |
| Flask (메인) | 웹 요청 응답 | 요청 시 |

> **`use_reloader=False`가 중요하다.** Flask의 debug 모드는 코드가 바뀌면 프로세스를
> 재시작하는데, 그러면 **스레드 3개가 두 번 뜬다**(카메라·시리얼 중복 점유 → 충돌).
> 그래서 `debug=True`를 쓰면서도 리로더만 끈 것이다.

#### 공유 상태와 락

```python
current_stats     = { "cpu": 0, "memory": 0, "fps": 0 }
detection_log     = deque(maxlen=20)
action_log        = deque(maxlen=20)
system_log        = deque(maxlen=20)
class_vote_buffer = deque(maxlen=VOTE_BUFFER_SIZE)
latest_results    = []
locked_target     = None

data_lock = threading.Lock()
```

- **`deque(maxlen=20)`** — 로그가 무한히 쌓여 메모리를 먹는 걸 막는다. 오래된 건 자동으로 밀려난다.
  라즈베리파이는 RAM이 적으니 필수다.
- **`data_lock` 하나로 모든 공유 상태를 보호**한다. 락을 여러 개 쓰면 교착(deadlock) 위험이
  생기니, 단순하게 하나로 통일했다.
