```python
def run_yolo_detection():
    global latest_results, locked_target
    print(f"🚀 YOLO 시작: R값 {RECTANGULARITY_THRESHOLD} 미만은 무조건 불량 처리합니다.")

    while True:
        if model is None or cam is None:
            time.sleep(1); continue

        if locked_target is not None:      # ★ 락 걸리면 분석 일시 중지
            time.sleep(0.05); continue

        frame = cam.get_frame()
        if frame is None:
            time.sleep(0.01); continue

        # 속도 최적화: imgsz=640 (느리면 320으로 변경)
        results = model(frame, imgsz=640, conf=0.5, iou=0.4,
                        agnostic_nms=True, verbose=False)
        ...
        time.sleep(0.01)
```

세부 포인트:

- **`locked_target`이 있으면 아예 추론을 건너뛴다.** 타겟이 확정되어 로봇이 집는 중에는
  CPU를 쓸 이유가 없다. **라즈베리파이 CPU 절약 장치**다.
- **`agnostic_nms=True`** — NMS(중복 제거)를 클래스 구분 없이 한다. 같은 박스가
  `big_normal_box`와 `big_abnormal_box` 양쪽으로 검출되는 걸 막는다. **어차피 클래스는
  다시 계산할 거니**(5단계) 겹치면 하나만 남기는 게 맞다.
- **`verbose=False`** — ultralytics가 프레임마다 찍는 로그를 끈다. 라즈베리파이 콘솔이
  마비되는 걸 막는다.
- **모델 로드는 `task='segment'` 명시**:
  ```python
  model = YOLO('yolov5/best.onnx', task='segment')
  ```
  ONNX는 `.pt`와 달리 task 정보를 ultralytics가 자동으로 못 읽는 경우가 있어 직접 알려준다.
  (**`yolov5/` 경로는 함정이다 — 15단계 참고**)
