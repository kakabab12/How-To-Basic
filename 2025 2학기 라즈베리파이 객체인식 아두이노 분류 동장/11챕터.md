# 11챕터 — 무게중심과 좌표 변환

> **라즈베리파이 객체인식 + 아두이노 분류** · 11/15챕터 · [📚 목차](README.md)

```python
# 🔥 무게 중심(Centroid) 정밀 계산
M = cv2.moments(points)
if M['m00'] != 0:
    cx = int(M['m10'] / M['m00'])
    cy = int(M['m01'] / M['m00'])
else:
    cx = int((x1 + x2) / 2)      # 폴백: 박스 중심
    cy = int((y1 + y2) / 2)

cz = PICKUP_Z_LEVEL              # 0

# 화면 중앙과의 거리 계산
dist = ((cx - 320)**2 + (cy - 480)**2)**0.5

# 좌표 계산
real_x = round((cx - 320) * 0.05, 1)
real_y = round((480 - cy) * 0.05 + 5.0, 1)
```

#### 왜 박스 중심이 아니라 무게중심인가

바운딩 박스의 중심 `((x1+x2)/2, (y1+y2)/2)`는 **모양을 무시**한다. L자로 찌그러진 물체는
박스 중심이 물체 밖(빈 공간)에 있을 수 있다. **로봇이 거기를 집으면 허공을 집는다.**

`cv2.moments`의 **1차 모멘트**로 구한 무게중심은 **마스크 픽셀들의 평균 위치**라
항상 물체 안에 있다. `M['m00']`(0차 모멘트)이 곧 면적이고, 0이면 나눌 수 없으니
박스 중심으로 폴백한다.

#### 좌표 변환 — 픽셀에서 현실로

```python
real_x = round((cx - 320) * 0.05, 1)        # 픽셀 → cm
real_y = round((480 - cy) * 0.05 + 5.0, 1)
```

- **`- 320`**: 화면 중앙(640 폭 기준)을 원점으로 옮긴다
- **`* 0.05`**: 1픽셀 = 0.05cm 라는 **실측 스케일**
- **`480 - cy`**: y축을 뒤집는다. 화면은 아래로 갈수록 y가 커지지만, 로봇 좌표계는
  **위로 갈수록 커진다**
- **`+ 5.0`**: 카메라와 로봇 원점의 오프셋

> **`dist = ((cx - 320)**2 + (cy - 480)**2)**0.5`를 눈여겨볼 것.**
> `(320, 480)`은 **화면 중앙이 아니다.** 640×480 프레임이면 중앙은 `(320, 240)`이고,
> `(320, 480)`은 **화면 아래쪽 정중앙**이다. 즉 "가장 가까운 객체" 선정 기준이
> **화면 하단에 가장 가까운 것**이다 — 컨베이어에서 로봇팔 쪽에 제일 가까운 박스를
> 먼저 집으려는 의도로 보인다. (주석에는 "화면 중앙과의 거리"라고 적혀 있어 **주석이 부정확**하다.)
