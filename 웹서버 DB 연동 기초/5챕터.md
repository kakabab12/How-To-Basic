# 5챕터 — 웹 서버 종류 ③ 실서비스는 둘을 겹쳐 쓴다 (WSGI/ASGI + Nginx)

> **웹 서버 + DB 연동 기초** · 5/15챕터 · [📚 목차](README.md)

---

#### `app.run()`은 개발용이다

```python
app.run(host="127.0.0.1", port=5000, debug=True)
```

**Flask를 실행하면 이런 경고가 뜬다:**

```
WARNING: This is a development server. Do not use it in a production deployment.
```

**왜 안 되나:**

| 문제 | 설명 |
|---|---|
| **한 번에 한 요청** | 기본 설정이 단일 스레드. 동시 접속에 취약 |
| **느림** | 성능 최적화가 안 되어 있다 |
| **보안** | `debug=True`면 **에러 화면으로 소스가 노출**되고, 디버거로 **원격 코드 실행**까지 가능 |
| **정적 파일이 느림** | CSS/이미지를 파이썬이 읽어서 보낸다 |

#### 실제 배포 구조 — 3층

```
   인터넷
     │
     ▼  :80 / :443
 ┌────────────────────────┐
 │  Nginx  (정적 웹 서버)   │   ← 3단계
 │                        │
 │  • CSS/이미지 = 직접 응답 (빠름)
 │  • HTTPS 처리 (SSL 종료)
 │  • 동시접속 1만 감당
 │  • 나머지는 뒤로 넘김 ────┼──┐
 └────────────────────────┘  │
                             ▼  :8000
                     ┌───────────────────┐
                     │ gunicorn/uvicorn  │  ← WSGI/ASGI 서버
                     │  (프로세스 여러 개) │
                     └─────────┬─────────┘
                               ▼
                     ┌───────────────────┐
                     │  Flask / FastAPI  │  ← 4단계, 우리 코드
                     └─────────┬─────────┘
                               ▼
                     ┌───────────────────┐
                     │   DB              │
                     └───────────────────┘
```

**각 층이 잘하는 일만 한다:**

| 층 | 잘하는 것 |
|---|---|
| **Nginx** | 파일 배달, HTTPS, 동시접속 |
| **gunicorn/uvicorn** | 파이썬 프로세스 여러 개 관리 |
| **Flask/FastAPI** | 우리 로직 |

#### WSGI와 ASGI가 뭔가

**"파이썬 웹 앱과 서버가 대화하는 규격"**이다.

| | **WSGI** | **ASGI** |
|---|---|---|
| 뜻 | Web Server Gateway Interface | Asynchronous ~ |
| 시기 | 2003 | 2018 |
| 방식 | **동기** (한 번에 하나) | **비동기** (동시에 여러 개) |
| 서버 | **gunicorn**, uWSGI | **uvicorn**, hypercorn |
| 프레임워크 | Flask, Django | **FastAPI**, Starlette |

**규격이 있어서 조합이 자유롭다:**

```
Flask   + gunicorn   ✅
Flask   + uWSGI      ✅
FastAPI + uvicorn    ✅
FastAPI + gunicorn   ✅ (uvicorn 워커로)
```

> **[젯슨 프로젝트](../2026%201학기%20젯슨나노%20객체인식%20분류+르로봇/README.md)의
> `uvicorn.run(app, host="0.0.0.0", port=5000)`이 ASGI 서버**다.
> FastAPI는 앱일 뿐이고 실제로 포트를 여는 건 uvicorn이다.

#### 실배포 명령

```bash
# Flask (WSGI)
gunicorn -w 4 -b 127.0.0.1:8000 app:app
#         ↑ 워커 프로세스 4개 (CPU 코어 수 × 2 + 1 이 관례)

# FastAPI (ASGI)
uvicorn main:app --host 127.0.0.1 --port 8000 --workers 4
```

**Windows에서는 gunicorn이 안 된다**(유닉스 전용). `waitress`를 쓴다:

```bash
pip install waitress
waitress-serve --port=8000 app:app
```

> **이 실습은 배포까지 가지 않는다.** `app.run()`으로 충분하다 —
> **혼자 localhost에서 배우는 게 목적**이니까. 다만 **실서비스에 그대로 올리면
> 안 된다는 것**은 알고 있어야 한다.
