# 3챕터 — 웹 서버 종류 ① 정적 웹 서버 (Apache · Nginx · IIS)

> **웹 서버 + DB 연동 기초** · 3/15챕터 · [📚 목차](README.md)

---

#### "웹 서버"라는 말이 두 가지를 뜻한다

**이게 초심자가 가장 헷갈리는 지점**이다.

| 뜻 | 예 | 하는 일 |
|---|---|---|
| **① 정적 웹 서버** | Apache, Nginx, IIS | **파일을 그대로 보낸다** |
| **② 애플리케이션 서버** | Flask, FastAPI, Express | **코드를 실행해서 응답을 만든다** |

**이 단계는 ①이다.**

#### 정적 웹 서버가 하는 일

```
요청: GET /style.css
   ↓
디스크에서 htdocs/style.css 를 찾는다
   ↓
파일 내용을 그대로 응답
```

**끝이다.** 계산도 DB 조회도 없다. **파일 배달부**다.

> **[XAMPP 웹 과제](../개인%20프로젝트/xampp%20웹%20자기소개서%20간단하게/README.md)가 정확히 이것이다.**
> `index.html`, `style1.css`를 Apache가 그냥 보내 준다. PHP도 DB도 안 썼으니
> **Apache는 파일 배달만 했다.** 사실 브라우저로 직접 열어도 똑같았던 이유다.

#### 3대 정적 웹 서버

| | **Apache** | **Nginx** | **IIS** |
|---|---|---|---|
| 등장 | 1995 | 2004 | 1995 |
| 만든 곳 | Apache 재단 | 이고르 시쇼프 (러시아) | Microsoft |
| OS | 전부 | 전부 | **Windows 전용** |
| 구조 | **프로세스/스레드** 기반 | **이벤트** 기반 | Windows 통합 |
| 강점 | 기능 많음, `.htaccess` | **동시접속에 강함**, 가벼움 | Windows·ASP.NET 통합 |
| 약점 | 무겁다 | 설정이 덜 유연 | Windows만 |
| 점유율 | 하락 중 | **1위** | 소수 |

#### Apache vs Nginx — 이 차이만 알면 된다

**동시에 1만 명이 접속하면?**

```
Apache (프로세스/스레드 모델)
   접속 1개 = 스레드 1개
   1만 접속 = 스레드 1만 개 → 메모리 폭발 💥

Nginx (이벤트 모델)
   스레드 몇 개가 1만 접속을 "돌아가며" 처리
   → 메모리 적게 씀 ✅
```

**이게 "C10K 문제"**(동시접속 1만 처리)이고, Nginx가 그걸 풀려고 만들어졌다.
**Nginx가 점유율 1위가 된 이유**다.

> **[젯슨 프로젝트](../2026%201학기%20젯슨나노%20객체인식%20분류+르로봇/README.md)의
> 4스레드 구조**를 떠올리면 이해가 쉽다. 카메라 2대에 스레드 4개면 괜찮지만,
> **접속자 1만 명에 스레드 1만 개**는 불가능하다. 그래서 이벤트 방식이 필요하다.

#### IIS를 이미 써봤다

> **[ASP 프로젝트](../개인%20프로젝트/2024년%202학기ASP_DB연동/README.md)가 IIS다.**
> `root/iisstart.htm`이 IIS 기본 페이지이고, `.asp` 파일을 IIS가 실행했다.
> 즉 **IIS는 정적 서버(①)이면서 ASP 실행기(②) 역할도 겸했다** —
> 이게 5단계의 "겹쳐 쓴다"의 원형이다.
