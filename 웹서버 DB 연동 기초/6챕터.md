#### 파일로 해보면 안 되나?

```python
# 방명록을 그냥 파일로 저장한다면
with open("guestbook.txt", "a", encoding="utf-8") as f:
    f.write(f"{name}|{message}\n")
```

**된다.** 실제로 작은 프로그램은 이렇게 한다.

> **[메디컬 프로젝트](../메디컬%20머신러닝%20대회/README.md)가 `.pkl` 파일을 쓰고,
> [젯슨](../2026%201학기%20젯슨나노%20객체인식%20분류+르로봇/README.md)이
> `.log` 파일에 기록한 게 이것이다.**

#### 그런데 곧 막힌다

| 하고 싶은 것 | 파일로 하면 |
|---|---|
| **"이지용이 쓴 글만"** | 전체를 읽어서 하나씩 비교 (O(n)) |
| **"최근 10건만"** | 전체를 읽고 정렬하고 자르기 |
| **동시에 2명이 글 작성** | ★ **한 명 글이 사라진다** (덮어쓰기 충돌) |
| **글 1개 수정** | 전체를 읽고 → 고치고 → 전체를 다시 쓰기 |
| **글 100만 개** | 전체를 메모리에 못 올린다 |
| **쓰다가 정전** | ★ **파일이 반만 써져서 깨진다** |

#### DB가 대신 해 주는 것 — 이게 DB의 정의다

| 기능 | 뜻 |
|---|---|
| **인덱스** | 100만 건에서 1건을 **즉시** 찾는다 (전부 안 읽고) |
| **동시성 제어** | 여러 명이 동시에 써도 **안 깨진다** (락) |
| **트랜잭션** | 여러 작업을 **전부 하거나 전부 안 하거나** |
| **영속성** | **정전이 나도** 커밋된 건 살아 있다 |
| **질의 언어** | "조건에 맞는 것만" 을 **선언적으로** 표현 (SQL) |

#### ACID — DB를 DB이게 하는 4가지

| 글자 | 뜻 | 예 |
|---|---|---|
| **A** | Atomicity (원자성) | 계좌이체: 출금과 입금이 **둘 다 되거나 둘 다 안 되거나** |
| **C** | Consistency (일관성) | 제약(NOT NULL 등)을 깨는 데이터는 못 들어간다 |
| **I** | Isolation (격리성) | 동시에 실행돼도 **하나씩 한 것처럼** 보인다 |
| **D** | Durability (지속성) | 커밋했으면 **정전이 나도 남는다** |

**계좌이체가 A를 설명하는 고전적 예다:**

```
출금 성공 → 💥 정전 → 입금 실패
                       = 돈이 사라진다!

트랜잭션으로 묶으면 → 둘 다 취소된다 (롤백)
```

#### 이 실습에서 볼 수 있는 것

```python
# db.py
conn.execute("INSERT INTO guestbook (name, message) VALUES (?, ?)", (name, message))
conn.commit()          # ★ 이게 있어야 진짜 저장된다 (D)
```

**`commit()` 전까지는 "임시"다.** 프로그램이 죽으면 없던 일이 된다.
파일 쓰기에는 이런 개념 자체가 없다.
