# 14챕터 — 실습 ⑤ 서버 갈아끼우기 / DB 갈아끼우기

> **웹 서버 + DB 연동 기초** · 14/15챕터 · [📚 목차](README.md)

---

**이 실습의 결론이다.** 계층을 나눈 게 실제로 값을 하는지 확인한다.

#### 실험 ① 웹 서버만 바꾼다 — `02_fastapi_sqlite`

| 파일 | 01 (Flask) | 02 (FastAPI) | 재사용? |
|---|---|---|---|
| **`db.py`** | 원본 | **그대로 import** | ✅ **100%** |
| `static/style.css` | 원본 | **그대로 mount** | ✅ **100%** |
| `templates/index.html` | Flask 문법 | **자체 사본** | ❌ `url_for` 문법이 다름 |
| `app.py` / `main.py` | Flask | FastAPI | ❌ 전부 다름 |

**`db.py`를 진짜로 그대로 쓴다:**

```python
# 02_fastapi_sqlite/main.py
FLASK_DIR = os.path.join(os.path.dirname(BASE_DIR), "01_flask_sqlite")
sys.path.insert(0, FLASK_DIR)
import db          # ★ 01 의 db.py 를 그대로
```

**검증했다:**
```bash
curl localhost:8000/health
→ {"status":"ok","entries":5}       # 같은 DB 파일을 읽는다
```

**결론**: **웹 서버를 바꿔도 DB 계층은 안 바뀐다.**

#### 실험 ② DB만 바꾼다 — `03_flask_mongodb`

**`app.py`에서 다른 줄은 딱 하나:**

```python
# 01_flask_sqlite/app.py
import db

# 03_flask_mongodb/app.py
import db_mongo as db          # ★ 이 줄만
```

**나머지 라우트·검증·PRG·에러처리는 글자 하나 안 바뀌었다.**
관계형 → 문서형으로 DB를 통째로 갈아엎었는데도.

**가능한 이유 — 함수 시그니처가 같다:**

```
list_entries    sqlite(limit=100)      mongo(limit=100)      ✅ 동일
add_entry       sqlite(name, message)  mongo(name, message)  ✅ 동일
delete_entry    sqlite(entry_id)       mongo(entry_id)       ✅ 동일
count_entries   sqlite()               mongo()               ✅ 동일
init_db         sqlite()               mongo()               ✅ 동일
```

**이걸 "인터페이스가 같다"고 한다.** 속은 완전히 다른데 겉이 같으니 갈아끼울 수 있다.

> **[광명테크 프로젝트](../광명테크/README.md)의 `detector.py`가 같은 발상**이다:
> ```python
> @dataclass
> class Detection:      # ← 검출기가 뭐든 이 형태로 내놓는다
>     class_id: int
>     class_name: str
>     ...
> ```
> **"검출기가 바뀌어도 `infer(frame) -> list[Detection]`만 지키면 나머지 층은 안 바뀐다."**
> 같은 원리를 DB에 적용한 것이다.

#### 무엇이 재사용되고 무엇이 안 되나 — 정리

```
                    서버 교체        DB 교체
db.py               ✅ 그대로        ❌ 새로 짬 (하지만 인터페이스는 유지)
templates/          ❌ url_for 다름  ✅ 그대로
static/style.css    ✅ 그대로        ✅ 그대로
app.py              ❌ 전부 다름     ✅ import 한 줄만
```

**CSS가 가장 자유롭다** — 프레임워크도 DB도 모르니까.
**`app.py`와 `db.py`가 서로에게서 자유롭다** — 계층을 나눈 덕분에.

#### MongoDB 판의 실제 차이

**같은 인터페이스인데 속은 이렇게 다르다:**

```python
# SQLite — 자동증가가 공짜
id INTEGER PRIMARY KEY AUTOINCREMENT

# MongoDB — 직접 만들어야 한다
def _next_seq():
    last = col.find_one(sort=[("seq", DESCENDING)])
    return (last["seq"] + 1) if last else 1
```

```python
# SQLite — 요청마다 열고 닫는다 (파일이라서)
def get_connection():
    conn = sqlite3.connect(DB_PATH)
    ...

# MongoDB — 전역에 하나 두고 재사용 (연결 풀이 있어서)
_client = None
def get_client():
    global _client
    if _client is None:
        _client = MongoClient(MONGO_URL)
    return _client
```

**그리고 서버DB라서 "연결 실패"를 다뤄야 한다:**

```python
@app.errorhandler(ServerSelectionTimeoutError)
def handle_mongo_down(e):
    return "<h1>MongoDB 에 연결할 수 없습니다</h1>...", 503
```

**SQLite는 파일이라 이런 게 필요 없다.**

> **⚠ 정직하게 밝힌다**: `03_flask_mongodb`는 **MongoDB 서버 없이 검증했다.**
> 확인한 것: 임포트 성공, 01과 함수 시그니처 완전 일치, 라우트 등록,
> **MongoDB가 꺼졌을 때 503과 안내 메시지가 정확히 나오는 것**.
> **확인 못 한 것**: 실제 MongoDB에 붙어 CRUD가 도는 것.
> MongoDB를 설치한 뒤 `py db_mongo.py init`부터 실행해 보시라.
