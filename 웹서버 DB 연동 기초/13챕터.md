#### 화면과 API를 둘 다 만들었다

```
화면 (사람이 봄)              API (기계가 읽음)
─────────────────            ─────────────────
GET  /                        GET    /api/entries
POST /add                     POST   /api/entries
POST /delete/<id>             DELETE /api/entries/<id>
                              GET    /health
```

**같은 데이터, 다른 표현**이다. HTML은 사람용, JSON은 프로그램용.

#### REST 규칙 — 동사를 URL이 아니라 메서드로

```
❌ 나쁜 설계                     ✅ REST
POST /add                       POST   /api/entries        (생성)
GET  /delete/5                  DELETE /api/entries/5      (삭제)
GET  /getEntries                GET    /api/entries        (조회)
```

**URL은 "무엇"(명사), 메서드는 "어떻게"(동사)**다.

| 메서드 | 뜻 | 안전한가 |
|---|---|---|
| **GET** | 읽기 | ✅ 여러 번 해도 변화 없음 |
| **POST** | 생성 | ❌ 할 때마다 늘어남 |
| **PUT** | 전체 수정 | 여러 번 해도 결과 같음 |
| **DELETE** | 삭제 | 여러 번 해도 결과 같음 |

#### ★ GET으로 삭제하면 안 되는 이유

```python
# app.py — 삭제를 POST 로 만들었다
@app.route("/delete/<int:entry_id>", methods=["POST"])
```

**GET으로 만들면:**

```
검색엔진 크롤러가 페이지를 긁는다
   → <a href="/delete/1">삭제</a> 링크를 발견
   → 링크를 따라간다 (크롤러는 GET 이 안전하다고 믿는다)
   → 글이 전부 지워진다 💥
```

**실제로 자주 일어나는 사고다.** 브라우저 프리페치나 백신도 링크를 미리 방문한다.

**"GET은 안전해야 한다"는 건 약속**이고, 세상이 그 약속을 믿고 동작한다.

#### 상태 코드 — 숫자로 결과를 말한다

**이 실습이 실제로 쓰는 것:**

| 코드 | 뜻 | 언제 |
|---|---|---|
| **200** | OK | 조회 성공 |
| **201** | **Created** | **생성 성공** ← 200이 아니다 |
| **302 / 303** | 리다이렉트 | PRG (10단계) |
| **400** | Bad Request | **클라이언트 잘못** (빈 값) |
| **404** | Not Found | 없는 id |
| **422** | Unprocessable | FastAPI의 검증 실패 |
| **500** | Server Error | **서버 잘못** (버그) |
| **503** | Unavailable | **DB 연결 실패** (MongoDB 꺼짐) |

**전부 실제로 테스트했다:**

```bash
POST /api/entries {"name":"curl",...}     → 201 ✅
POST /api/entries {"name":"", ...}        → 400 ✅
DELETE /api/entries/99999                 → 404 ✅
GET /health (MongoDB 꺼짐)                → 503 ✅
```

#### 400과 500을 구분하는 게 중요하다

```
4xx = 클라이언트 잘못    → "네가 잘못 보냈어. 고쳐서 다시 보내"
5xx = 서버 잘못          → "내 잘못이야. 그대로 다시 보내봐"
```

**전부 500으로 뭉뚱그리면** 클라이언트가 재시도할지 포기할지 판단할 수 없다.

```python
# 빈 값은 클라이언트 잘못이다
if not name or not message:
    return jsonify({"error": "name 과 message 는 필수입니다"}), 400   # 500 아님!
```

#### `/health` — 배포하면 반드시 필요해진다

```python
@app.route("/health")
def health():
    try:
        return jsonify({"status": "ok", "entries": db.count_entries()})
    except Exception as e:
        return jsonify({"status": "error", "detail": str(e)}), 500
```

**로드밸런서·모니터링이 주기적으로 찔러 본다.**
`count_entries()`를 부르므로 **DB까지 살아있는지** 확인된다 — 단순히 "서버가 떴나"가
아니라 **"실제로 일할 수 있나"**를 본다.
