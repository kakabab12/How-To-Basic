# 2챕터 — 전체 그림 — 브라우저에서 DB까지

> **웹 서버 + DB 연동 기초** · 2/15챕터 · [📚 목차](README.md)

---

**주소창에 `localhost:5000`을 치면 무슨 일이 일어나나?**

```
 ┌──────────┐
 │ 브라우저  │  ① "GET / 주세요"  (HTTP 요청)
 └────┬─────┘
      │
      ▼  TCP 5000번 포트
 ┌────────────────────────────────────────┐
 │  웹 서버 (Flask)                        │
 │                                        │
 │  ② 어느 함수가 처리할지 찾는다 (라우팅)   │
 │       @app.route("/") → index()        │
 │                                        │
 │  ③ 함수 실행                            │
 │       entries = db.list_entries()  ────┼──┐
 │                                        │  │
 │  ⑤ HTML 만들기 (템플릿)                 │  │
 │       render_template("index.html")    │  │
 └────┬───────────────────────────────────┘  │
      │                                      │ ④ SQL
      │                                      ▼
      │                            ┌──────────────────┐
      │                            │  DB (SQLite)     │
      │                            │  SELECT * FROM   │
      │                            │  guestbook       │
      │                            └──────────────────┘
      │
      ▼  ⑥ HTML 응답
 ┌──────────┐
 │ 브라우저  │  ⑦ 화면에 그린다
 └──────────┘
```

#### 각 단계에서 실제로 일어나는 일

| # | 무엇 | 이 실습의 코드 |
|---|---|---|
| ① | 브라우저가 HTTP 요청 | (브라우저가 함) |
| ② | **라우팅** — 주소 → 함수 | `@app.route("/")` |
| ③ | **핸들러** 실행 | `def index():` |
| ④ | **DB 질의** | `db.list_entries()` → `SELECT ...` |
| ⑤ | **템플릿 렌더링** — 데이터 + HTML 틀 | `render_template("index.html", entries=...)` |
| ⑥ | HTTP 응답 | `return` |
| ⑦ | 브라우저가 렌더링 | (브라우저가 함) |

#### 왜 이걸 먼저 외워야 하나

**"웹 서버 종류"와 "DB 종류"는 이 그림의 두 상자를 뭘로 채우냐의 문제**다.

```
       [웹 서버 상자]              [DB 상자]
   Apache / Nginx / IIS       SQLite / MySQL / PostgreSQL
   Flask / FastAPI / Express  MongoDB / Redis / Access
   Spring / Django / ASP      ...
```

**그림은 안 바뀐다. 상자 안의 부품만 바뀐다.**
그래서 하나를 제대로 알면 나머지는 금방 배운다 — 그게 이 실습이 같은 앱을
3벌 만드는 이유다.
