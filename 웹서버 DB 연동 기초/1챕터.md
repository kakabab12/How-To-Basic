#### 사실 처음이 아니다

이 저장소를 훑어보면 **웹 서버와 DB를 이미 여러 번 다뤘다**:

| 시기 | 프로젝트 | 웹 서버 | DB |
|---|---|---|---|
| **2020** | [Express 지역 도서관](../개인%20프로젝트/2020_express-locallibrary-tutorial/README.md) | **Express** (Node.js) | **MongoDB** (mongoose) |
| **2024** | [ASP + Access](../개인%20프로젝트/2024년%202학기ASP_DB연동/README.md) | **IIS** + 클래식 ASP | **Access** (.mdb) |
| **2025** | [XAMPP 웹 과제](../개인%20프로젝트/xampp%20웹%20자기소개서%20간단하게/README.md) | **Apache** (XAMPP) | 없음 (정적) |
| **2025** | [라즈베리파이 관제](../2025%202학기%20라즈베리파이%20객체인식%20아두이노%20분류%20동장/README.md) | **Flask** | 없음 (메모리 `deque`) |
| **2026** | [젯슨 관제](../2026%201학기%20젯슨나노%20객체인식%20분류+르로봇/README.md) | **FastAPI** + uvicorn | 없음 (메모리 리스트) |
| **2026** | [메디컬 서비스](../메디컬%20머신러닝%20대회/README.md) | **Flask** | 없음 (`.pkl` 파일) |

**Apache, IIS, Express, Flask, FastAPI를 이미 다 써봤다.** MongoDB와 Access도 만졌다.

#### 그런데 빠진 게 있다

**2024년 ASP 이후로 DB를 안 썼다.** 라즈베리파이·젯슨·메디컬은 전부:

```python
detection_log = deque(maxlen=20)     # 라즈베리파이 — 메모리
action_log    = deque(maxlen=20)
detection_log_1 = []                 # 젯슨 — 메모리
```

**서버를 끄면 전부 사라진다.** 그게 관제 화면용이라 문제가 아니었을 뿐이다.

> **이 폴더가 채우는 빈칸**: "웹 서버가 받은 데이터를 **어떻게 영구히 저장하나**".
> 라즈베리파이가 하루치 불량률을 기록하려면, 젯슨이 지난주 처리량을 보여주려면,
> **DB가 필요하다.**

#### 왜 방명록인가

**2024년 ASP 실습의 `c` 시리즈와 같은 것**이다:

```asp
' cwrite.asp (2024)
SQL = "INSERT INTO IdeaTime (Idea) VALUES ('"& X &"')"
```

```python
# 이 실습 (2026)
db.execute("INSERT INTO guestbook (name, message) VALUES (?, ?)", (name, message))
```

**같은 일인데 2년 뒤에 제대로 하는 것**이다. 그리고 위 두 줄의 차이가
[12단계](#12단계)의 주제다 — 2024년 코드는 **SQL 인젝션에 뚫린다.**
