#### 웹 서버 고르기

```
파이썬을 쓴다
├── 배우는 중이다 / 작은 API          → Flask
├── API 위주 + 자동 문서 + 성능       → FastAPI
└── 관리자화면·인증이 통째로 필요      → Django

이미 JS를 안다                        → Express (Node.js)
대기업·금융권                          → Spring Boot (Java)
```

**앞에 세울 것:**
```
개발 중        → app.run() / uvicorn --reload   (그대로)
실서비스       → Nginx + gunicorn/uvicorn        (5단계)
```

#### DB 고르기

```
데이터를 저장해야 한다
│
├── 혼자 쓰는 앱 / 프로토타입 / 설치를 못 함
│      → SQLite                      ★ 이 실습
│
├── 웹서비스, 여러 명이 동시에 씀
│   ├── 잘 모르겠다                   → PostgreSQL   ★ 기본값
│   ├── 이미 MySQL 을 안다            → MySQL
│   └── Windows·.NET 환경            → MS SQL Server
│
├── 구조가 자주 바뀜 / 문서 덩어리
│      → MongoDB
│
├── 빠른 캐시·세션                     → Redis (다른 DB와 함께)
│
└── 센서 데이터·모니터링               → InfluxDB / TimescaleDB
```

#### 이 저장소의 프로젝트라면 뭘 골랐어야 하나

| 프로젝트 | 지금 | 골랐다면 | 왜 |
|---|---|---|---|
| [라즈베리파이 관제](../2025%202학기%20라즈베리파이%20객체인식%20아두이노%20분류%20동장/README.md) | `deque` (휘발) | **SQLite** | 라즈베리파이에 서버DB는 무겁다. 파일 하나면 충분 |
| [젯슨 관제](../2026%201학기%20젯슨나노%20객체인식%20분류+르로봇/README.md) | 리스트 (휘발) | **SQLite** 또는 InfluxDB | 시계열이면 후자 |
| [메디컬 서비스](../메디컬%20머신러닝%20대회/README.md) | `.pkl` | **파일 그대로 OK** | 모델은 DB에 넣을 게 아니다 |
| [Express 도서관](../개인%20프로젝트/2020_express-locallibrary-tutorial/README.md) | MongoDB | **PostgreSQL** | 책·저자·장르는 **관계**가 뚜렷하다 → 관계형이 맞다 |

> **마지막이 흥미롭다.** MDN 튜토리얼이 MongoDB를 쓴 건 **가르치기 쉬워서**이지
> 도서관 데이터에 맞아서가 아니다. 책↔저자↔장르는 **전형적인 관계형 데이터**다.

#### 고르지 말아야 할 이유들

| 나쁜 이유 | 왜 나쁜가 |
|---|---|
| "요즘 뜨니까" | 유행은 3년이면 바뀐다 |
| "빠르다니까" | 당신 규모에선 **아무거나 충분히 빠르다** |
| "대기업이 쓰니까" | 그들은 **당신과 문제가 다르다** |

**좋은 이유:**
- 팀이 이미 안다
- 문서·예제가 많다
- **내 데이터 모양에 맞는다**
- 운영할 수 있다
