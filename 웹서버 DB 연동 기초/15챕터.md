#### 준비 — 가상환경 (권장)

```bash
cd "웹서버 DB 연동 기초/실습코드"
py -m venv .venv
.venv\Scripts\activate          # Windows
```

**왜 가상환경을 쓰나**: 프로젝트마다 패키지 버전이 다르다.
전역에 깔면 A 프로젝트가 요구하는 Flask 2와 B의 Flask 3이 충돌한다.

> **`python`이 아니라 `py`를 쓴다.** 이 PC의 `python`은 Microsoft Store 스텁이라
> 아무것도 하지 않는다. 실제 파이썬은 **`py` 런처**(3.13.14)다.

#### ① Flask + SQLite (기본) — ✅ 전 기능 검증됨

```bash
cd 01_flask_sqlite
pip install -r requirements.txt      # Flask 하나뿐
py db.py init                        # ← 처음 한 번 (DB 생성 + 샘플 3건)
py app.py                            # → http://localhost:5000
```

**확인해 볼 것:**
```bash
curl localhost:5000/health           # {"status":"ok","entries":3}
curl localhost:5000/api/entries      # JSON 목록
```

#### ② FastAPI + SQLite — ✅ 전 기능 검증됨

```bash
cd 02_fastapi_sqlite
pip install -r requirements.txt
py -m uvicorn main:app --reload --port 8000
# → http://localhost:8000
# → http://localhost:8000/docs      ★ 자동 API 문서
```

**`/docs`를 꼭 열어보시라.** Flask 판에는 없는 것이고, FastAPI를 쓰는 가장 큰 이유다.

> **01의 DB를 그대로 쓴다.** `01_flask_sqlite/guestbook.db`를 읽으므로
> **①을 먼저 `py db.py init` 해 둬야** 한다.

#### ③ Flask + MongoDB — ⚠ MongoDB 설치 필요

```bash
# 1. MongoDB 설치
#    https://www.mongodb.com/try/download/community
# 2. 실행
net start MongoDB                    # 서비스로 설치했다면
# 또는
mongod --dbpath C:\data\db

# 3. 앱 실행
cd 03_flask_mongodb
pip install -r requirements.txt
py db_mongo.py init
py app.py                            # → http://localhost:5001
```

**MongoDB 없이 실행하면** 503과 안내 메시지가 뜬다(죽지 않는다).

#### 포트 정리

| 앱 | 포트 |
|---|---|
| 01 Flask + SQLite | **5000** |
| 02 FastAPI + SQLite | **8000** |
| 03 Flask + MongoDB | **5001** |

**같은 포트를 두 프로그램이 못 쓴다** — `Address already in use` 에러의 원인이다.
셋 다 동시에 띄울 수 있게 일부러 다르게 뒀다.

#### ⚠ 실서비스에 올린다면 (지금 코드로는 안 된다)

| # | 문제 | 고칠 것 |
|---|---|---|
| 1 | **`debug=True`** | **반드시 끈다.** 소스 노출 + 원격 코드 실행 위험 |
| 2 | **`app.run()`** | gunicorn/waitress + Nginx (5단계) |
| 3 | **CSRF 방어 없음** | `Flask-WTF` — 남의 사이트에서 내 폼을 제출하는 공격 |
| 4 | **인증 없음** | 누구나 남의 글을 지울 수 있다 |
| 5 | **rate limit 없음** | 스팸 폭탄에 무방비 |
| 6 | **SQLite 동시 쓰기** | 트래픽이 있으면 PostgreSQL로 |
| 7 | **시간대** | `CURRENT_TIMESTAMP`가 UTC (11단계) |

> **이 실습의 목적은 "원리를 이해하는 것"**이지 "서비스 배포"가 아니다.
> 위 7개는 **다음 단계**의 목록이다.

#### 다음에 배울 것 — 순서 추천

```
1. UPDATE 추가          ← CRUD 의 U. 수정 폼을 만들어 보기
2. 페이지네이션          ← LIMIT / OFFSET. 글이 1000개면?
3. 검색                  ← WHERE message LIKE ?  (바인딩 잊지 말 것!)
4. 테이블 2개 + JOIN     ← users 와 guestbook 을 나누고 연결하기
5. ORM (SQLAlchemy)      ← SQL 을 파이썬 객체로
6. 인증 (로그인)          ← 세션·쿠키·비밀번호 해싱
7. 마이그레이션 (Alembic) ← 스키마가 바뀌면 기존 데이터는?
8. 배포 (Nginx + gunicorn)
```

**4번(JOIN)이 관계형 DB의 진짜 시작**이다. 지금은 테이블이 하나라
"관계형"의 관계가 안 나온다.
