#### 파일 구조

```
01_flask_sqlite/
├── app.py              ← 웹 계층 (SQL 이 한 줄도 없다)
├── db.py               ← DB 계층 (HTTP 를 모른다)
├── schema.sql          ← 스키마 (11단계)
├── templates/
│   └── index.html      ← 화면
├── static/
│   └── style.css
└── requirements.txt
```

#### 계층을 나눈 이유

```
┌─────────────────────────────────────┐
│  app.py     "HTTP 요청을 받아       │
│              무엇을 할지"            │
│                                     │
│    @app.route("/")                  │
│    def index():                     │
│        entries = db.list_entries()  │ ← DB 를 "부를" 뿐
│        return render_template(...)  │   SQL 을 모른다
└───────────────┬─────────────────────┘
                │
┌───────────────▼─────────────────────┐
│  db.py      "데이터를 어떻게         │
│              저장/조회할지"          │
│                                     │
│    def list_entries():              │
│        SELECT id, name, ...         │ ← SQL 이 여기만 있다
│                                     │   HTTP 를 모른다
└─────────────────────────────────────┘
```

**이렇게 나누면 얻는 것:**

| 이점 | 증거 |
|---|---|
| **DB 교체가 쉽다** | `03_flask_mongodb`는 `import db_mongo as db` **한 줄만** 다르다 |
| **서버 교체가 쉽다** | `02_fastapi_sqlite`가 `db.py`를 **그대로** 재사용 |
| **테스트가 쉽다** | 웹 서버 없이 `db.py`만 불러서 검증 가능 |
| **읽기 쉽다** | `app.py`만 봐도 "무엇을 하는지" 안다 |

**실제로 웹 서버 없이 DB만 테스트했다:**

```bash
py -c "import db; print(db.count_entries())"
# → 3
```

> **[ASP 프로젝트](../개인%20프로젝트/2024년%202학기ASP_DB연동/README.md)와 정반대**다.
> 거기선 `insert.asp` 하나에 **연결·SQL·리다이렉트가 다 섞여** 있었고,
> **DB 연결 코드가 20개 파일에 복사**되어 있었다. DB 이름을 바꾸려면 20군데를 고쳐야 했다.
> 여기는 `db.py` 한 곳만 고치면 된다.

#### `app.py` 훑어보기

```python
from flask import Flask, jsonify, redirect, render_template, request, url_for
import db                                    # ← DB 계층

app = Flask(__name__)

@app.route("/")
def index():
    entries = db.list_entries()
    return render_template("index.html", entries=entries, count=len(entries))

@app.route("/add", methods=["POST"])
def add():
    name = (request.form.get("name") or "").strip()
    message = (request.form.get("message") or "").strip()
    if not name or not message:              # ★ 서버 검증
        return redirect(url_for("index"))
    db.add_entry(name, message)
    return redirect(url_for("index"))        # ★ PRG
```

**두 가지가 중요하다:**

**① 서버에서 반드시 다시 검증한다**

```html
<input type="text" name="name" required maxlength="50">
```

**HTML의 `required`는 브라우저에서 얼마든지 우회된다.** 개발자도구로 속성을 지우거나,
curl로 직접 쏘면 그만이다. **검증은 반드시 서버에서 한 번 더.**

**② PRG 패턴 — 리다이렉트로 끝낸다**

```python
db.add_entry(name, message)
return redirect(url_for("index"))    # ← HTML 을 돌려주지 않는다
```

**왜:**
```
리다이렉트 없이 HTML 반환
   → 사용자가 F5
   → 브라우저: "양식을 다시 제출하시겠습니까?"
   → 같은 글이 또 등록됨 💥

리다이렉트하면
   → F5 는 그냥 GET /
   → 안전 ✅
```

**실제 서버 로그로 확인했다:**
```
"POST /add HTTP/1.1" 302 -      ← 리다이렉트
"GET / HTTP/1.1" 200 -          ← 목록 다시 로드
```

> **[ASP 프로젝트](../개인%20프로젝트/2024년%202학기ASP_DB연동/README.md)의
> `Response.Redirect "insert.htm"`이 같은 패턴**이었다. 의도했든 아니든 **결과적으로 옳았다.**

#### `db.py` 훑어보기

```python
def get_connection():
    conn = sqlite3.connect(DB_PATH)
    conn.row_factory = sqlite3.Row        # ★ 컬럼 이름으로 접근
    conn.execute("PRAGMA foreign_keys = ON")
    return conn
```

**`row_factory = sqlite3.Row`가 편의의 핵심이다:**

```python
row[0]          # 기본 — 컬럼 순서가 바뀌면 깨진다
row["name"]     # Row 사용 — 이름으로 접근, 안 깨진다
```

**경로를 파일 기준으로 잡는 것도 중요하다:**

```python
BASE_DIR = os.path.dirname(os.path.abspath(__file__))
DB_PATH = os.path.join(BASE_DIR, "guestbook.db")
```

**상대 경로(`"guestbook.db"`)로 두면 실행 위치에 따라 DB 파일이 딴 데 생긴다.**

> **같은 문제를 이 저장소가 세 번 풀었다:**
> - ASP: `server.mappath("nwind.mdb")`
> - 광명테크: `config_loader.py`의 `root_dir` 계산
> - 여기: `os.path.dirname(os.path.abspath(__file__))`
>
> **"상대 경로를 절대 경로로 고정"은 언어·시대를 안 가리는 문제**다.
