# 4챕터 — 웹 서버 종류 ② 애플리케이션 프레임워크 (Flask · FastAPI · Express …)

> **웹 서버 + DB 연동 기초** · 4/15챕터 · [📚 목차](README.md)

---

**3단계가 "파일 배달"이라면, 이건 "코드 실행"**이다.

```
요청: GET /
   ↓
함수를 실행한다:
   entries = db.list_entries()      ← DB 조회
   return render_template(...)      ← HTML 생성
   ↓
만들어진 HTML 을 응답
```

**디스크에 `/`라는 파일은 없다.** 매 요청마다 **새로 만든다.**

#### 주요 프레임워크

| | 언어 | 성격 | 특징 |
|---|---|---|---|
| **Flask** | Python | **마이크로** | 최소한만 준다. 필요한 걸 골라 붙인다 |
| **FastAPI** | Python | 현대적 | **타입 힌트 → 자동 검증·문서**. 비동기 |
| **Django** | Python | **풀스택** | ORM·관리자화면·인증 전부 내장 |
| **Express** | JavaScript | **마이크로** | Node.js 표준. Flask와 성격이 같다 |
| **Spring Boot** | Java | 풀스택 | 대기업·금융권 표준 |
| **클래식 ASP** | VBScript | (구식) | 1996년. 지금은 안 쓴다 |

#### 마이크로 vs 풀스택

**이 축이 가장 중요하다.**

```
마이크로 (Flask, Express)          풀스택 (Django, Spring)
────────────────────────          ──────────────────────
라우팅만 준다                       라우팅 + ORM + 인증 + 관리자화면
                                   + 폼 + 마이그레이션 + ...
DB? 네가 골라                       DB? 우리 ORM 써
구조? 네 맘대로                     구조? 우리 방식 따라

작게 시작하기 좋음                   큰 프로젝트에 좋음
자유롭다 = 매번 정해야 한다          정해져 있다 = 빨리 만든다
```

> **이 실습이 Flask를 쓴 이유**: 라우팅 말고는 아무것도 안 해 주니까
> **DB 연결·SQL·계층 분리를 직접 하게 된다.** 배우는 게 목적이면 그게 맞다.
> Django를 쓰면 ORM이 SQL을 다 가려서 **SQL을 안 배우게 된다.**

#### Flask vs FastAPI — 실습으로 확인한 차이

**`02_fastapi_sqlite`가 `01_flask_sqlite`와 같은 앱**이다. 실제로 다른 점:

**① 입력 검증**

```python
# Flask — 직접 꺼내고 직접 검증
data = request.get_json(silent=True) or {}
name = (data.get("name") or "").strip()
if not name or not message:
    return jsonify({"error": "..."}), 400
```

```python
# FastAPI — 선언만 하면 끝
class EntryIn(BaseModel):
    name: str = Field(min_length=1, max_length=50)
    message: str = Field(min_length=1, max_length=500)

def api_add(entry: EntryIn):     # ← 검증 코드가 0줄
    ...
```

**실제로 테스트해 봤다:**

```bash
curl -X POST localhost:8000/api/entries -d '{"name":"","message":"x"}'
→ HTTP 422    # FastAPI 가 자동으로 막았다. 우리 코드는 실행조차 안 됨
```

**② 자동 API 문서**

```
http://localhost:8000/docs      ← FastAPI 가 자동 생성. Flask 에는 없다
```

브라우저에서 **API를 바로 테스트**할 수 있는 화면이 공짜로 생긴다.

**③ 서버가 따로 필요하다**

```bash
py app.py                              # Flask — 이걸로 끝 (서버 내장)
py -m uvicorn main:app --reload        # FastAPI — uvicorn 이 따로 필요
```

**④ 템플릿 문법이 미묘하게 다르다** (실제로 걸렸다)

```html
<!-- Flask -->
<link href="{{ url_for('static', filename='style.css') }}">
<!-- FastAPI (Starlette) -->
<link href="{{ url_for('static', path='style.css') }}">
```

> **`filename=` vs `path=`.** 같은 Jinja2인데 인자 이름이 다르다.
> 그래서 **템플릿은 프레임워크를 바꾸면 같이 고쳐야 한다** —
> 반면 **`db.py`는 한 글자도 안 고쳤다**(14단계).

> Flask에서 FastAPI로 갈아탄 이유**가 이제 보인다. Pydantic 모델
> (`RobotRequest`, `ClaudeRequest`)로 요청을 선언하면 검증이 공짜다.
> 엔드포인트가 늘수록 이득이 커진다.
