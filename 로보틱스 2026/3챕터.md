# 3챕터 — 문제 1 — 문제 정의와 시스템 아키텍처

> **로보틱스 2026** · 3/12챕터 · [📚 목차](README.md)

---

`1_definition.md` — **가장 중요한 문서**다. 나머지 8개가 여기서 파생된다.

#### 문제 정의

> 김대리는 부품과 제품을 공정 간에 운송하는 과정을 자동화하는 프로토타입 운반 로봇을
> 개발하고자 하며, 이 로봇은 바닥에 그려진 검은색 선(경로)을 인식하여 자율적으로
> 이동하고, 이동 기록 데이터를 수집하는 것이 목표로 한다

**시나리오 형식**이다. "라인트레이서를 만드시오"가 아니라 **"김대리가 왜 필요한지"**부터
시작한다. 요구사항이 어디서 나오는지를 잊지 않게 하는 서술 방식이다.

#### 요구사항 정리 — 5개

```
1. 바닥의 검은색 선(경로)을 인식할 수 있어야 한다.
2. 인식된 경로를 따라 자율적으로 이동할 수 있어야 한다.
3. 이동 경로와 시간, 위치 데이터를 기록하고 저장할 수 있어야 한다.
4. 실시간으로 데이터를 수집하고 저장하는 기능이 필요하다.
5. 프로토타입 하드웨어와 소프트웨어를 제작할 수 있어야 한다.
```

**서술형 문제를 검증 가능한 항목으로 쪼갠 것** — 이게 요구공학의 첫 단계다.
`1`은 문제 4(센서)로, `2`는 문제 6~7(구동·제어)로, `3~4`는 아키텍처의 로깅 모듈로 이어진다.

#### 하드웨어 아키텍처 — 선정 **근거**가 달려 있다

| 부품 | 선정 | 근거 (문서 그대로) |
|---|---|---|
| 메인 제어기 | **NVIDIA Jetson Orin Nano** | 영상 데이터를 실시간으로 처리하고 주행 알고리즘 연산 및 전체 시스템을 제어 |
| 스토리지 | **NVMe SSD** | 대용량 주행 로그 데이터(CSV)를 **병목 없이** 고속으로 안정성 있게 로컬에 저장 |
| 비전 센서 | **CSI 카메라** | 바닥의 검은색 주행 경로 영상을 **지연 없이** 실시간 획득 |
| 구동부 | **스테퍼 모터** | 좌우 바퀴에 장착되어 **오차 없는 정밀한** 회전 각도 및 속도 제어 |
| 모터 제어 | **모터 드라이버** | 제어 신호를 받아 스텝 모터를 구동할 **전력**을 공급 |
| 전원부 | **Li-Po + 레귤레이터** | 모터 구동용 전력과 제어기용 전압을 **분리**하여 안정적 공급 |

**부품마다 "왜"가 붙어 있는 게 핵심**이다. 세 가지가 특히 눈에 띈다:

- **NVMe SSD를 굳이 명시** — 요구사항 3·4(실시간 로깅)에서 나온 것이다. SD카드는
  대용량 연속 쓰기에서 병목이 된다.
- **전원 분리** — 모터는 기동 시 큰 전류를 당겨 전압을 떨어뜨린다. 제어기와 전원을
  공유하면 **모터가 돌 때마다 보드가 리셋**된다. 실무에서 반드시 겪는 문제다.
- **스테퍼 모터** — 문제 2에서 DC/서보/스테퍼를 비교한 결과가 여기에 반영된다.

#### 소프트웨어 아키텍처 — 3모듈 멀티스레딩

> 시스템은 크게 **영상 처리, 주행 제어, 데이터 로깅**의 세 가지 모듈로 구성되며,
> **멀티스레딩(Multi-threading)** 환경에서 병렬로 동작한다.

**비전 모듈의 처리 흐름**이 구체적이다:

```
1. CSI 카메라로부터 실시간 프레임 획득
2. 그레이스케일 변환 및 이진화(Thresholding)로 하얀 바닥과 검은 선 분리
3. 관심 영역(ROI) 설정 및 검은 선의 무게중심(Center of Mass) X 좌표 추출
4. 화면 중심(로봇의 중앙)과 선의 중심 간의 픽셀 차이를 오차(Error)로 산출
```

**4번이 문제 7(PID)로 바로 연결된다** — "픽셀 차이 = 오차 e"가 PID의 입력이다.
**문서 1편이 이미 문서 7편을 예고**하고 있다.

> **무게중심(Center of Mass) 추출**은 라즈베리파이 프로젝트에서 실제로 쓴 기법과 같다
> (`cv2.moments`로 박스의 무게중심을 구했다). 12단계 참고.
