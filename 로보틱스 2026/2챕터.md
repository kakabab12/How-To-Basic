# 2챕터 — 커리큘럼 설계 — 9문제의 흐름

> **로보틱스 2026** · 2/12챕터 · [📚 목차](README.md)

---

**9개 문제가 무작위가 아니다.** 명확한 순서가 있다:

```
  문제 1  ─────────────────────────────  왜 만드나 · 무엇으로 만드나
    │                                     (요구사항 → 아키텍처)
    ▼
  ┌────────── 부품을 하나씩 ──────────┐
  │  문제 2  모터      ← 힘을 만든다   │
  │  문제 3  바퀴      ← 힘을 전달한다 │   "몸"
  │  문제 4  센서      ← 환경을 읽는다 │
  │  문제 5  엔코더    ← 나를 읽는다   │
  └────────────────┬───────────────────┘
                   ▼
  문제 6  차동 구동  ← 두 바퀴를 어떻게 움직여 원하는 경로를 그리나   "운동학"
                   │
                   ▼
  문제 7  PID 제어   ← 오차를 어떻게 줄이나                        "제어"
                   │
                   ▼
  문제 8  MCU        ← 이걸 무엇으로 실행하나                      "구현"
                   │
                   ▼
  문제 9  제어 실습   ← 전부 합쳐서 실제 상황을 풀어 본다            "종합"
```

**설계 원리**: **감지(센서) → 판단(제어) → 구동(모터)**이라는 로봇의 기본 루프를
부품 단위로 분해했다가 문제 9에서 다시 합친다.

| 문제 | 답하는 질문 |
|---|---|
| 1 | 무엇을·왜 만드나 |
| 2~5 | 부품 각각이 어떻게 동작하나 |
| 6 | 부품들을 어떻게 조합해 **움직임**을 만드나 |
| 7 | 움직임을 어떻게 **정확하게** 만드나 |
| 8 | 이 계산을 **무엇이** 실행하나 |
| 9 | 실제 상황에서 전부 어떻게 맞물리나 |

**문제 5(엔코더)가 문제 7(PID) 앞에 오는 게 핵심**이다. 엔코더가 없으면 피드백이 없고,
피드백이 없으면 PID를 쓸 수 없다. **순서 자체가 의존 관계**다.
